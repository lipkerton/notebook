самое сложное.
регистрация кнопок, которые будут отвечать за движение персонажа.
я делал это для 2D игры, поэтому и регистрацию я буду производить с 2D вектором.
для начала я сделаю переменную типа `InputAction`:
```C#
class PlayerConroller2D : MonoBehaviour
{
	...
	private InputAction moveAction;
	...
	void Awake()
	{
		moveAction = new InputAction(
			name: "Move",
			type: InputActionType.Value,
			binding: "<Gamepad>leftStick"
		);
	}
}
```
я создаю экземпляр класса `InputAction`.
экземпляр класса `InputAction` - это простой объект, к которому привязаны клавиши, вызывающие событие. как я понял имя события я указываю в атрибуте `name`, а затем я должен перечислить все клавиши, которые в моей архитектуре будут отвечать за передвижение.
в атрибуте `type` я указал класс `InputActionType`. это класс типа `Enum` следовательно его можно использовать как хранилище для набора значений. из этого хранилища я достаю параметр `Value`. этот параметр укажет, что после получения информации о нажатых клавишах нужно объединять ее в одном значении и отдавать это значение на выход, чтобы произошло считывание ввода.
атрибут `binding` позволяет сразу указать путь до клавиши, которую нужно привязать к считыванию ввода.
после того как я сделал экземпляр класса мне еще нужно указать какое конкретно значение он будет возвращать. так как я делаю 2D игру мне нужен вектор, который состоит только из двух элементов, поэтому я попрошу именно такой:
```C#
void Awake()
{
	moveAction = new InputAction(
		name: "Move",
		type: InputActionType.Value,
		binding: "<Gamepad>leftStick"
	);
	moveAction.AddCompositeBinding("2DVector");
}
```
метод `.AddCompositeBinding()` принимает первым аргументом имя композита (пока не буду вдаваться в подробности). в данном случае `"2DVector"` будет записывать ввод с клавиш в 2D вектор - клавиши вправо и влево будут лежать на оси X, а клавиши вверх и вниз будут лежать на оси Y.
что вообще такое композит?
здесь важно понять механику двух вещей - *магнитуда* и *disambiguation*.
**магнитуда** - это что-то вроде спидометра для каждой кнопки. если кнопка нажата будет использована единица, а если кнопка отпущена, то будет использован ноль. для стиков на геймпаде используются не целые значения, потому что стик может находится под разным наклоном. магнитуда есть абсолютно у каждой клавиши по умолчанию вне зависимости привязан ли к клавише какой-то `InputAction` или нет.
**disambiguation** - это процесс использования магнитуды, чтобы определить какой из контролов должен управлять действием. это происходит, когда у `InputAction` тип `Value`/`Button` или когда у одного `InputAction` есть несколько активных привязанных клавиш в один момент времени.
короче, у нас есть действие - `InputAction` и привязанные к нему клавиши, которые управляют этим действием.
теперь, что такое композит?
с *disabiguation* проблема заключается лишь в том, что всегда выбирается одна приоритетная клавиша для управления. если я захочу пойти диагонально (влево и вверх), то я просто не смогу этого сделать, т.к. будет выбрано одно направление. композит помогает исправить эту ситуацию - он комбинирует внутри себя клавиши и выдает какой-то общий результат. в случае композита `"2DVector"` мне должен вернуться один итоговый вектор, даже если я нажму клавиши во все направления движения. итоговый вектор вычисляется из суммы соответствующих координат всех других векторов.
 прежде чем получать значение в код, мне нужно привязать кнопки:
```C#
moveAction.AddCompositeBinding("2DVector")
	.With("Up", "<Keyboard>/w")
	.With("Down", "<Keyboard>/s")
	.With("Right", "<Keyboard>/d")
	.With("Left", "<Ketboard>/a");
```

```C#
moveAction.expectedControlType("Vector2");
```
он опциональный и просто дополнительно указывает ожидаемый тип данных от экземпляра класса.
я устанавливаю систему управления в методе `Awake()`, чтобы все нужные действия проделались сразу после загрузки объекта. но у экземпляра `moveAction` есть так же два метода - `Enable()` и `Disable()`, которые отключают и включают систему управления.
```C#
private void OnEnable() {
	moveAction?.Enable();
}
private void OnDisable() {
	moveAction?.Disable();
}
```
каждый из этих методов обращается к экземпляру за активацией и деактивацией. для чего я делаю так? чтобы, если мой объект будет удален со сцены, мой экземпляр `InputAction` перестал считывать клавиши, на которые он подписан.
кстати, синтаксис `moveAction?` является примером *null-условного оператора*. он используется, когда объект может быть объектом, а может быть равен `NULL`. если он является объектом, то будет вызван метод `Enable()`, а если нет, то ничего не произойдет. так я избегаю ошибки *у NULL нет метода `Enable()`*.
