мне нужно было создать collectible, т.е. такой объект, который я могу своим персонажем внутри игры подобрать. чтобы сделать collectible мне нужно написать код, который определит как один объект будет реагировать на другой объект, т.е. нужно написать код, который заставит collectible исчезнуть при касании с персонажем.
в такой ситуации мне как раз нужен триггер.
сначала я создам объект.
затем, т.к. мне нужно, чтобы один объект физически реагировал на другой я накину коллайдер на новый объект. и как раз в параметрах коллайдера найду чекбокс `IsTrigger`:
![[col1.png]]
если просто включить чекбокс, то коллайдер при физическом взаимодействии с любым другим объектом будет пропускать объект через себя, но при этом зарегистрирует "попадание".
т.к. объект регистрирует попадание, я могу написать какую-то логику в коде на случай, когда такая регистрация произойдет.
например, я могу использовать метод `Destroy()`, чтобы уничтожить объект:
```C#
private void OnTriggerEnter(Collider other) 
{
	Destroy(gameObject);
}
```
или я могу уничтожить объект и применить какой-нибудь эффект. например, сделать какие-нибудь искры и т.д.
такие эффекты называются **VFX** = *Visual Effects*. чтобы применить VFX к объекту нужно сначала написать новый параметр для объекта с помощью установки атрибута класса:
```C#
public class Collectible : MonoBehaviour
{
	...
	public GameObject onCollectEffect; 
	...
}
```
`GameObject` - это контейнер, который объединяет все элементы текущего объекта внутри себя. грубо говоря `GameObject` - это репрезентация панели инспектора в коде. поэтому когда я хочу создать новую переменную для хранения VFX я создам ее именно с типом `GameObject` и это позволит мне увидеть поле для заполнения в инспекторе.
![[game_object.png]]
вообще поле должно быть пустым, но тут я уже добавил объект, так что.
после того как объект добавлен нужно прописать логику его включения - эффект не сработает просто так. это я опять сделаю в `OnTriggerEnter()`:
```C#
private void OnTriggerEnter(Collider other) 
{
	...
	Instantiate(onTriggerEffect, transform.position, transform.rotation);
	...
}
```
функция `Instantiate()` клонирует объект и создает его реальную копию на сцене. в данном случае я клонирую `onTriggerEffect` - значение параметра для VFX, который я объявил ранее. далее два параметра указывают позицию на сцене где объект должен находиться.
на данный момент эффект будет появляться как только предмет столкнется с *любым* объектом. но это не совсем то, что мне нужно, т.к. представим, что у меня в игре есть еще NPC или вражеские юниты - они тоже являются объектами, поэтому тоже могут стриггерить collectable.
чтобы исправить такую ситуацию нужно обратить внимание на тот аргумент, который ожидает принять функция `OnTriggerEnter()`:
```C#
private void OnTriggerEnter(Collider other) {...};
```
функция ожидает получить на вход `Collider`. это логично, ведь наш объект должен стриггериться чужим коллайдером, следовательно, он *регистрирует* этот коллайдер и может просматривать или менять его свойства внутри себя.
сейчас мне важно сделать так, чтобы только игрок триггерил все события, которые я описал внутри `OnTriggerEnter()`. для этого я использую конструкцию `if`:
```C#
private void OnTriggerEnter(Collider other)
{
	if (other.CompareTag("Player"))
	{
		Destroy(gameObject);
		Instantiate(onTriggerEffect, transform.position, transform.rotation);
	}
}
```
как видно здесь я использую метод класса `CompareTag()` - он сравнит тэг объекта входящего коллайдера с указанным в скобках. если они совпадут, выполнится вся логика внутри фигурных скобок.
