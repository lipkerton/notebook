input system - это система, которая является интерфейсом для считывания ввода с клавиатуры в коде. в unity их две - старая и новая. у обеих разный синтаксис, но при этом нельзя просто поменять систему прямо в коде, изменив синтаксис. когда я попытался так сделать, то сразу столкнулся с ошибкой:
![[inputsys1.png]]
на самом деле система меняется в настройках проекта:
![[project_settings.png]]
можно использовать обе системы через параметр `Both`, но я хочу попробовать адаптировать мой вариант кода под новую систему.
проблематичный код указан тут:
```C#
private void Update()
{
	if (Input.GetButtonDown("Jump"))
	{
		rb.AddForce(Vector3.up * jumpForce, ForceMode.VelocityChange);
	}
}
```
`Input.GetButtonDown("Jump")` - устаревшая система. пример новой системы описан тут:
```C#
if (Keyboard.current.wKey.isPressed || Keyboard.current.upArrowKey.isPressed) moveInput.y = 1f;
```
в новой системе я обращаюсь к объекту клавиатуры `Keyboard`, получаю его текущее состояние через `current` и обращаюсь к клавише `wKey`, чтобы спросить нажата ли она. то же самое я сделаю и тут:
```C#
private void Update()
{
	if (Keyboard.current.spaceKey.wasPressedThisFrame)
	{
		rb.AddForce(Vector3.up * jumpForce, ForceMode.VelocityChange);
	}
}
```
атрибут `wasPressedThisFram` содержит булевское значение - если клавиша была нажата в этом кадре, то условие будет выполнено, а если нет, то не будет.
с этим все же есть небольшая проблема - клавиша проверяется раз в кадр, значит, что каждый кадр я могу снова нажать клавишу. это создает ситуацию, в которой я могу нажать клавишу несколько раз подряд и объект начнет подниматься, как по лестнице вверх. думаю это фиксится дополнительными ограничениями.
