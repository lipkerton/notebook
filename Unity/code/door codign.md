мне нужно сделать так, чтобы дверь реагировала на объекты, которые в нее врезаются.
для этого есть такой скрипт:
```C#
using UnityEngine;


public class DoorOpener : MonoBehaviour
{
   private Animator doorAnimator;

   void Start()
   {
       doorAnimator = GetComponent<Animator>();
   }

   private void OnTriggerEnter(Collider other)
   {
       if (other.CompareTag("Player"))
       {
           if (doorAnimator != null)
           {
               doorAnimator.SetTrigger("Door_Open");
           }
       }
   }
}
```
что тут происходит?
по задаче мне нужно сделать анимацию. поэтому внутри класса `DoorOpener` сразу объявляю `private Animator doorAnimator`. `Animator` отвечает за все анимации внутри Unity. поэтому именно к нему я обращаюсь, если хочу настроить какой-либо скрипт анимации.
в методе `Start()` я извлекаю из объекта двери свойство `Animator` и отдаю его в переменную `doorAnimator`.
после делаю условную конструкцию, чтобы дверь реагировала на столкновением с коллайдером игрока.
этот скрипт интересен в первую очередь использованием метода `Animator.SetTrigger()`. чтобы понять, что происходит нужно понимать что такое триггер.
**триггер** с точки зрения *Animator Controller* - это событие, которое срабатывает моментально сразу же как только устанавливается. триггер активирует переход между анимациями и сразу же исчезает.
в данном случае я устанавливаю триггер, который переводит дверь из одной анимации к анимации открытия.
