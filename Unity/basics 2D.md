теперь я хочу написать скрипт, который контролирует движения персонажа.

теперь я напишу первый метод - `Awake()`.
метод будет вызван после прогрузки сцены, поэтому в нем удобно подготовить объекты для контроля вводимых клавиш. так, именно здесь я сделаю экземпляр класса `InputAction`, чтобы с его помощью подписаться на клавиши, которые мне нужны для управления персонажем:
```C#
private void Awake() {
	moveAction = new InputAction(
		name: "Move",
		type: InputActionType.Value
	);
	moveAction.AddCompositeBinding("2DVector")
		.With("Up", "<Keyboard>/w")
		.With("Down", "<Keyboard>/s")
		.With("Right", "<Keyboard>/d")
		.With("Left", "<Keyboard>/a");
		
	moveAction.expectedControlType = "Vector2";
}
```
все, что здесь происходит я описал в [[InputAction]]
сейчас я просто скажу, что здесь я "регистрирую" нужные мне для управления кнопки и объявляю тип, который должен возвращаться, когда кнопка была нажата - двумерный вектор.
теперь следует описать два метода, которые будут отключать и включать считывание ввода:
```C#
private void OnEnable() {
	moveAction?.Enable();
}
private void OnDisable() {
	moveAction?.Disable();
}
```
эти два метода будут применяться каждый раз, когда объект на сцене включается или выключается. внутри них я отключаю и включаю подписку на клавиши, чтобы лишний раз не перегружать логику программы.
теперь я могу написать метод `Start()`, который будет применяться единожды при активации скрипта перед методом `Update()`:
```C#
private void Start() 
{
	rb = GetComponent<Rigidbody2D>();	
}
```
здесь я завел переменную, которую не объявил ранее. так как здесь в переменную попадает `Rigidbody` нашего персонажа, с которым мне придется работать в остальных методах, экономичнее всего будет записать приватную переменную `rb` в атрибуты класса, а не объявлять ее каждый раз в локальном пространстве, чтобы записать туда `Rigidbody`.
введу новую переменную:
```C#
public class PlayerConroller2D : MonoBehaviour
{
	...
	private Rigidbody2D rb;
	...
}
```
и запрещу внешним физическим предметам раскручивать наш предмет в результате взаимодействия, потому что хочу, чтобы персонаж крутился только, если я ему скажу крутиться:
```C#
private void Start()
{
	rb = GetComponent<Rigidbody2D>();
	rb.constraints = RigidBodyConstraints2D.FreezeRotation;
}
```
теперь я, наконец, начну метод `Update()` - он будет применять код внутри себя каждый кадр:
```C#
private void Update() {
	Vector2 moveInput = moveAction.ReadValue<Vector2D>();
	
	float horizontalInput = moveInput.x;
	float verticalInput = moveInput.y;
}
```
сначала я получаю текущий вектор движения, чтобы прочитать константы `x` и `y`. 
теперь я хочу проверить настройку `CanMoveDiagonaly` и, если она правдива, сделать логику для поворота персонажа вокруг своей оси:
```C#
if (CanMoveDiagonaly) {
	movement = new Vector2(horizontalInput, verticalInput);
	if (movement.magnitude > 1f) movement.Normalize();
}
```
переменная `movement` будет хранить вектор, который состоит из координат, извлеченных `moveAction`. следующим условным запросом я прошу магнитуду - это константа, которая равна вычисленной длине вектора `movement`.
если длина больше $1$, то мы двигаемся диагонально. но т.к. я перемещаю персонажа на конец вектора, то было бы неправильно позволить персонажу переместиться на расстояние дальше по диагонали, чем он смог бы сделать по горизонтали. поэтому я применяю метод `Normalize()`, который приводит длину вектора к единице.
