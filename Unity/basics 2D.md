теперь я хочу написать скрипт, который контролирует движения персонажа.
любой скрипт начинается с импорта движка + мне понадобится система движения (новая) для контроля движений игрока:
```C#
using UnrealEngine;
using UnrealEngine.InputSystem;
```
после импорта мне нужно объявить основной класс, который называется `PlayerController` и наследуется от класса `MonoBehaviour`:
```C#
public class PlayerController2D: MonoBehaviour
{
	...
}
```
из туториала по системе скриптинга в моем классе может быть ограниченный набор сущностей. но перед тем как их описать я установлю атрибуты класса, которые хочу менять с помощью инспектора или которые хочу иметь в распоряжении во всем классе:
```C#
public class PlayerController2D: MonoBehaviour
{
	public float speed = 5f;  // скорость персонажа
	public bool canMoveDiagonally = true;  // может ли персонаж двигаться диагонально
	...
}
```
если сейчас сохранить скрипт в таком состоянии и глянуть на параметры моего объекта с этим скриптом в инспекторе, то можно увидеть два новых:
![[2D3.png]]
после объявления базовых переменных я могу начать первый метод скрипта - метод `Start()`:
```C#
void Start() 
{
	rb = GetComponent<Rigidbody2D>();	
}
```
здесь я завел переменную, которую не объявил ранее. так как здесь в переменную попадает `Rigidbody` нашего персонажа, с которым мне придется работать в остальных методах, экономичнее всего будет записать приватную переменную `rb` в атрибуты класса, а не объявлять ее каждый раз в локальном пространстве, чтобы записать туда `Rigidbody`.
введу новую переменную:
```C#
public class PlayerConroller2D : MonoBehaviour
{
	...
	private Rigidbody2D rb;
	...
}
```
