## check.py
Две функции тестирования `test_check_correct_positions()` и `test_check_positions` передают в `check()` аргументы:
1) `query_positionRUS`, `query_positionUSD`
2) `'server'`
`query_positionRUS` и `query_positionUSD` - это имена блоков в ini файле `query_settings.ini`. Внутри каждого из блоков лежит переменная `positionRUS`/`positionUSD` - обе переменные хранят пути до SQL файлов, где описаны команды выбора этих данных из БД.
Функция `check()` принимает позиционные аргументы:
```Python
def check(query, data_query, body, message, position, server):
	pass
```
Ошибка возникает на следующем шаге - при передаче `query` и `server` в `db_query`:
```Python
def check(query, data_query, body, message, position, server):
	dataset = dbconnector.db_query(server, query)
```
В `query` и `server` значения:
1) в `query` лежит строка:
```Python
f'query_position{currency}'
```
2) в `server` лежит строка:
```Python
'server'
```
В `currency` попеременно лежат два значения `RUB` или `USD`.
## dbconnector.py
Тут вся работа происходит в функции `db_query`.
Сначала мы достаем конфиг:
```Python
config = get.get_config('query_settings.ini')
```
Из конфига достаем все нужные файлы для коннекта с БД.
После коннекта используем параметр, который был передан, как `query` из функции `check()` в функцию `db_query()`:
```Python
def check(query, data_query, body, message, position, server):
	dataset = dbconnector.db_query(server, query)
...
def db_query(server, param):
	config = get.get_config('query_settings.ini')
	...
	queries = get.get_config_keys(config, param)
```
Функция `get_config_keys()`нужна, чтобы из нужной части конфига достать все ее ключи и значения. Часть конфига, из которой мы будем доставать значения передается в аргументе `param`, а сам конфиг в`config`.
В данном случае наш `param` = `query` = `query_positionRUS\query_positionUSD`. То есть из конфига нам нужна секция `query_positionsRUS\query_positionsUSD` в двух разных случаях.
На выходе должен получится словарь, который содержит ключи и их значения из ini файла конфигураций. В качестве значений будут даны пути до SQL файлов, где прописаны `SELECT` запросы для выборки нужных данных.
Далее функция `insert_pararms()` поменяет в словаре пути до SQL файлов на те SQL команды, которые лежат внутри них.
