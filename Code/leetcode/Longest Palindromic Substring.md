### задача
Функция получает на вход строку. Нужно найти в этой строке наиболее длинную подстроку-палиндром.
Например:
```
1) input = 'abadib'
   output = 'aba'

2) input = 'abb'
   output = 'bb'
```
### мое решение
Моя идея заключалась в следующем - получаем строку, делаем внешний цикл `while` для проходов по строке. Затем делаем внутренний цикл `for`, чтобы по очередно для каждой буквы просканировать все оставшиеся символы с конца.
Получается следующий алгоритм:
1) Получаем строку `x`, делаем поинтер `l_pointer = 0`.
2) `while` запускает цикл `for`
3) `for` берет индексы каждого элемента с конца.
4) `if` проверяет, что подстрока от `l_pointer` до текущего индекса в `for` является палиндромом.
	1) Если является, то эту строку мы записываем в переменную `result` и идем делать проверки дальше
5) Если не является, то прибавляем к `l_pointer` единицу и идем на следующий круг `while`. Повторяем, пока `l_pointer` меньше длины `x`.
В итоге код функции выглядит так:
```Python
def longestPalindrome(self, s: str) -> str:

counter = 0
l_pointer = 0
result = ''
length_s = len(s)

if s[::-1] == s:
	return s

while l_pointer < length_s:
	for index in range(length_s - 1, l_pointer - 1, -1):
		sample = s[l_pointer:index + 1]
		length_sample = len(sample)
		if sample[::-1] == sample and length_sample > counter:
			counter = length_sample
			result = sample
	l_pointer += 1
return result
```
### проблемы
Мой алгоритм вышел очень долгим. 
Во-первых, на время очень сильно влияют вложенные циклы.
Во-вторых, даже если логически следующие найденные палиндромы не могут быть больше найденного на текущий момент, код продолжит работать.
## решения
Моя идея заключалась в использовании "окна" для проходок по алгоритму. Однако возможно, что я начал не с того конца. Возможно начать нужно было с центра последовательности символов, постепенно расширяя "окно". Но тогда осталась бы проблема - нужный палиндром может находится не прямо идеально в центре строки, а где-нибудь слева. 
Индус подсказывает - для нас важен центр палиндрома, это верно, но мы не знаем с какого места в последовательности его начать отсчитывать, поэтому можно считать центром поочередно каждую букву последовательности. Например:
```
input = "abacaba"
center = "a"
l_pointer = ""
r_pointer = "b"
```
В данном примере я предположил, что центром палиндрома является первая буква.Это верно - одна буква является палиндромом, но теперь я должен проверить окружающие ее буквы, чтобы вычислить больший палиндром, если он тут есть. 
Для этого я делаю два указателя - `r_pointer` (для движения вправо от центра) и `l_pointer` (для движения влево от центра). 
Чтобы проверить подстроку на больший палиндром мне нужно сравнить два символа справа и слева, но мой левый указатель пуст, потому что слева у нас начало строки. Следовательно буква `a` является только центром собственного палиндрома и можно двигаться к следующей букве.
Я реализую эту механику с помощью срезов, т.к. ориентируясь только на индексы можно получить кучу неудобных `IndexError` прямо посередине кода.
Шаг первый - реализовать цикл `for`:
```Python
for index in range(len(s)):
	l_pointer = index  # каждый раз с позиции индекса.
	r_pointer = index  # каждый раз с позиции индекса.
```
Шаг второй - реализовать цикл `while`, который будет расширять "воронку":
```Python
for index in range(len(s)):
	l_pointer = index
	r_pointer = index
	while s[l_pointer:r_pointer + 1] == s[r_pointer:l_pointer - 1: - 1]:
		pass
```
Немного пояснений к циклу: берем срез, который получился от левого указателя до правого (включительно), а потом сравниваем его с его же перевернутой версией. Если они совпадут, значит у нас палиндром, а если нет, то палиндром закончился.

---
Здесь нужно сделать паузу. Я пошел смотреть видос индуса и понял, что самая важная проблема этой задачи в том, что нам могут попасться палиндромы с четным набором букв, которые при этом находятся в нечетных последовательностях (и наоборот). Например:
```
abbaq, caba, abaq
```
Поэтому мы должны сканировать последовательность сразу двумя способами - для поиска четных подстрок и нечетных подстрок. Как это сделать?
Для скана четной подстроки нам нужно прибавлять к правому указателю единицу. Почему? Возьмем случай `abbaq`:
```Python
while s[left:right + 1] == s[left:right + 1][::-1]:
	if left > 0:
		left -= 1
	right += 1
# первая итерация - буква "a"
# вторая итерация - подстрока "ab" (завершаем цикл)
# третья итерация - буква "b"
# четвертая итерация - подстрока "abb" (завершаем цикл)
# ...
```
Из примера выше видно, что мы пытаемся взять центр палиндрома так будто это одна буква, хотя в строке `abbaq`, очевидно, центром палиндрома можно считать две буквы `bb`. Следовательно, нам нужно схватить их обе - чтобы это сделать мы и прибавим к правому поинтеру единицу. Тогда итерации будут выглядеть так:
```Python
while s[left:right + 1] == s[left:right + 1][::-1]:
	if left > 0:
		left -= 1
	right += 1
# первая итерация - буквы "ab" (завершаем цикл)
# вторая итерация - буквы "bb"
# третья итерация - буквы "abba"
# четвертая итерация - буквы "abbaq" (завершаем цикл)
```
Короче смысл в том, что нам нужно использовать сразу оба варианта - для нечетного палиндрома и для четного палиндрома, потому что в строке может оказаться любой из этих вариантов. 
По следам индуса я написал программу, которая закидывает обе разновидности в функцию, а после добавляет результаты в список. В конце в списке ищется максимальное по количеству символов значение. Итоговое решение в 200 раз быстрее предыдущего:
```Python
def longestPalindrome(self, s: str) -> str:

	def x (s, left, right):
		sample = s[left]
		while s[left:right + 1] == s[left:right + 1][::-1]:
			# переменная нужна, чтобы хранить результат
			# перед выходом из цикла.
			sample = s[left:right + 1]
			if left > 0:
				left -= 1
				right += 1
		return sample

	if s == s[::-1]:
		return s

	result = []

	for index in range(len(s)):
		result.append(x(s, index, index))
		result.append(x(s, index, index + 1))

	return max(result, key=len)
```
