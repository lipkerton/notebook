## интро
Иногда мне нужно установить что-то из исходников. Я так делал с Python, а теперь хочу провернуть тот же фокуc c PostgresSQL.
Так как у меня формально нет GUI в моем WSL, мне нужно каким-то образом получить архив с исходниками с официального сайта PostgresSQL. В данном случае я выбрал архив с расширением `.tar.gz`, который лежит вот по этой ссылке: https://www.postgresql.org/ftp/source/v16.9/. После того как архив выбран нужно его скачать. Я это сделаю с помощью утилиты `wget`.
## wget
`wget` позволяет скачивать файлы из интернета. На вход она просит ссылку на сам файл для скачки. Я захотел попробовать использовать `wget` вместе с обычной ссылкой на интернет страницу (https://habr.com/ru/companies/ruvds/articles/346640/):
```Bash
wget https://habr.com/ru/companies/ruvds/articles/346640/
```
И она мне тупо скачала файл под названием `index.html` прямо в ту директорию, из которой я ее и вызвал.
Теперь я передам в `wget` ссылку буквально на скачку файла `postgresql-16.9.tar.gz`:
```Bash
wget https://ftp.postgresql.org/pub/source/v16.9/postgresql-16.9.tar.gz
```
И, ожидаемо, она скачает мне архив прямо в ту директорию, из которой она была вызвана.
## tar
Окей, это уже круто, но как мне распаковать этот архив?
Команда `tar` является встроенной утилитой Linux. Она позволяет создавать архивы из ряда файлов в формате `.tar.gz` + она позволяет распаковывать такие же архивы из подобного же формата.
Для начала я хочу попытаться создать архив. Синтаксис команды для создания архива простой:
```Bash
tar <параметры> <имя архива> <набор файлов для архивирования>
```
Синтаксис команды для распаковки:
```Bash
tar <параметры> <имя архива>
```
Для теста я хочу заархивировать файл `test.txt`. Для этого мне понадобится два параметра команды `tar` + сама команда `tar`:
```Bash
tar -cf archive.tar test.txt
```
После выполнения в текущей директории появится файл `archive.tar`.
Параметры:
1) `-c` - (create) создает новый архив и записывает в него файлы.
2) `-f` - (file) после этого параметра указывается имя файла.
Теперь я хочу распаковать архив. Для этого мне понадобятся уже немного другие параметры:
```Bash
tar -xf archive.tar
```
После выполнения в текущей директории появится файл `file.txt`.
Параметры:
1) `-x` - (extract) извлекает файлы из архива.
2) `-f` - (file) после этого параметра указывается имя файла.
Дополнительно обычно указывают параметр:
3) `-v` - (verbose) заставляет утилиту сообщить что она делает в процессе своей работы. Сообщение отобразится в терминале.
Теперь я хочу распаковать тот файл который я скачал изначально - `postgresql-16.9.tar.gz`. Но с ним не все так просто: в конце расширения указаны еще две буквы - `gz`. `gz` - это указание на то, что при архивации файлов использовалась утилита `gzip`. Точно такую же нам нужно использовать, чтобы распаковать архив. Для этого команде `tar` нужно выставить дополнительные параметры:
```Bash
tar -xzvf postgresql-16.9.tar.gz
```
После этого в директории появится папка `postgresql-16.9`.
Помимо уже упомянутых параметров появился новый:
1) `-z` - это указатель на то, что нужно пропустить файлы через утилиту `gzip`.
## компиляция и установка
Когда я распаковал архив мне нужно зайти в папку, которая была в нем и прописать:
```Bash
./configure --prefix=/var/lib/postgresql/16
```
Параметр `--prefix` взят из официального руководства по установке PostgreSQL. В него можно передать директорию, в которую мы хотим установить наш PostgreSQL.
Теперь нужно собрать сконфигурированные файлы в итоговую сборку:
```Bash
make world
sudo make install-world
```
Параметр `world` означает, что я хочу собрать вообще все DLC, которые идут в комплекте с этой версией PostgreSQL (например, документация + некоторые дополнительные модули).
Пойдет долгая установка. Когда она завершится мне нужно "переоформить" права на получившуюся директорию на себя. Для этого я воспользуюсь командой `chown`:
```Bash
sudo chown -R lipketon /var/lib/postgresql/16
```
Параметры:
1) `-R` - (recursive) означает, что я хочу поменять владельца для этой директории, для ее поддиректорий и для всех ее файлов.
Теперь нужно внести изменения в переменные окружения - нужно обозначить папку, в которую будет размещена наша первая база данных. Изменять переменные окружения можно двумя способами - открыть файл `.zshrc` на редактуру и написать там:
```Bash
export PGDATA=/var/lib/postgresql/16/main
```
Либо воспользоваться утилитой `echo`:
```Bash
echo "export PGDATA=/var/lib/postgresql/16/main" >> ~/.zshrc
```
Дополнительно лучше всего еще прописать команду прямо в оболочке:
```Bash
export PGDATA=/var/lib/postgresql/16/main
```
Потому что в этом случае можно будет продолжать процесс установки в рамках текущей сессии без необходимости перезапуска оболочки.
Теперь нужно инициализировать первый кластер БД. Для этого нужно обратиться к скрипту `pg_ctl` и прописать для него параметр `initdb`:
```Bash
/var/lib/postgresql/16/bin/pg_ctl initdb
```
Теперь нужно просто запустить наш сервер БД:
```Bash
/var/lib/postgresql/16/bin/pg_ctl -D /var/lib/postgresql/16/main start
```
Эта команда разбита на две части:
1) `/var/lib/postgresql/16/bin/pg_ctl` - указывает на скрипт `pg_ctl`.
2) `-D` - параметр утилиты `pg_ctl`, после которого указывается место размещения конфигурационных файлов кластера.
3) `/var/lib/postgresql/16/main` - значение, которое мы прописали для `PGDATA`. Адрес конфигурационных файлов кластера.
4) `start` - команда на старт сервера.
>[!important] Часть команды `-D /var/lib/postgresql/16/main` указана для понта. На самом деле она не нужна, т.к. если отсутствует прямое указание на папку с конфигами, то `pg_ctl` возьмет адрес этой папки из переменной `PGDATA`.
## htop и создание своей БД
Чтобы проверить работает ли сервер БД можно воспользоваться утилитой `htop` - в ней я нажму на кнопку фильтрации `F4` и пропишу `postgresql`. Должны появится строчки двух или более процессов по этому запросу, которые покажут, что сервак работает.
Теперь я хочу написать какой-то запрос в свою новую БД. Чтобы это сделать нужно воспользоваться утилитой `psql`:
```Bash
/var/lib/postgresql/16/bin/psql -h localhost -U lipkerton postgres
```
После этой команды откроется консоль для ввода запросов от лица пользователя `postgres`. Параметры:
1) `-h` - (host) после этого параметра указывается адрес хоста, на котором стоит БД и к которому мы подключаемся.
2) `-U` - (User) имя пользователя, под которым мы заходим в БД.
3) `postgres` - имя БД, к которой мы подключаемся.
По умолчанию в только что созданном сервере уже есть одна БД - `postgres`.
Теперь я хочу сделать так, чтобы утилита `psql` вызывалась не через полный путь до нее, а просто через команду `psql`. Чтобы это сделать мне нужно внести путь до нее в переменную окружения `PATH`:
```Bash
export PATH="$PATH:/var/lib/postgresql/16/bin"
echo 'export PATH="$PATH:/var/lib/postgresql/16/bin"' >> ~/.zshrc
```
Теперь я могу писать:
```Bash
psql -h localhost -U lipkerton -d postgres
```
Находясь внутри БД `postgres` я могу начать создавать свою БД:
```SQL
create database lessons;
```
