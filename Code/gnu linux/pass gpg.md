## intro
я решил, что хранить пароли на листочке не слишком целесообразно, так как мой листочек начал стремительно устаревать и портиться. Поэтому я начал искать решение для хранения всех своих паролей.
существуют коммерческие решения - хранилище ключей Apple, Google и прч. Однако их БД сливаются так часто, что пароль придется менять на каждом сервисе раз в месяц.
поэтому нужно найти что-то в опенсорсе.
## gpg
`gpg` - это утилита, которая нужна, чтобы шифровать файлы и создавать цифровые подписи. На самом деле шифрование файлом мне не так и необходимо, но побочный эффект `gpg` в том, что мы с ее помощью можем создать пару ключей. Эту пару ключей можно применить, чтобы сделать локальное хранилище паролей утилитой `pass`.
Для начала утилиту нужно поставить:
```Bash
sudo apt install gpg
```
### создаем ключи
Как создать ключи?
```Bash
gpg --full-gen-key
```
Далее прога попросит ввести пару настроек:
1) Алгоритм шифрования (выбирать можно любой).
2) Длинну ключа в битах (чем длиннее ключ, тем сложнее его расшифровать).
3) Время жизни ключа (0 - для бессрочного ключа).
4) Данные для подписи ключа (можно указать любые).
5) Пароль.
Пока генерируются ключи можно шевелить мышкой и вводить что-то в клавиатуре - это делается для внесения большей случайности в процесс генерации.
Чтобы посмотреть на получившиеся ключи можно ввести команду:
```Bash
# публичный ключ
gpg -k
# секретный ключ
gpg -K
```
### шифруем файл
Как зашифровать файл?
```Bash
gpg -e -a -r <id_ключа> <файл>
```
Параметры такие:
1) `e` - (encrypt) параметр означает, что будем шифровать.
2) `a` - (ASCII) итоговый файл будет содержать шифр не в бинарном формате, а в формате ASCII.
3) `r` - (recipient) имя, которое маркирует ключ для выполнения шифрования (те самые данные для подписи ключа).
4) `<id_ключа>` - здесь указываем данные, которыми подписали ключ (в моем случае - электронная почта).
По итогу выполнения команды появится зашифрованная версия файла с расширением `.txt.asc`. Ее можно открыть с помощью любого редактора.
Теперь как расшифровать файл?
```Bash
gpg -d -o file.txt file.txt.asc
```
Параметры такие:
1) `d` - (decrypt) режим дешифрования.
2) `o` - (output) после этого параметра указываем файл, куда положим расшифрованный результат.
### сохраняем ключи в файлы
Хранить ключи только на одном компе плохо, потому что эти ключи будут использоваться для шифрования и дешифрования паролей утилитой `pass`. Ключ нужно уметь импортировать. Лучше всего, конечно, делать это на флэшку.
Но прежде чем импортировать на флэшку нужно сначала получить файлы:
```Bash
# экспортируем публичный ключ
gpg --export <id_ключа> > public.gpg
# экспортируем приватный ключ
gpg --export-secret-key <id_ключа> > secret.gpg
```
Оба файла лучше всего сохранить на каком то внешнем носителе.
Когда нам понадобится использовать утилиту `pass` на другом компьютере с той же библиотекой паролей, которую мы создавали на этом, то нам так же понадобятся и ключи `gpg`, которые мы использовали при создании паролей здесь. Поэтому нужно перенести эти файлы на новое устройство и распаковать их содержимое в новую версию `gpg`.
Я перенес сами файлы, но как распаковать их содержимое в `gpg`?
```Bash
gpg --import public.gpg
gpg --import secret.gpg
```
Все, файлы на данном компьютере больше хранить не стоит.
## pass
`pass` - это надстройка над `gpg`, которая по сути представляет из себя просто Bash-скрипт. Это менеджер паролей.
Для начала нужно создать хранилище для будущих паролей:
```Bash 
pass init <id_ключа>
```
`id_ключа` нужен, чтобы хранилище знало с помощью какого ключа мы будем шифровать пароли.
После команды создастся хранилище `~/.password-store`. Внутри хранилища будут директории категорий наших паролей + каждый пароль будет храниться в отдельном файле.
Я хочу сохранить какой-нибудь пароль в хранилище:
```Bash
pass insert Email/lipgmail
```
После будут предложены поля для заполнения пароля, а в директории `.password-store` появится папка `Email`, где будет лежать зашифрованный бинарный файлик `lipgmail.gpg`.
Чтобы посмотреть пароль в этом файле:
```Bash
pass Email/lipgmail
```
Так же пароль можно сгенерировать:
```Bash
pass generate Email/lipgmail
```
Эта команда сгенерирует пароль, выведет его в консоль, а еще сохранит автоматически в папку `.password-store/Email`.
