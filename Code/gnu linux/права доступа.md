## как смотреть права?
Что можно делать с файлом в системе linux? Варианта всего три:
1) читать файл
2) редактировать файл
3) запускать файл на исполнение
Каждый имеет свой символ - чтение обозначается символом `r` ('read'), редактирование обозначается символом `w` ('wright') и запуск файла на исполнение обозначается символом `x` ('execute').
Обозначения можно увидеть, если вызвать команду `ls` с параметром `-l`:
```Bash
ls -l
-rw-r--r-- 1 peterpetukhov peterpetukhov 12 May 17 11:51 hello
```
Я вызвал команду `ls` из директории, где лежит один файл `hello` - она мне показала характеристики этого одного файла.
В самом начале записи показана последовательность странных символов - `-rw-r--r--`. Это режимы доступа к файлу. Они делаться на три части:
1) для создателя - `rw-`
2) для группы - `r--`
3) для остальных пользователей - `r--`
Первый символ в последовательности передает тип файла - если это папка, то вначале будет написана буква `d`:
```Bash
ls -l
-rw-r--r-- 1 peterpetukhov peterpetukhov   12 May 17 11:51 hello
drwxr-xr-x 2 peterpetukhov peterpetukhov 4096 May 17 16:42 test_dir
```
Если указан прочерк, то файл является обычным файлом. Еще может быть указана буква `l` - это означает, что файл является ссылкой.
После указания прав доступа на файл идет подряд два упоминания моего пользователя - первым упомянуто имя владельца файла, а вторым - имя группы, которая владеет файлом.
В данном случае:
```Bash
ls -l
-rw-r--r-- 1 peterpetukhov peterpetukhov 12 May 17 11:51 hello
```
У владельца файла есть права на чтение и редактирование файла (`rw-`), у группы, которая владеет файлом есть права только  на чтение файла (`r--`) и у всех остальных тоже есть права только на чтение (`r--`).
## как менять права?
Базово изменение прав происходит через команду `chmod`. Например, здесь я хочу отобрать у всех, кроме моей группы и меня самого, права на чтение файла `hello`:
```Bash
chmod o-r hello
ls -l
-rw-r----- 1 peterpetukhov peterpetukhov   12 May 17 11:51 hello
```
Как видно - буква `r` убралась из последней тройки символов. Теперь, если любой другой пользователь захочет прочитать файл, то получится ошибка:
```Bash
bat hello
[bat error]: 'hello': Permission denied (os error 13)
```
Синтаксис команды `chmod` немного непонятный. Я говорю об этих символах: `o-r`.  Для команды `chmod` существует четыре группы пользователей:
1) владелец (`u`)
2) группа владеющая файлом (`g`)
3) остальные (`o`)
4) вообще все (`a`)
Чтобы отобрать или добавить права нужно написать конструкцию со знаками минус или плюс - `o+r` (добавляем права на чтение всем остальным), `o-r` (отнимаем права на чтение у всех остальных), `g-r` (отнимаем права на чтение у группы). 
>[!important] Может быть так, что владелец файла не состоит в группе, которая владеет файлом - в этом нет никакой ошибки.
## права директорий
Для директории существуют те же самые группы прав. Но что означают в контексте директории? 
1) `r` ('read') - означает, что мы можем читать директорию с помощью команды `ls`.
2) `w` ('write') - означает, что мы можем создавать и изменять файлы в директории.
3) 'x' ('execute') - означает, что мы можем входить в директорию и получать доступ к атрибутам файлов и самим файлам.
Нужно разобрать по порядку.
### read для директории
Я отберу у директории для `others` все права, кроме прав на чтение:
```Bash
chmod o-wx test_dir
ls -l
drwxr-xr-- 2 peterpetukhov peterpetukhov 4096 May 17 16:42 test_dir
```
Теперь от лица другого пользователя попытаюсь зайти в директорию:
```Bash
cd test_dir/
bash: cd: test_dir/: Отказано в доступе
```
Создать файл в директории:
```Bash
touch test_dir/hello_1
touch: cannot touch 'test_dir/hello_1': Permission denied
```
Просмотреть директорию:
```Bash
ls -l test_dir/
ls: cannot access 'test_dir/hello': Permission denied
-????????? ? ? ? ?            ? hello
```
Из всех действий удалось выполнить только последнее. При этом видно, что для файла `hello` все атрибуты (группы доступа, время создания, пользователь и группа) скрыты за знаками вопросов.
### write для директории
Если отобрать у директории все права, кроме права на запись то с ней ничего не удастся сделать - в интернете пишут, что право на `write` дает возможность создавать каталоги и подкаталоги в директории. На самом деле это не так - директория не позволит сделать с собой ничего, если нет других прав.
Пытаемся создать файл:
```Bash
touch test_dir/hello_1
touch: cannot touch 'test_dir/hello_1': Permission denied
```
Однако! Если выдать права на `execute` для директории, то файл создать уже получится.
### execute для директории
Это право позволяет заходить в директорию. Все. Больше оно не позволяет ничего.
```Bash
chmod o=x test_dir/
ls -l
drwxr-x--x 2 peterpetukhov peterpetukhov 4096 May 18 10:06 test_dir
```
Что можно делать от другого пользователя? Буквально можно зайти в директорию с помощью команды `cd`.
Можно ли, находясь в ней создать файл? Нет:
```Bash
touch hello_1
touch: cannot touch 'hello_1': Permission denied
```
Можно ли просмотреть ее содержимое? Нет:
```Bash
ls -l
ls: cannot open directory '.': Permission denied
```
Зато можно воспользоваться файлами, но это работает только если знаешь их название буквально до буквы - оболочка не подскажет тебе, т.к. права на чтение директории у тебя отсутствуют. С файлами можно делать все на что хватает прав.
## как менять владельца?
Существует две команды для этого.
Первая меняет владельца файла:
```Bash
sudo chown <пользователь> <файл>
```
Вторая меняет группу, которая владеет файлом:
```Bash
sudo chgrp <группа> <файл>
```
А еще можно поменять все вместе внутри команды `chown`:
```Bash
sudo chown <пользователь>:<группа> <файл>
```
