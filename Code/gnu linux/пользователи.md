В linux есть система пользователей, на которой основана в целом безопасность данных. Два ключевых компонента этой системы - пользователи и группы. Несколько пользователей могут принадлежать к одной группе. Для каждого пользователя автоматически создается группа с именем этого пользователя.
## пользователи
Всех пользователей можно посмотреть по адресу `/etc/passwd`:
```Bash
cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
...
peterpetukhov:x:1000:1000:peterpetukhov,,,:/home/peterpetukhov:/usr/bin/zsh
...
```
Там будет много пользователей - я оставил только своего.
`peterpetukhov` - имя пользователя.
`x` - пароль пользователя (здесь стоит "заглушка" - пароль в открытом виде содержится в `/etc/shadow`).
`1000` - user-id.
`1000` - group-id
`peterpetukhov,,,` - эта графа нужна для заполнения имени пользователя и дополнительной информации о пользователе, например, его телефона. или электронной почты.
`/home/peterpetukhov` - директория пользователя.
`/bin/zsh` - оболочка, которая запускается при заходе пользователя в систему.
Узнать имя текущего пользователя можно с помощью команды `whoami`:
```Bash
whoami
peterpetukhov
```
А еще можно посмотреть имя текущего пользователя с помощью переменной окружения `$USER`:
```Bash
echo $USER
peterpetukhov
```
## группы
Все группы можно посмотреть по адресу `/etc/group`:
```Bash
cat /etc/group
...
cdrom:x:24:peterpetukhov
floppy:x:25:peterpetukhov
audio:x:29:peterpetukhov
dip:x:30:peterpetukhov
video:x:44:peterpetukhov
plugdev:x:46:peterpetukhov
users:x:100:peterpetukhov
netdev:x:106:peterpetukhov
peterpetukhov:x:1000:
...
```
Группа моего пользователя указана в самом низу. В ней никто не состоит. Это странно, потому что в ней по идее должен состоять хотя бы мой пользователь. Но на самом деле он просто состоит в ней по умолчанию, так что можно забить. Выше обозначены еще группы, а в конце каждой строки после двоеточия указан набор пользователей, которые состоят в этих группах.
Узнать список своих групп можно командой `groups`:
```Bash
groups peterpetukhov
peterpetukhov : peterpetukhov cdrom floppy audio dip video plugdev users netdev
```
Сначала указывается имя пользователя, а через двоеточие идут все группы, которые ему доступны.
## добавить пользователя
### добавление
Чтобы добавить пользователя нужно обладать правами администратора. Команда которая добавляет пользователя `adduser`:
```Bash
sudo adduser myfriend
```
Далее появится меню заполнения пароля и дополнительной информации. Как только информация будет дана пользователь будет добавлен в систему и для него будет создана собственная группа.
Нужно проверить:
```Bash
cat /etc/passwd
...
myfriend:x:1001:1001:Igor,228,228-1488-1380,1488-228-1380,goida:/home/myfriend:/bin/bash
...
cat /etc/group
...
peterpetukhov:x:1000:
myfriend:x:1001:
...
groups myfriend
myfriend : myfriend users
...
```
Новый пользователь добавился в систему + для него автоматически создалась личная группа + он стал участником группы `users`. Так же добавилась папка пользователя в директории `/home`. 
Туда не может попасть никто, кроме самого пользователя и пользователя с правами суперпользователя (это поведение можно изменить, но по умолчанию оно именно такое).
Теперь можно зайти в систему через профиль нового пользователя.
### особенности новых пользователей
Их нет в списке `sudoers`, то есть они не могут пользоваться никакими командами от имени администратора + они не могут даже посмотреть список `sudoers`.
Чтобы добавить нового пользователя нужно перейти к пользователю, который имеет доступ к правам суперпользователя и вписать имя нового пользователя в список `sudoers`.
>[!important] Список `sudoers` находится по адресу `/etc/sudoers`.
## как менять пользователей?
Через команду `su`:
```Bash
su myfriend
su peterpetukhov
su root
```
## как создавать группы?
Создать группу можно командой `groupadd`:
```Bash
sudo groupadd friends
```
`friends` в данном случае - название группы.
Проверить ее создание можно через файл `group`:
```Bash
cat /etc/group
...
friends:x:1002:
...
```
## как добавлять в группы?
```Bash
sudo usermod -aG friends peterpetukhov
sudo usermod -aG friends myfriend
```
`-a` - параметр, который отвечает за добавление в группу (сокращение от `append`).
`-G` - параметр, после которого перечисляются имена групп, куда мы хотим добавить пользователя.
## удаление
### удалить пользователя
Чтобы удалить пользователя нужно применить команду `deluser`:
```Bash
sudo deluser myfriend
```
Однако не все так просто - у нас остаются еще и файлы, которые каким-то образом привязаны к пользователю. Например, домашняя директория пользователя, которая лежит по адресу `/home/friend` или файлы, которые пользователь успел создать в других папках. Ими мы тоже можем управлять через ту же команду.
Например, с таким параметром удалиться и пользователь и его домашняя директория:
```Bash
sudo deluser --remove-home myfriend
```
А так, удаляться все файлы, владельцем которых был этот юзер:
```Bash
sudo deluser --remove-all-files myfriend
```
Еще есть функция - удалить пользователя из всех групп:
```Bash 
sudo deluser --remove-all-groups myfriend
```
Из интересных способностей - можно оставить бэкап. Даже если мы в рамках одной команды делаем и бэкап, и удаление, бэкап все равно будет сохранен:
```Bash
sudo deluser --backup-to /my_backups/myfriend --remove-all-files myfriend
```
### удалить группу
Удалить группу можно с помощью команды `delgroup` -  у нее примерно такие же параметры, как у `deluser`; полный список параметров можно посмотреть через команду `man`.
```Bash
sudo delgroup friends
```
Однако не может быть удалена основная группа пользователя, если пользователь, который за ней закреплен все еще есть в системе.
То есть я не могу удалить группу `peterpetukhov`, к которой принадлежит пользователь `peterpetukhov`, если пользователь все еще существует в системе.
Это ограничение можно обойти с помощью альтернативной команды:
```Bash
sudo groupdel --force friends
```
### удалить пользователя из группы
Чтобы удалить пользователя из группы нужно понимать как устроена команда `usermod`. Когда мы набираем:
```Bash
sudo usermod -aG <группа> <пользователь>
```
На самом деле мы обновляем список групп для пользователя путем добавления в него конкретной группы.
Точно такую же махинацию мы можем провернуть, если укажем весь список текущих групп пользователя + группу, в которую мы хотим добавить пользователя.
```Bash
sudo usermod -G users,myfriend,friends myfriend
```
У нового пользователя всего две группы по умолчанию - `users` (общая группа для всех пользователей) и `myfriend` (именная группа для этого конкретного пользователя). Команда перечисляет стандартные группы для пользователя + добавляет новую.
Теперь, очевидно, чтобы удалить пользователя из группы нам нужно просто переопределить его список групп:
```Bash
sudo usermode -G users,myfriend myfriend
```
 Все, теперь у пользователя снова только его группы.
 Однако! Легко понять, что такая система удаления из группы не совсем удобная. Поэтому есть альтернатива - `gpasswd`:
```Bash
sudo gpasswd -d <пользователь> <группа>
```
Она просто удалит пользователя из группы. Кстати этой же командой можно добавить пользователя в группу.
## сменить пользователю пароль
Это можно сделать с помощью команды `passwd`:
```Bash
sudo passwd <пользователь>
```
