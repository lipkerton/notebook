## база
Я немного поебался с этим вопросом. Что такое "локаль"? Это такая настройка в операционной системе, которая определяет как операционная система будет отображать данные. Например, мы можем видеть все буквы в бинарном формате, а можем видеть их на русском языке.
Чтобы сделать использование системы удобным нужно установить локали. Они устанавливаются в рамках оболочки - в Linux настройки оболочки храняться для каждого пользователя в файлах типа `.bashrc`, `.zshrc`. Чтобы изменить настройки оболочки нужно просто зайти в этот файл и выставить нужные параметры - при следующем запуске оболочки эти параметры будут применены.
Когда я переходил с оболочки `bash` на оболочку `zshell`, я, видимо, оставил все предыдущие настройки локали именно в старой оболочке. Поэтому у меня начала появляться ошибка:
```Bash
perl: warning: Setting locale failed.
perl: warning: Please check that your locale settings:
        LANGUAGE = (unset),
        LC_ALL = (unset),
        LC_CTYPE = "UTF-8",
        LANG = "ru_RU.UTF-8"
    are supported and installed on your system.
perl: warning: Falling back to a fallback locale ("ru_RU.UTF-8").
```
Эта ошибка означает, что где-то потерялась локаль великого русского языка. Она называется `ru_RU.UTF-8`.
Сначала я попытался забрать эту локаль с помощью команды:
```Bash
sudo locale-gen "ru_RU.UTF-8"
```
Но это ситуацию не спасло. Хотя я не знаю насколько это вообще на что-то повлияло, потому что я попробовал несколько способов и в итоге мне помогла настройка в `.zshrc` - `LC_ALL`.
```Bash
echo "export LC_ALL='ru_RU.UTF-8'" >> ~/.zshrc
```
Кстати, посмотреть все текущие локали можно с помощью команды:
```Bash
locale
```
## история
Я устанавливал PostgreSQL из исходников и при инициализации кластера он вдруг решил сказать, что в моем Linux Debian на UTM нет значений для переменных окружения `LANG` и `LC_*` и, следовательно, он не может установить правильно локали для кластера:
```Bash
➜  postgresql-17.5 pg_ctl initdb
The files belonging to this database system will be owned by user "peterpetukhov".
This user must also own the server process.

initdb: error: invalid locale settings; check LANG and LC_* environment variables
pg_ctl: database system initialization failed
```
Я решил проверить:
```Bash
➜  postgresql-17.5 locale
locale: Cannot set LC_CTYPE to default locale: No such file or directory
locale: Cannot set LC_MESSAGES to default locale: No such file or directory
locale: Cannot set LC_ALL to default locale: No such file or directory
LANG=en_US.UTF-8
LANGUAGE=
LC_CTYPE=UTF-8
LC_NUMERIC="en_US.UTF-8"
LC_TIME="en_US.UTF-8"
LC_COLLATE="en_US.UTF-8"
LC_MONETARY="en_US.UTF-8"
LC_MESSAGES="en_US.UTF-8"
LC_PAPER="en_US.UTF-8"
LC_NAME="en_US.UTF-8"
LC_ADDRESS="en_US.UTF-8"
LC_TELEPHONE="en_US.UTF-8"
LC_MEASUREMENT="en_US.UTF-8"
LC_IDENTIFICATION="en_US.UTF-8"
LC_ALL=
```
Утилита `locale` вдруг написала, что у нее нет возможности установить три константы для правильной работы локалей. Что это такое? Сначала я обновил локали командой:
```Bash
➜  postgresql-17.5 sudo update-locale
perl: warning: Setting locale failed.
perl: warning: Please check that your locale settings:
        LANGUAGE = (unset),
        LC_ALL = (unset),
        LC_CTYPE = "UTF-8",
        LANG = "en_US.UTF-8"
    are supported and installed on your system.
perl: warning: Falling back to the standard locale ("C").
```
Из сообщения видно, что обновление локалей не удалось. Почему? Исходя из сообщения понятно, что моя система не в курсе о существовании локали `en_US.UTF8`, поэтому все настройки, которые с ней связаны выдают ошибки.
Чтобы это исправить, я решил переконфигурировать настройки локалей моей системы:
```Bash
sudo dpkg-reconfigure locales
```
Эта команда врубает что-то вроде установочного окна Debian и позволяет отметить те локали из списка, подгрузка которых необходима для работы с языками в системе. Из списка я выбрал две нужные мне локали - `ru_RU.UTF-8` и `en_US.UTF-8`. После этого все резко начало работать.
### проблемы
Перед тем как решить проблему я подумал, что можно просто обойти транспортировку системных локалей в кластер при инициализации. Так реально можно сделать:
```Bash
/var/lib/postgresql/17/bin/initdb --locale="ru_RU.UTF8"
/var/lib/postgresql/17/bin/initdb --no-locale
```
Однако в этом случае как бы мы не пытались настроить сам кластер - утилита `psql` тупо не сможет нормально работать с кодировками. Например, у меня была проблема - когда я начинал печатать на русском и `psql` настолько сатанел, что отказывался в принципе от нормального ввода.
