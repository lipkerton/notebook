Я немного поебался с этим вопросом. Что такое "локаль"? Это такая настройка в операционной системе, которая определяет как операционная система будет отображать данные. Например, мы можем видеть все буквы в бинарном формате, а можем видеть их на русском языке.
Чтобы сделать использование системы удобным нужно установить локали. Они устанавливаются в рамках оболочки - в Linux настройки оболочки храняться для каждого пользователя в файлах типа `.bashrc`, `.zshrc`. Чтобы изменить настройки оболочки нужно просто зайти в этот файл и выставить нужные параметры - при следующем запуске оболочки эти параметры будут применены.
Когда я переходил с оболочки `bash` на оболочку `zshell`, я, видимо, оставил все предыдущие настройки локали именно в старой оболочке. Поэтому у меня начала появляться ошибка:
```Bash
perl: warning: Setting locale failed.
perl: warning: Please check that your locale settings:
        LANGUAGE = (unset),
        LC_ALL = (unset),
        LC_CTYPE = "UTF-8",
        LANG = "ru_RU.UTF-8"
    are supported and installed on your system.
perl: warning: Falling back to a fallback locale ("ru_RU.UTF-8").
```
Эта ошибка означает, что где-то потерялась локаль великого русского языка. Она называется `ru_RU.UTF-8`.
Сначала я попытался забрать эту локаль с помощью команды:
```Bash
sudo locale-gen "ru_RU.UTF-8"
```
Но это ситуацию не спасло. Хотя я не знаю насколько это вообще на что-то повлияло, потому что я попробовал несколько способов и в итоге мне помогла настройка в `.zshrc` - `LC_ALL`.
```Bash
echo "export LC_ALL='ru_RU.UTF-8'" >> ~/.zshrc
```
Кстати, посмотреть все текущие локали можно с помощью команды:
```Bash
locale
```
