Допустим у меня есть приложение на FastAPI, но еще я хочу докрутить к этому приложению БД. Не очень удобно заранее разворачивать БД на целевом сервере, потому что это делает все приложение не самым мобильным.
Лучше, чтобы БД сама по себе являлась контейнером, который так же, как и мое приложение, можно развернуть. Не трудно заметить, что проблема мобильности все еще остается - да, теперь не нужно запариваться с перемещением БД с системы на систему (из-за контейнера), но перемещать два контейнера отдельно друг от друга все еще не очень круто.
Чтобы справиться с трудностями была придумана **оркестрация** - автоматическое управление сразу несколькими контейнерами одновременно из одного файла.
Этот один файл называется `docker-compose.yml`. Он пишется в формате `YAML`.
