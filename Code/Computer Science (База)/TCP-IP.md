TCP/IP - это “система организации протоколов” или “абстрактная концепция, которая помогает нам раскидать протоколы по категориям и не выходить за рамки категории при работе с каким-нибудь протоколом, чтобы не нарушить работу протоколов в другой категории”. Существует еще одна такая же концепция - OSI и она отличается только тем, что описывает больше “полок для протоколов”. TCP/IP - мейнстрим.
Сколько полок? Четыре:
1. Канальный уровень (???)
2. Сетевой уровень (Network lvl)
3. Транспортный уровень (Transport layer)
4. Уровень приложения (Application lvl)
Каждая из полок отвечает за свою основную функцию и еще пачку дополнительных. Но зачем вообще все это необходимо в целом? Когда мы кидаем запросы в сеть, на самом деле мы отправляем пакеты - эти пакеты по сути являются конфетами, которые перед отправкой нужно завернуть в кучу слоев обертки. Вот протоколы и обеспечивают наши сообщения этой самой оберткой, например, они подписывают для какого IP-адреса предназначается пакет.
### Канальный уровень (Network link layer)
**Ethernet** - основной протокол и я, честно, пока мало про него знаю. Вроде как он заворачивает MAC-адрес, кодирует само сообщение и вообще базово готовит сырое сообщение к отправке.
### Сетевой уровень (Network layer)
Здесь приложению присваивается адрес отправителя и адрес получателя.
**IP (Internet Protocol)** - протокол, который определяет и добавляет к пакету адрес отправителя и адрес получателя.
**ICMP** - протокол, который работает в связке с IP и является по сути протоколом обработки ошибок (его еще иногда используют для DDoS-атак).
**IGMP** - не помню.
### Транспортный уровень (Transport layer)
Здесь определяется порт и способ доставки.
**UDP (User Datagram Protocol)** - протокол отправки быстрых датаграм на IP-адрес. Он не требует установки предварительного соединения с получателем, потому что просто выкидывает пакеты на указанный IP-адрес. Обычно он используется для таких данных, которые необходимо быстро доставить, например, голос в сервисах, которые позволяют говорить по инету (если будете настраивать сервер TeamSpeak на своем компе - придется открыть UDP-порты). Его проблема в том, что пакеты иногда могут не приходить на тот адрес, куда они были отправлены.
**TCP (Transmission Control Protocol)** - протокол, который буквально Контролирует Соединение между отправителем и получателем. Перед отправкой сообщения устанавливается соединение и только затем по нему начинают течь сообщения. Если пакет не дойдет, то он будет отправлен заново.
**QUIC (???)** - все, что я знаю: это протокол, который выдумал Гугл, для того, чтобы отправлять сообщения быстрее, чем с помощью TCP, но надежнее, чем с помощью UDP. Этот же протокол используется для работы HTTP протокола версии 3.0.

**HTTP (Hyper Text Transfer Protocol)** - протокол, который работает с гипертекстом (сейчас уже не только с ним) и обрабатывает ответы/запросы, которые предназначаются преимущественно для браузерных приложений.
