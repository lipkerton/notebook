# condition
условная конструкция состоит из двух элементов `if` и `else`. `else` - это необязательный элемент, его можно пропускать при использовании условной конструкции:
```C#
if (condition) {
	...
}
// ИЛИ
if (condition) {
	...
} else {
	...
}
```
вместо блока кода может быть написана любая другая конструкция:
```C#
if (condition) int.TryParse(args[0], out int result);
```
тут нужно быть осторожным, т.к. условие без блока кода кончается точкой с запятой - любой следующий код будет написан уже вне условия.
`else` и `if` можно скомбинировать в `else if`:
```C#
if (condition) {...}
else if (condition) {...}
else {...}
```
# switch
этот оператор позволяет определить шаблон, а затем проводит автоматические сопоставления значений с шаблоном. при совпадении значения он может запустить группу операторов внутри `case`:
```C#
switch (args[1])
{
	case "привет":
		Console.WriteLine("Привет!");
		break;
	case "пока":
		Console.WriteLine("Пока!");
		break;
}
```
`break` здесь нужен потому, что `case` принимает не один оператор, а несколько и при этом не требует включать их в блок кода. `break` просто прерывает текущий `case`, чтобы конструкция могла перейти к следующему `case`.
так же есть ключевое слово `default`, которое определяет как будет вести себя конструкция, если встретит значение, которое не совпадает ни с одним шаблоном:
```C#
switch (args[1])
{
	case "привет":
		Console.WriteLine("Привет!");
		break;
	case "пока":
		Console.WriteLine("Пока!");
		break;
	default:
		Console.WriteLine("Что?");
		break;
}
```
постановка `break` это правило, которое удовлетворяет требованию C#. это требование исключает *сквозное выполнение* - это фишка языков программирования семейства C, когда операторы `case` выполняются сверху вниз один за другим.
если же такое поведение все же нужно, то его следует отдельно запросить:
```C#
case "привет":
	Console.WriteLine("Привет!");
	goto case "пока";
case "пока":
	Console.WriteLine("Пока!");
	goto default;
default:
	Console.WriteLine("Что?");
	break;
```
нужно помнить, что здесь используется не сам `goto`, а `goto case`, который является вариацией `goto`.
сам `goto` не очень любят.
