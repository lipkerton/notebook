шаблоны, которые я использовал в [[condition switch#switch]] на самом деле могут быть сложнее. то, что было в примерах там:
```C#
switch (x) {
	case "привет":
		...
}
```
на самом деле считается *шаблоном константы*. это когда я сравниваю входящее значение с каким-то конкретным выражением ($5$, *привет* и т.д.).
так же почти шаблоном константы является *шаблон кортежа* в таком виде:
```C#
switch (x) {
	case (0, 0):
		...
}
```
по сути я тут так же как и в первом примере сравниваю входящее значение с конкретным выражением кортежа.
однако существуют более интересные экземпляры шаблона - *шаблоны типа*:
```C#
switch (x) {
	case string a:
		...
}
```
здесь я пытаюсь проверить, что входящее значение принадлежит к типу `string`. дополнительно внутри самого `case` я могу пользоваться входящим значением через переменную `a` (при успешном прохождении проверки):
```C#
switch (args[1]) {
	case string a:
		Console.WriteLine(a.Length);
}
```
шаблоны типа можно комбинировать с шаблонами кортежа:
```C#
switch (args[0], args[1]) {
	case (string x, string y):
		Console.WriteLine($"{x.Length}, {y.Length}");
		break;
```
при этом если мне не нужна такая переменная:
```C#
switch (args[0], args[1]) {
	case (string _, string _):
		Console.WriteLine("Проверка была сделана!");
}
```
это все называется *рекурсивный шаблон* - шаблон внутри шаблона.
так же есть еще комбинация между шаблоном типа, шаблоном кортежа и шаблоном константы:
```C#
switch (args[0], args[1]) {
	case ("привет", string x) {
		Console.WriteLine($"{args[0]}, {x}");
		break;
	}
}
```
