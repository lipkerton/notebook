# base
логический тип в C# всего один - `bool`, а в CLR он определен как `System.Boolean`.
про него следует знать только то, что в C# он работает строже, чем в других языках программирования. если в C или в Python я могу использовать числа для замены стандартных `true` или `false`, то в C# такая механика отсутствует:
```C#
if (someNumber) {...}
```
код задумывается так, чтобы при ненулевом значении `someNumber` начать выполнять блок кода. однако в C# это вернет ошибку.
C# требует явного использования типа `bool`, поэтому придется писать так:
```C#
if (someNumber != 0) {...}
```

нужно помнить, что операторы `AND` и `OR` имеют особенности. в случае с `AND`, если первый операнд оказался `false`, то второй даже не будет учитываться, а в случае с `OR` при правдивости первого операнда не учитывается значение второго.
чтобы избежать такого поведения (чтобы компилятор проверял оба операнда) нужно использовать одинарный символ `&` или одинарный `|`.

---
раньше для проверки на `not null` использовалась широко примерно такая конструкция:
```C#
if (s != null && s.Length < 10) {...}
```
здесь мы сначала проверяем переменную на `null`, а затем проверяем, что в ней лежит значение меньше десяти. в новых версиях C# появилась возможность схлопнуть две эти операции вот в такую форму:
```C#
if (s?.Length < 10) {...}
```
оператор `?` проверяет, что значение переменной не равно `null` перед тем как выполнять операции.

# условный оператор
в С# есть особый оператор - `?`. я уже говорил о нем ранее, когда описывал конструкцию `s?.Length`, но он так же используется и в других контекстах.
например, его можно использовать с тремя операндами, где первый операнд будет логическим выражением, а второй и третий операнды будут возвращаемыми значениями. в данном случае `?` вернет в случае правдивости первого операнда второй операнд, а в случае ложности вернет третий операнд.
```C#
int x = 1;
int y = 2;
int a = (x < y) ? x : y;
Console.WriteLine(a);  // 1
```
так как `y` больше `x`  вернется `x`, так как он указан вторым операндом.

---
еще одна разновидность - `??`
этот оператор вычисляет свой первый операнд и если он не является `null`, то использует его в качестве результата выражения, а если он является `null`, то использует второй операнд:
```C#
string? x = null;
string result = x ?? "Hello!";
Console.WriteLine(result);
```
