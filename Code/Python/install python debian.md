# исходники

## установка нужных пакетов

Я поставлю питон из исходников. Чтобы это сделать сначала необходимо установить все нужные библиотеки. Для этого можно найти сразу подходящую команду в инете, но прикол в том, что такие команды могут упустить какие-то библиотеки, которые именно для вашей работы являются важными. Если питон установится без этих библиотек, то, чтобы их добавить потом, нужно будет повторять процесс установки. Например:

```Bash
sudo apt install -y make build-essential \
	libssl-dev \
	zlib1g-dev \
	libbz2-dev \
	libreadline-dev \
	libsqlite3-dev \
	wget \
	curl \
	llvm \
	libncursesw5-dev \
	xz-utils \
	tk-dev \
	libxml2-dev \
	libxmlsec1-dev \
	liblzma-dev
```

Здесь есть куча важных библиотек, однако, если вы захотите, например, воспользоваться Pandas, то Python покажет ошибку. Почему? Потому что здесь не хватает библиотеки `libffi-dev` _._ Ее тоже необходимо будет добавить.

```Bash
sudo apt install -y make build-essential \
	libssl-dev \
	zlib1g-dev \
	libbz2-dev \
	libreadline-dev \
	libsqlite3-dev \
	wget \
	curl \
	llvm \
	libncursesw5-dev \
	xz-utils \
	tk-dev \
	libxml2-dev \
	libxmlsec1-dev \
	libffi-dev \
	liblzma-dev
```

**Крутая история:** я зашел на сайт, посмотрел список устанавливаемых пакетов и пропустил `libffi-dev` ; конечно, я сразу пошел писать о том как я умен и пояснять как правильно устанавливать питон из исходников. На самом деле команда с сайта устанавливала `libffi-dev` , а я просто так написал кучу текста.

## как получить питон?

Нужно просто скачать его через `wget.
```Bash
wget https://www.python.org/ftp/python/3.13.1/Python-3.13.1.tgz
```

И распаковать:

```Bash
tar -xzfv Python-3.13.1.tgz
```

Но есть проблема - команда распаковки выдает ошибку:

```Bash
tar: z: Cannot open: No such file or directory
tar: Error is not recoverable: exiting now
```

Раньше такого не было. Я выяснил, что проблема связана с использованием дефиса перед последовательностью аргументов для команды `tar` - я не знаю почему конкретно дефис необходимо убрать, но если так сделать, то все успешно сработает.

```Bash
tar xzfv Python-3.13.1.tgz
```
