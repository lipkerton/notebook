Телом запроса называют данные (и их форму), которые клиент отправляет на сервер.
Телом ответа называют данные (и их форму), которые сервер отправляет клиенту.
Чтобы клиенты могли отправлять данные в теле запроса (например, регистрироваться в сервисе), в FastAPI нужно использовать модели Pydantic. При этом пользователь, когда шлет информацию на ресурс работает с одним из HTTP-методов - `POST, PUT, DELETE, PATCH`, поэтому в FastAPI для настройки такого функционала нужно пользоваться соответствующими декораторами.
## немного о Pydantic
Это библиотека Python, которая используется для валидации данных и быстрой конвертации данных в разные форматы. 
Например, я хочу, чтобы в мое приложение приходила информация о пользователе - имя, возраст и электронная почта. При этом я знаю, что клиент может отправить эти данные только частично или заполнить поля каким-то бредом, поэтому мне нужно выдумать какую-то валидацию входящих данных. Тут на помощь приходит Pydantic.
Сначала мне нужно импортировать основной класс для работы - `BaseModel` и унаследовать от него свой собственный класс:
```Python
from pydantic import BaseModel
class User(BaseModel):
	name: str
	age: int
```
Теперь я могу проверять входящие данные на соответствие типам:
```Python
user = User(name="Alice", age=30)
```
Если все типы совпадут с заданными в классе, то все пройдет гладко - в переменную будет записан экземпляр класса, к которому мы будем обращаться за значениями.
А если будет передан ошибочный ввод:
```Python
user = User(name=30, age="Alice")
```

## создаем модель + делаем запрос
Я создам точно такую же модель, какую прописали в руководстве FastAPI - `Item`:
```Python
from typing import Union
from pydantic import BaseModel
class Item(BaseModel):
	name: str
	price: float
	tax: Union[str, None] = None
	description: Union[str, None] = None
```
Далее, я хочу написать функцию, которая будет принимать на вход тело запроса, проверять его через модель и возвращать обратно пользователю - эхо функция.
```Python
@app.post("/items/")
def create_item(item: Item):
	return item
```
Теперь, чтобы обратиться к этому эндпоинту мне нужно отправить на него данные. Данные должны быть отправлены в JSON формате. Сделать я это могу с помощью утилиты `curl`:
```Bash
 curl -X POST -H 'Content-Type: application/json' -d '{"name": "Foo", "price": 45.9}' http://127.0.0.1:8000/items/
```
Аргументы такие:
1) `-X` - позволяет определить HTTP метод, для отправки запроса.
2) `-H` - позволяет определить дополнительные HTTP заголовки запроса, которые нужно отправить, наряду с основными заголовками. В данном случае я передал в заголовке, что тип отправляемой мной информации - JSON.
3) `-d` - после этого аргумента прописывается строка, которую нужно передать в качестве сообщения.
В ответ мне вернулся мой же запрос:
```Bash
{"name":"Foo","description":null,"price":45.9,"tax":null}
```
При этом сервер отреагировал положительно на запрос:
```Bash
INFO:     127.0.0.1:39202 - "POST /items/ HTTP/1.1" 200 OK
```
## преимущества использования модели Pydantic
Во-первых, я могу извлечь из пришедшего экземпляра класса `Item` значения, переданные в теле запроса, с помощью стандартного интерфейса Python:
```Python
@app.post("/items/")
def create_item(item: Item):
	item_dict = item.dict()
	if item.tax is not None:
		price_with_tax = item.price + item.tax
		item_dict.update({"price_with_tax": price_with_tax})
	return item_dict
```
Теперь, если отправить запрос на эндпоинт с заполнением поля `tax`, то нам должен вернуться расширенный ответ с новым ключом - `price_with_tax`:
```Bash
# запрос
curl -X POST -H "Content-Type: application/json" -d '{"name": "Foo", "price": 45.6, "tax": 5.1}' http://127.0.0.1:8000/items/
# ответ
{"name":"Foo","description":null,"price":45.6,"tax":5.1,"price_with_tax":50.7}
```
Так получилось сделать, потому что использование класса Pydantic дает возможность проверять данные, приводить данные к определенным типам + он дает возможность вытаскивать данные из тела запроса по атрибуту: `item.price`, `item.tax`.
Во-вторых, FastAPI достаточно умный, чтобы отделить параметры пути от тела запроса, если они будут переданы одновременно в функцию:
```Python
@app.route("/items/")
def create_item(item: Item, number: int = 1):
	item.price = item.price * number
	return item
```
Теперь запрос:
```Bash
 curl -X POST -H "Content-Type: application/json" -d '{"name": "foo", "price": 40}' http://127.0.0.1:8000/items/\?number\=
3
{"name":"foo","description":null,"price":120.0,"tax":null}
```
Точно так же все работает, если передать комбинацию из параметров пути + query-параметров + тело запроса.
