FastAPI крайне похож на Flask, но у него есть свои отличия.
## установка и запуск
### установка
Для начала я установлю FastAPI в папку виртуального окружения:
```Bash
python -m pip isntall fastapi
python -m pip install "uvicorn[standard]"
```
Как я понял существует куча разных способов поставить FastAPI на комп. Мой способ устанавливает отдельно пакет FastAPI и ASGI сервер для него. ASGI сервер в данном случае называется `"uvicorn[standard]"` и эта запись на самом деле ссылается на проект веб-сервера Univer.
Веб-сервер нам нужен, чтобы запустить наше приложение на FastAPI.
Альтернативная установка - через команду:
```Bash
python -m pip install "fastapi[standard]"
```
### запуск
Под запуском приложения на фреймворке обычно понимается запуск сервера разработки с возможностью зайти на какой-нибудь ресурс. В обоих случаях нам нужно запустить сервер Uvicorn, однако это можно сделать разными способами.
Чтобы запустить сервер первого варианта установки, нам нужно обратиться к утилите `uvicorn`:
```Bash
uvicorn main:app --reload
```
Параметр `--reload` в данном случае указывает на то, что мы будем перезагружать сервер каждый раз, когда вносим изменения.
Чтобы запустить сервер второго варианта установки нужно обратиться к утилите `fastapi`:
```Bash
fastapi dev main.py
```
По какой-то причине команда `fastapi dev main.py` у меня работает через жопу - запускается сервер, все идет нормально, а после из-за странного глича сервер не хочет завершать свою работу через `ctrl + C` и продолжает крутиться, пока я не отключу его принудительно через `htop`.
## создаем класс
Как и во Flask, первое, что нужно сделать в FastAPI - экземпляр основного класса:
```Python
from fastapi import FastAPI


app = FastAPI()
```
Дальше я сразу создал маршрут и функцию, которые будут отвечать за главную страницу ресурса:
```Python
from fastapi import FastAPI


app = FastAPI()


@app.get("/")
def homepage():
	return {"message": "Привет мир!"}
```
Если я запущу сервер и перейду по ссылке, то попаду на страницу, которая покажет мне мой словарь с сообщением:
![[message_hello_fastapi.png]]
Создавать инстанс `FastAPI` обязательно для запуска приложения на FastAPI. Далее, я написал функцию `homepage` - она украшена декоратором `@app.get("/")`. `get` описывает метод, обращение по которому должна принимать эта функция по пути `/`. Путь пишется в скобках для метода. При запущенном сервере, когда будут поступать запросы к приложению, фреймворк будет по этим декораторам искать подходящий шаблон пути.
### немного про async
В документации по FastAPI разработчики часто пишут функции с приставкой `async`, например:
```Python
@app.get("/")
async def homepage():
	return {"message": "Привет мир!"}
```
Они сами написали страницу на своем ресурсе о том в каком случае нужно использовать асинхронные функции [ссылка](https://fastapi.tiangolo.com/async/#in-a-hurry). Вкратце, там описано, что `async` следует использовать, когда этого требуют пакеты сторонних модулей. Во всех остальных случаях лучше использовать  просто `def`.
## документация
FastAPI пишет автоматическую документацию к сервису. Ее можно посмотреть по адресу:
```
http://127.0.0.1:8000/docs
```
Так же можно посмотреть альтернативную версию той же документации:
```
http://127.0.0.1:8000/redoc
```
## я хочу шаблоны
В FastAPI существует встроенный коннект с шаблонизатором jinja2, однако я захотел разобраться как будет работать мой проект, если я не буду пользоваться исключительно встроенными инструментами фреймворка, а попытаюсь воспользоваться отдельно jinja2.
Для начала нужно установить шаблонизатор:
```Bash
python -m pip install jinja2
```
jinja2 работает через свой главный класс `Environment`:
```Python
from jinja2 import Environment, PackageLoader, select_autoescape


env = Environment(
	loader=PackageLoader("appname"),
	autoescape=select_autoescape()
)
```
По порядку:
1) `Environment` - основной класс, который всегда должен присутствовать, если нам необходимо работать с шаблонами через jinja2. Этот класс содержит все первичные настройки шаблонизатора и указывает папку, где jinja2 должна искать сами шаблоны.
2) `loader` - параметр, который определяет какой загрузчик шаблонов будет использован в проекте.
3) `PackageLoader` - класс, который по переданному аргументу определит окружение, где хранится наш проект + все шаблоны к нему. Внимание! Вместо `"appname"` должно быть написано имя папки, которая лежит в той же директории, где находится текущий исполняемый файл.
4) `autoescape` - это параметр, который определяет какой движок будет использован для экранирования наших шаблонов.
Экранирование - это защита нашего кода от XSS-атак. То есть если плохой человек захочет нам в код вставить вредоносный скрипт, то все служебные опасные символы этого скрипта будут преобразованы в безопасные.
