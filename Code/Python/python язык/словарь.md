в основе python словарей лежит *хэш-таблица*.
# словарные включения
строит объект *dict* из пар *key:value*, которые должны быть извлечены из итерируемого объекта.
```Python
digital_codes = [
    (880, "Bangladesh"),
    (55, "Brazil"),
    (86, "China"),
    (91, "India"),
    (62, "Indonesia"),
    (81, "Japan"),
    (234, "Nigeria"),
    (92, "Pakistan"),
    (7, "Russia"),
    (1, "United States")
]
country_codes = {country: code for country, code in digital_codes}
print(country_codes)  # => {880: 'Bangladesh', 55: 'Brazil', 86: 'China', 91: 'India', 62: 'Indonesia', 81: 'Japan', 234: 'Nigeria', 92: 'Pakistan', 7: 'Russia', 1: 'United States'}
```
внутри включения, как и во всех других включениях можно использовать условную конструкцию:
```Python
other_cc = {country.upper(): code for code, country in sorted(country_codes) if code > 70}
print(other_cc)  # => {'UNITED STATES': 1, 'RUSSIA': 7, 'BRAZIL': 55, 'INDONESIA': 62, 'JAPAN': 81, 'CHINA': 86, 'INDIA': 91, 'PAKISTAN': 92, 'NIGERIA': 234, 'BANGLADESH': 880}
```
# оператор `**`
если оператор `*` может "распаковать" список, то `**` открывает аналогичные операции со словарем.
```Python
def my_func(x, y, z):
	print(x, y, z)
	
a = {'x': 1, 'y': 2, 'z': 3}
my_func(**a)  # => 1 2 3
```
оператор так же можно применять к более чем одному аргументу вызываемой функции: 
```Python
def my_func1(**kwargs):
	print(kwargs)
	
a = {'x': 1, 'y': 2, 'z': 3}
b = {'a': 10, 'b': 20}
my_func1(**a, **b)  # => {'x': 1, 'y': 2, 'z': 3, 'a': 10, 'b': 20}
```
в этом случае **запрещено** предавать два словаря с одинаковыми ключами.
так же можно с помощью оператора "распаковать" словарь в другой словарь: 
```Python
c = {'c': 30, **a, **b}
print(c)  # => {'c': 30, 'x': 1, 'y': 2, 'z': 3, 'a': 10, 'b': 20}
```
в этом случае можно, чтобы оба словаря содержали одинаковые ключи, но при записи в один словарь следующее вхождение ключа заменит собой предыдущее.
Ромальо называет выводы словарей "отображения" и мне кажется, что это слово применяется ко всем выводам, т.к. они являются отображениями структур данных. 
# оператор `|`
оператор `|` можно использовать для объединения словарных отображений:
```Python
a = {'x': 1, 'y': 2, 'z': 3}
b = {'a': 10, 'b': 20}
print(a | b)  # => {'x': 1, 'y': 2, 'z': 3, 'a': 10, 'b': 20}
```
с его помощью так же можно создавать новые отображения.
еще есть сокращенная версия оператора - `|=`.
# сопоставление с образцами
сопоставлять с образцами с помощью `match/case` можно не только кортежи и списки, но еще и словари, что является удобным при работе с JSON:
```Python
def my_func(values: dict) -> list:
	match values:
		case {"type": "book", "api": 2, "authors": [*names]}:
			return names
		case {"type": "book", "api": 1, "author": name}:
			return name
		case {"type": "book"}:
			raise ValueError(f"Invalid 'book' record: {record!r})")
		case {"type": "movie", "director": name}:
			return [name]
		case _:
			raise ValueError(f"Invalid record: {record!r}")

b1 = dict(
	api=1, type="book", author="Lev Tolstoy", title="War and Peace"
)
print(my_func(b1))  # => Lev Tolstoy
```
примечательно, что `OrderedDict` так же будет обрабатываться, как и обычная версия словаря:
```Python
from collections import OrderedDict

b2 = OrderedDict(
	api=2,
	"type"="book",
	"authors"="Matelli Ravenscroft Holden".split(),
	title="Python in Nutshell"
)
print(my_func(b2))  # => [Matelli, Ravenscroft, Holden]
```
при этом если я попрошу у функции какую-то фигню, то она мне вернет ее машинное представление благодаря `!r`:
```Python
my_func("Spam, spam, spam")  # => ValueError: Invalid record: 'Spam, spam, spam'
```
>[!important] важно, что для сопоставления отображений (в отличии от сопоставления последовательностей) нам не нужно полное совпадение всех элементов. в примере выше я передаю в функцию словарь с дополнительным ключом `title` и `case/match` все равно находит сопоставления с первым и вторым шаблонами, где  ключа `title` нет.
