## база
При запуске программы на Python все строки, которые находятся в файле с расширением `.py` компилируются в байт-код. Байт-код - это низкоуровневое представление исходного кода.
## байт-код файлы
В версиях Python до 3.2 скомпилированные файлы исходного кода сохранялись в директории, рядом с файлами исходного кода, с расширением `.pyc`. В поздних версиях такие скомпилированные файлы сохраняются в папке `__pycache__`, которая создается автоматически при запуске программы. 
Файлы, которые сохраняются в __pycache__, именуются по типу `script.cpython-33.pyc`. Это название создано, чтобы отображать текущую реализацию Python `cpython` + используемую версию Python `33`.
Иногда такие файлы сохраняются в оперативную память, время их жизни в этом случае составляет одну сессию.
Байт-код формируется каждый раз когда изменяется временная метка изменения файла.
Байт-код файлы облегчают повторный запуск Python-программы, если они уже были созданы, - мы пропускаем шаг компиляции.
>[!important] Байт-код сохраняется лишь для тех файлов, которые импортируются, но не для файлов верхнего уровня программы, исполняемых как сценарии.
## PVM
После того как байт-код сделан он все еще не может быть сразу напрямую отправлен процессору для исполнения. Для этого ему нужно совершить еще один шаг - пройти через виртуальную машину Python (Python Virtual Machine).
# управление памятью
Когда говорят "интерпретатор Python" обычно имеют в виду систему CPython - одну из реализаций Python. CPython включает в себя компилятор (который превращает мои программы в байт-код) и PVM.
Интерпретатор Python выполняет некую работу с памятью в компьютере, на котором запущен python-код. Что конкретно он делает?
Оперативная память компьютера состоит из *набора ячеек (sequence of words)*. Каждая *ячейка (word)* - это единица памяти. Размер такой единицы памяти может быть фиксированным (чаще всего), а может быть переменным.
Каждый процесс требует ресурсов. Для того чтобы свободно распоряжаться этими ресурсами Python создает для себя "песочницу" в оперативной памяти, где сохраняет все свои объекты и конструкции. Эта песочница называется **private heap**. То есть каждый раз, когда я запускаю программу на Python операционная система выделяет под нее процесс. На каждый процесс выделяется часть оперативной памяти и ресурсы (процессорное время, доступ к файлам и т.д.). За менеджмент этими ресурсами и этим процессом отвечает **Python memory manager**.
Внутри **private heap** существует еще две разновидности памяти - **HEAP memory** и **STACK memory**.
В **HEAP memory** хранятся значения переменных и значения функций.
В **STACK memory** хранятся сами переменные и сами функции.
![[stack-heap.webp]]
Значения переменных и выводы функций - это **объекты**. Ссылки на объекты хранятся в **STACK memory**.
Python считает для каждого объекта в памяти количество ссылок на него. Это необходимо, т.к. в случае нулевого количества ссылок объект может быть свободно очищен из памяти **сборщиком мусора (garbage collector)**.
# GIL
Python медленный, потому что существует GIL. **GIL** - это механизм в CPython, который позволяет Python-коду выполняться только в одном потоке, даже если потоков доступно несколько.
GIL - это мьютекс. А **мьютекс** - это слово для описания механизмов, которые обеспечивают эксклюзивный доступ к общему ресурсу. Делается это, чтобы один ресурс не был "разорван" несколькими разными потоками, т.к. использование одного ресурса несколькими потоками вызывает аномалии.
