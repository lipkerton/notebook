# База

Менеджер контекста - это все те места, где применяется слово `with` .

```Python
with open('file.txt', 'r') as file:
	info = file.readlines()
	print(info)
```

Для чего это нужно? В случае чтения файла менеджер контекста используется, чтобы открыть файл в каком-нибудь режиме, сделать с ним какую-нибудь тему (например, вытащить из файла данные), а потом автоматически закрыть.

```Python
file = open('file.txt', 'r')
info = file.readlines()
file.close()
print(info)
```

Фишка в том, что без использования контекстного менеджера приходится самостоятельно “закрывать файл”. “Открытый файл” занимает место в оперативной памяти - тратит лишние ресурсы. В маленьком скрипте это не так заметно, но когда программа разрастается менеджер контекста помогает лаконично тратить ресурсы системы.

# Что еще?

Кроме файлов, с чем еще можно использовать менеджер контекста? При открытии соединения с базой данных SQL и при создании сокета.

```Python
import sqlite3


with sqlite.connect('database.db') as conn:
	cursor = conn.cursor()
	cursor.execute('CREATE TABLE test(name TEXT)')
```

После окончания последней строчки кода соединение с базой автоматически закроется.

```Python
import socket


with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
	pass
```

После окончания блока менеджера сокет автоматически закроется и память будет освобождена.
