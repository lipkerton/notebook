Для начала: модель данных - это абстрактное понятие, которое я не совсем понимаю. Когда мы говорим о Python под моделью данных обычно имеют в виду то как объекты взаимодействуют с языковыми конструкциями.
Например, у нас есть языковая конструкция "вызов элемента по индексу":
```Python
m = range(1, 100)
m[55]  # вызов по индексу.
```
Здесь есть операция `[55]`, которая является извлечением индекса + есть объект `m`, с которым происходит эта операция, - такое взаимодействие и есть уникальная для Python модель данных.
Грубо говоря модель данных - это способ взаимодействия конструкций языка с объектами.
Здесь в игру вступает понятие "интерфейс": модель данных - это интерфейс, следуя которому мы можем построить с взаимодействие своих собственных объектов с языковыми конструкциями Python.
На практике это означает, что мы можем настроить свои собственные экземпляры классов так, чтобы они могли работать в связке с конструкциями языка Python. В этом нам помогут [[магические методы]].
## абстрактные классы и интерфейс
Магические методы - важная штука в Python не только потому, что с их помощью можно построить удобные пользовательские объекты. Главная фишка магических методов в том, что их сочетание (в разных комбинациях) составляет основу для базовых типов данных в Python.
Как это работает? Существует вещь, которая называется "абстрактный базовый класс" - это такой класс, в котором существует набор методов, но от которого нельзя сделать ни одного объекта. Зачем тогда он нужен? Прежде чем ответить на этот вопрос для "абстрактного базового класса", нужно сначала разобраться с вопросом "что такое абстрактный класс?". 
Такой класс является чем-то вроде схемы для нормального класса. Если мы унаследуем наш нормальный класс от абстрактного, то код будет возвращать ошибку каждый раз, когда мы будем его запускать, не прописав все методы абстрактного класса в классе-наследнике.
Строго говоря абстрактный класс определяет набор методов, которые точно должны быть в любом классе, который от него наследуется.
Что такое интерфейс? Его еще иногда называют API. Интерфейс - это ПО, к которому мы можем обращаться, чтобы с помощью его средств построить какие-то свои структуры. В данном случае абстрактные классы являются интерфейсами, потому что с их помощью мы можем построить свои собственные классы.
### создать абстрактный класс
Чтобы создать абстрактный класс нужно воспользоваться модулем `abc`. Я вытащу из этого модуля класс `ABC` и унаследую от него мой абстрактный класс:
```Python
import abc

class Shape(abc.ABC):
    @abc.abstractmethod
    def area(self): pass

class MyClass(Shape):
    pass

a = MyClass()
```
Здесь в классе `MyClass` нет метода, который заявлен в абстрактном классе. Поэтому, если запустить код, то получится ошибка:
```Python
    a = MyClass()
TypeError: Can't instantiate abstract class MyClass without an implementation for abstract method 'area'
```
### что такое базовый абстрактный класс?
Базовый абстрактный класс в Python - это в сущности шаблон, с помощью которого в коде представлены все стандартные типы Python. Можно вспомнить, что список, например, имеет метод `.append()` - где определен этот метод? В классе встроенного типа. 
А как мы вообще отличаем список от словаря? По набору инструментов, которыми можно работать с этими типами данных. Но где этот набор определен? Набор представлен в виде абстрактного базового класса. И мы можем достать этот базовый класс, чтобы он подсказал как можно сделать, например, коллекцию самостоятельно.
```Python
from collections import abc

class MyClass(abc.Collection):
    pass

a = MyClass()
```
Если выполнить эту программу, то она выдаст ошибку:
```Python
	a = MyClass()
Traceback (most recent call last):
TypeError: Can't instantiate abstract class MyClass without an implementation for abstract methods '__contains__', '__iter__', '__len__'
```
Ошибка возникает, потому что мы не определили обязательные для любой коллекции магические методы. Это означает, что любая коллекция должна их содержать, потому что без них коллекция не будет коллекцией.
Что в Python является коллекцией? Коллекцией являются списки, строки, словари и множества. Все они должны содержать метод `__contains__()`, метод `__iter__()` и метод `__len__()`, но их различие в том, что перед тем как стать коллекцией они еще имели кучу своих фишек, которыми они и различаются.
Наборы таких фишек называются "специализация коллекции" - у коллекции всего три специализации: `Sequence`, `Mapping`, `Set`. Все три являются интерфейсами для встроенных типов данных:
1) `Sequence` - класс, который является основой для `list`, `str`.
2) `Mapping` - класс, который является основой для `dict`.
3) `Set` - класс, который является основой множеств `set`, `frozenset`.
Мы можем вызвать их, чтобы вручную построить встроенные типы:
```Python
from collections import abc

class MyClass(abc.Sequence):
	pass

a = MyClass()
```
Загадка состоит в следующем - в книжке Лусиану Рамальо пишет, что для реализации встроенного типа нужно несколько компонентов; частности `Sequence` должен обладать следующими: `__getitem__`, `__contains__`, `__iter__`, `__reversed__`, `index`, `count`; внимание вопрос - почему в результате запуска программы появляется ошибка:
```Python
    a = MyClass()
TypeError: Can't instantiate abstract class MyClass1 without an implementation for abstract methods '__getitem__', '__len__'
```
А когда я их прописываю, то ошибка пропадает и все работает. Моя теория в том, что метод `__getitem__` реализует все минимальные запросы абстрактного класса `Sequence`. Например, с его помощью можно перебирать последовательность без метода `__iter__`, можно проверять наличие элемента в последовательности без метода `__contains__`.
