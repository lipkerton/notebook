## о магических методах
Магические методы - это специальные функции в пакете Python. Еще они называются "dunder-методы", потому что обрамлены двойным подчеркиванием.
Метод может выглядеть, например, так: `__init__()`.
Обычно магические методы используются, чтобы задать объектам собственного класса какое-то специальное поведение. Однако не все так просто - на самом деле магические методы используются, чтобы интегрировать собственные классы в модель данных Python.
Магические методы никогда не должны вызываться пользователем. Хотя такое поведение не вызывает ошибки - оно все равно не является рекомендуемым.
Python однако сам иногда не пользуется магическими методами для встроенных типов, как `list`, `bytearray`, `str` - вместо этого он использует конструкции языка C. Например, вместо вызова метода `__len__()` для объекта `x` Python пойдет и узнает длину массива из атрибута `ob_size`, который принадлежит к структуре `PyVarObject`. 
## `__getitem__()`
Eсли нам нужно, чтобы к элементам объекта нашего класса можно было получить доступ по индексу, то можно воспользоваться методом `__getitem__(self, index)`.
```Python
class Numbers:
	numbers = range(1, 100)

	 def __getitem__(self, index):
		 return self.numbers[index]
```
Теперь можно получать элементы по индексу и использовать срезы:
```Python
p = Numbers()
p[0]
p[-1]
p[2:21:2]
```
Так же по объекты, которые будут созданы от такого класса можно будет спокойно перебрать в цикле.
Если сделать комбинацию методов `__getitem__()` и `__len__()`, то к нашему экземпляру можно будет применить функцию `choice()` из модуля `random`:
```Python
class Numbers:
	numbers = range(1, 100)

	def __getitem__(self, index):
		return self.numbers[index]

	def __len__(self):
		return len(self.numbers)

import random
p = Numbers()
print(random.choice(p))
```
Если попытаться использовать `choice()` без `__len__()`, то получится ошибка `TypeError` - функции `choice()` нужно знать количество элементов в последовательности.
### немного про итерирование
Магические методы часто могут выполнять чужую работу (работу других магических методов), но иным способом.
Например, если у нас написан класс `Numbers`, в котором есть метод `__getitem__()`, то мы можем использовать индексацию, итерировать по последовательности и т.д.
А еще мы можем использовать на такой последовательности оператор `in`, чтобы проверить содержит ли последовательность какой-то элемент. Если мы это сделаем, то оператор `in` в таком случае пройдет по всей последовательности, сравнивая каждый элемент с нужным.
Не самая эффективная стратегия, поэтому существует магический метод `__contains__()`, который отвечает за альтернативную вариацию `in`.
### немного про сортировки
Когда мы используем `__getitem__()`, то у нас появляется возможность сортировать наши объекты. Сортировать можно и обычным способом, например:
```Python
class NewClass:
	number = range(1, 100)

	def __getitem__(self, index):
		return self.number[index]

a = NewClass()
print(sorted(a))
print(sorted(a), reverse=True)
```
А можно придумать какой-нибудь ключ сортировки и сортировать по нему.
## `__contains__()`
Метод `__contains__()` вызывается итератором, когда к объекту класса применяется оператор `in`.
```Python
class NewClass:
	numbers = range(1, 100)

	def __contains__(self, value):
		return value in self.numbers
```
## булевы значения пользовательских типов
Казалось бы - зачем этот принцип нужен тут? Мы знаем на практике, что выяснить является ли объект правдивым, либо ложным можно тупо запихнув этот объект в функцию `bool()`.
```Python
print(bool(0))
print(bool(1))
print(bool([1, 2, 3]))
```
Так зачем нам нужна собственная реализация `__bool__()`?
Проблема со стандартной функцией `bool()` в том, что она считает правдивыми по умолчанию все пользовательские объекты. То есть, если мы запихнем в нее любой экземпляр нашего собственного класса, она скажет, что он правдив.
```Python
class NewClass:
	def __init__(self, number):
		self.numbers = range(number)

a = NewClass(0)
print(bool(a))  # True.
```
Важная особенность функции `bool()` в том, что сначала интерпретатор попытается вызвать реализацию `__bool__()`, которая предусмотрена для объекта. Если он такой не найдет, то попытается определить длину объекта через реализацию `__len__()`. Если ничто из этого не реализовано, то по умолчанию вернется `True`.
Значит у нас есть целых два способа вернуть `False`, если пользователь передал в качестве аргумента классу ноль.
Первый заключается в использовании метода `__bool__()`:
```Python
class NewClass:
	def __init__(self, number):
		self.numbers = range(number)
	def __bool__(self):
		return bool(self.numbers)

a = NewClass(0)
print(bool(a))  # False.
```
 Второй заключается в использовании метода `__len__()`:
```Python
class NewClass:
	def __init__(self, number):
		self.numbers = range(number)
	def __len__(self):
		return len(self.numbers)

a = NewClass(0)
print(bool(a))  # False
```
## `__add__()`
Метод, который отвечает за реализацию сложения какого-нибудь значения с объектом. Например, у нас есть два экземпляра класса, которые являются итерируемыми коллекциями, и мы хотим их сложить.
```Python
class NewClass:
	def __init__(self, iterable):
		self.iterable = list(iterable)
	def __add__(self, other):
		return self.iterable + other.iterable

a = NewClass([1, 2, 3])
b = NewClass('5678')
print(a + b)  # [1, 2, 3, '5', '6', '7', '8']
```
Так же мы можем немного извращаться. Например, я хочу, чтобы к объектам моего класса могли прибавляться не только объекты моего же класса, но еще и последовательности.
```Python
class NewClass:
    def __init__(self, iterable):
        self.iterable = list(iterable)
    def __add__(self, other):
	    if isinstance(other, NewClass):
	        return self.iterable + other.iterable
	    return self.iterable + list(other)
```
## `__repr__()` и `__str__()`
Чем они различаются? В сущности они различаются целью выполнения.
`__repr__()` нужен, чтобы представить максимально точное описание объекта для разработчика, а `__str__()` нужен, чтобы отобразить подробное представление объекта для пользователя.
Самая большая разница между ними обнаруживается, если попытаться сделать два класса для `__repr__` и `__str__` во встроенном интерпретаторе.
```Python
def MyClass1:
	a = 1
	def __repr__(self):
		return f'{self.a}'

def MyClass2:
	a = 1
	def __str__(self):
		return f'{self.a}'

a = MyClass1()
b = MyClass2()
>>> a  # => 1 
>>> b  # => <__main__.MyClass2 object at 0x102ad4590>
```
Почему для метода `__str__` вернулась такая строка? Потому что метод `__repr__` именно репрезентирует сам объект, а метод `__str__` просто описывает строковое представление для объекта.
>[!important] Хороший способ определить, что метод `__repr__` удачно репрезентирует объект - функция `eval()`. Она выполняет строку, которая ей передана, если внутри строки есть валидное выражение на языке Python. Метод `__repr__` должен возвращать такое значение, чтобы функция `eval()` могла его обработать и выполнить.
