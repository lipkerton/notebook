В питоне есть встроенный модуль для создания логов.
```Python
import logging

logging.debug('Привет!')
logging.warning('Предупреждаю!')
```
Если выполнить такую программу, то в консоли напечатается:
```Bash
WARNING:root:Предупреждаю!
```
Почему не напечаталось "Привет!"? 
В `logging` есть несколько уровней сообщений:
1) `DEBUG` - самый высокий уровень. На нем выводится вся информация программы.
2) `INFO` - как DEBUG, только выводится меньше информации. Обычно используется для вывода сообщений о том, что все работает как нужно.
3) `WARNING` - предупреждение о том, что что-то в программе пошло не по плану или о том, что, например, осталось мало места на диске.
4) `ERROR` - произошла ошибка. Обычно означает, что программа не смогла выполнить какое-то действие.
5) `CRITICAL` - крупная ошибка. Означает, что программа не может продолжить работу.
По умолчанию `logging` печатает сообщения только уровня `WARNING` и ниже. Это поведение можно изменить. Для этого. нужно вызвать класс `basicConfig` и прописать в нем параметр `level`.
```Python
import logging

logging.basicConfig(level='INFO')
logging.info('Привет!')
```
Кроме этого `basicConfig` помогает настроить вывод логов, например, таким образом:
```Python
import logging

logging.basicConfig(
	level='INFO',
	filename='test.log'
)
```
В папке появится файл `test.log` и все записи логов будут отправлены в него.
Кроме этого можно задать формат сообщений в логах:
```Python
import logging

logging.basicConfig(
	level='INFO',
	filename='test.log',
	format='%(asctime)s %(levelname)s %(lineno)s %(message)s'
)
logging.info('Where is my shoe??')
```
Тогда в логах появится сообщение:
```log
2025-05-02 13:51:21,997 INFO 9 Where is my shoe??
```
Модуль `logging` сам рассчитает значение для переменных `asctime`, `levelname`, `lineno` (номер строки).
