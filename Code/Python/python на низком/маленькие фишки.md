# slice
вместо буквального написания среза:
```Python
m = 'sdfsdfsdf'
m[0:6]
```
можно запаковать срез в переменную с помощью функции `slice()`:
```Python
m = 'sdfsdfsdf'
SKY = slice(0, 6)
m[SKY]
```
# список списков и конкатенация
иногда нужна фишка построить список, который состоит из списков. Когда решается такая проблема, сразу возникает желание выполнить конкатенацию через умножение списка:
```Python
m = [['_'] * 3] * 3
[['_', '_', '_'], ['_', '_', '_'], ['_', '_', '_']]
```
при любом использовании конкатенации важно помнить, что в результате конкатенации создается новый объект. Эта особенность при подобном умножении выливается в то, что по сути мы не создаем список списков, наоборот, мы создаем список, который содержит три ссылки на второй список. В итоге, если мы захотим поменять элемент в одном из подсписков:
```Python
m[0][0] = 'X'
[['X', '_', '_'], ['X', '_', '_'], ['X', '_', '_']]
```
Чтобы такой ситуации избежать лучше использовать генераторное выражение.
# загадка составного присваивания
Оператор составного присваивания это вот такая штука: `*=`, `+=`, `-=` и т.д. Загадка: что произойдет при выполнении такого кода?
```Python
a = ('a', 'b', [10, 20])
a[2] += [30, 40]
```
Ответ: ошибка **НО** сами значения окажутся в списке.
```Python
a = ('a', 'b', [50, 60])
a[2] += [70, 60]

Traceback (most recent call last):
  File "<python-input-16>", line 1, in <module>
    a[2] += [70, 60]
    ~^^^
TypeError: 'tuple' object does not support item assignment

a

('a', 'b', [50, 60, 70, 60])
```
Эту же операцию можно выполнить без ошибок с помощью метода `.extend()`.
# enum
В Python существует куча модулей, которые могут быть использованы чисто для понта, предлагая при этом минимальные улучшения кода. Один из таких модулей - `enum`.
Сначала я воспользовался только классом `Enum` из `enum`:
```Python
from enum import Enum

class Cool(Enum):
	number_one = 1
	number_two = 2
	number_three = 3

print(Cool.number_one.name)  # 'number_one'
print(Cool.number_one.value)  # 1
```
Механика проста - делаем класс, прописываем туда некие константы, а затем вызываем эти константы через точечную нотацию, обращаясь к имени класса. Зачем это нужно? Кто знает. Однако через `enum` можно на практике записывать какие-то константы в коде.
Например, в одном из прошлых проектов у меня был файл с константами `constants.py` и константы в нем были написаны на чистом листе примерно так:
```Python
СONST_ONE = 1
CONST_TWO = 2
CONST_THREE = 3
```
Очевидно, что иногда количество констант может достигнуть такого числа, что будет тяжело разобраться с хаосом: "К какому модулю относится эта константа, а к какому модулю эта?"
Как раз в этом случае очень удобно использовать `enum`.

# str с запасом в памяти
Лусиану Ромальо говорит, что экземпляры класса `str` создаются специально с запасом в памяти, чтобы при конкатенации строк "не нужно было копировать всю строку".
Это означает, что операция конкатенации `+=` со строками работает эффективнее, чем с другими неизменяемыми последовательностями.
# bisect.insort
Допустим у меня есть отсортированная последовательность и при этом мне нужно вставлять в нее элементы так, что порядок сортировки сохранялся. В этом случае простой `append()`, например, не подойдет, потому что он вставит элемент в конец списка и не будет заботится о порядке всех остальных элементов внутри.
Здесь на помощь приходит модуль `bisect`, который имеет функцию `insort`. Она принимает на вход список и значение, которое нужно в этот список вставить, а затем она автоматически определяет позицию, где должен находиться вставляемый элемент, чтобы список оставался в упорядоченном состоянии.
