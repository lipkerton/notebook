## объекты в памяти
Каждый объект в Python имеет заголовок с метаданными. Метаданные представляют сведения о значении объекта, количестве ссылок на объект и указатель типа объекта (иногда метаданных может быть больше). Например, число `1.28` в памяти будет иметь три поля - два поля обозначают метаданные об объекте, а третье его значение:
1) `ob_refcnt` (reference counter) - счетчик ссылок на объект.
2) `ob_type` - указатель на тип объекта.
3) `ob_fval` (value??) - число типа `double` (язык C), в котором хранится значение с плавающей точкой.
## последовательности
Последовательности в Python делятся на две группы (по типу объектов, которые хранятся в последовательностях):
1) Контейнерные - `list, tuple, collections.deque`
2) Плоские - `str, bytes, array.array`
Их основное различие в том, что контейнерные последовательности хранят не сами объекты внутри себя в памяти, а ссылки на эти объекты.
Напротив, плоские последовательности хранятся непрерывно в памяти, т.е. они держат внутри себя сами значения, а не ссылки на объекты. Плоские последовательности компактнее и быстрее работают, потому что они представляют из себя один единый объект в памяти. Однако, плоские последовательности являются плоскими именно потому, что могут хранить только один тип объектов (например, только строковые значения или только числовые значения). В этом смысле они хранят только примитивные машинные значения.
>[!important] Массив (`array`) (например, чисел с плавающей точкой) гораздо компактнее, чем кортеж (таких же чисел), потому что он представляет собой один объект в памяти, а кортеж (или другие контейнерные последовательности) представляют собой набор из ссылок на множество объектов.

Так же существует разделение по изменяемости последовательности. Существуют:
1) Изменяемые - `list, bytearray, array.array`
2) Неизменяемые - `str, bytes, tuple`
Изменяемость в данном случае означает, что при добавлении элементов в последовательность или изъятии элементов из нее сам объект последовательности останется прежним. 
Напротив, если попытаться изменить неизменяемую последовательность, то в памяти просто будет создан новый объект с теми изменениями, которые мы хотели в него внести.
Изменяемые последовательности (mutable sequence) наследуют все свои методы от обычных неизменяемых (sequence) + реализуют несколько своих уникальных методов.
>[!important] Встроенные типы не унаследованы от классов `abc.Sequence` и `abc.MutableSequence` в стандартном смысле термина "наследование", но они являются "виртуальными подклассами" этих классов. Я пока не знаю, что означает "виртуальный подкласс".
## кортежи
В учебниках кортежи часто описывают как неизменяемые списки, однако такое их обозначение часто сужает представление о кортеже, как о типе данных.
Если представлять кортеж, как неизменяемый список, то важность порядка элементов в нем может быть важна, а может быть не важна в зависимости от контекста. Но если представлять кортеж как запись с неименованными полями, то порядок элементов в такой записи становится крайне важным, а само количество элементов ограниченным.
### кортеж как запись
Когда мы говорим о кортеже, как о записи, то семантика (смысл) элемента напрямую зависит от его позиции + смысл всей записи меняется или уничтожается при изменении позиций элементов.
Например, у нас может быть пара элементов - широта и долгота. Я размещу их в кортеже и в данном случае позиция каждого из двух элементов важна, т.к. при смене их позиций меняются координаты точки:
```Python
lax_coordinates = (33.9425, -118.408056)
```
### кортеж как неизменяемый список
Кортеж часто используют как разновидность списка. У этого есть свои преимущества:
1) мы точно знаем, что набор полей кортежа не изменится
2) кортеж занимает меньше пространства в памяти, чем список
Однако неизменяемость кортежа в данном случае распространяется только на сам набор ссылок внутри него, а не на его объекты. Это означает, что мы не можем внести изменения в количество ссылок и не можем подменять ссылку на конкретный объект ссылкой на другой объект, однако мы можем изменять содержимое объекта, который доступен по ссылке:
```Python
>>> cort = ('A', 'B', [1, 2, 3])
>>> cort[0] = 'C'
Traceback (most recent call last):
  File "<python-input-10>", line 1, in <module>
    cort[0] = 'C'
    ~~~~^^^
TypeError: 'tuple' object does not support item assignment
>>> cort[2][0] = 4
>>> cort
('A', 'B', [4, 2, 3])
```
Кортежи лучше делать неизменяемыми всегда, потому что изменяемость элементов внутри кортежей может вызвать ошибки. Например, объект допускает хеширование только когда его значение никогда не может быть изменено, изменяемый элемент внутри кортежа не допустит хеширование.
#### кортеж эффективнее списка?
Если мне нужно сохранить зафиксированное количество элементов, то, да, кортеж будет эффективнее списка. В книге есть несколько не совсем понятных разъяснений, однако самое существенное в том, что при создании списка под него в памяти выделяется так же запасное место (кроме того основного, которое он занимает), а для кортежа нет. Так происходит потому, что мы не ожидаем, что кортеж будет расширяться.
Еще одна штука - и у кортежа и у списка есть некий массив, который хранит в себе ссылки на те объекты, которые содержит кортеж/список. Такой массив у кортежа хранится в самой структуре кортежа, а у списка он хранится в других местах (при этом в списке сохраняется указатель на этот массив). Это называется **косвенность**, что является дополнительным уровнем абстракции между элементами в памяти - такой подход нужен списку, чтобы перемещаться по памяти, когда в нем становится больше элементов.
### распаковка
Кортеж можно "распаковать". Эта процедура часто означает, что мы берем компоненты кортежа и присваиваем их в качестве значений переменным. Например так:
```Python
>>> cort = ('A', 'B')
>>> a, b = cort
>>> a
'A'
>>> b
'B'
```
Так же это иногда может означать, что мы берем компоненты кортежа и подставляем их в качестве элементов в форматированную строку:
```Python
>>> traveler_ids = [('USA', '31985555'), ('BRA', 'CE342567'), ('ESA', 'XDA2056856')]
>>> for passport in sorted(traveler_ids):
...     print('%s/%s' % passport)
...
BRA/CE342567
ESA/XDA2056856
USA/31985555
```
В данном случае оператор `%` трактует каждый элемент кортежа, как отдельное поле.
По какой-то причине этот механизм не работает со списками:
```Python
>>> traveler_ids = [['USA', '31985555'], ['BRA', 'CE342567'], ['ESA', 'XDA2056856']]
>>> for passport in sorted(traveler_ids):
...     print('%s/%s' % passport)
...
Traceback (most recent call last):
  File "<python-input-3>", line 2, in <module>
    print('%s/%s' % passport)
          ~~~~~~~~^~~~~~~~~~
TypeError: not enough arguments for format string
```
