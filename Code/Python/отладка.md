как отладить код? То есть как посмотреть все переменные, которые используются в функциях, все объекты всех классов и проч.? Помимо самой стандартной идеи - использовать отладчик, который встроен в какую-нибудь IDE, - можно воспользоваться встроенными в Python методами.
### vars, locals, globals
`vars()` - внутри функции показывает все значения локальных переменных и переданных в функцию параметров. При этом если ее напечатать снаружи пользовательской функции, то она покажет все глобальные переменные, которые связаны с этим файлом.
```Python
def two_numbers(*args):
	print(vars())  # внутри функции.
print(vars())  # снаружи функции.
```
Вывод:
```Python
{'args': (1, 2, 3, 4, 5)}  # внутри.
{
	'__name__': '__main__',
	'__doc__': None,  # строка документации на первой строчке в файле.
	'__package__': None,
	'__loader__': <_frozen_importlib_external.SourceFileLoader object at 0x7f813061ab10>,
	'__spec__': None,
	'__annotations__': {},
	'__builtins__': <module 'builtins' (built-in)>,
	'__file__': '/home/lipkerton/lessons/my_unittest/file_
2.py',
	'__cached__': None,
	'two_numbers': <function two_numbers at 0x7f8130477100>

```
По схожему принципу работает функция `locals()`.
Только функция `global()` в любом месте программы покажет именно глобальные переменные - даже если будет вызвана из пользовательской функции.
### декораторы
декораторы еще один вариант отладки функций. В декораторе можно прописать какую информацию функция должна выдавать на выходе. А еще можно сделать так, чтобы декоратор был универсальным и мог бы обрабатывать сразу несколько функций.
```Python
def dump(func):
	def wrapped(*args, **kwargs):
		print('Имя функции:', func.__name__)
		print('Аргументы функции:', vars())
		result = func(*args, **kwargs))
		print('Результат функции:' result)
		return result
	return wrapped


@dump
def mult_numbers(*args):
	return sum(args)


print(mult_numbers(1, 2, 3, 4, 5))
```
Результат будет таким:
```Python
Имя функции: two_numbers
Аргументы функции: {
	'args': (1, 2, 3, 4, 5),
	'kwargs': {},
	'func': <function two_numbers at 0x7faf4ce02700>
}
Результат работы функции: 15
```
### встроенный отладчик
pdb - это наименование встроенного отладчика. 
Чтобы его использовать сначала нужно запустить файл в режиме отладки:
```Python
python -m pdb <имя_файла>
```
Внутри отладчика есть несколько простых команд:
`s`(step) - шаг вперед. Отладчик перейдет на следующую строку кода.
`c`(continue) - программа продолжит работать в естественном режиме пока не возникнет ошибка или пока она не завершится.
`n`(next) - программа перепрыгнет через текущую функцию к следующему шагу. Так, например, она может перепрыгнуть какое-нибудь включение (списковое или словарное).
Самый главный прикол любого отладчика - точки останова. Такие тут тоже есть. Чтобы поставить точку нужно включить отладчик и написать:
`b <номер_строки>` - такая команда делает точку останова в конкретном месте. После установки точки можно прокрутить программу до нее с помощью команды `c`.
`l` - покажет все текущие точки останова + продемонстрирует кусочек кода.
