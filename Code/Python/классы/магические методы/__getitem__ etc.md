проблема конечно только в том, что это было очень поверхностное описание
да, базово метод помогает итерировать по собственным последовательностям:
```Python
class GetItem1:
    
    get_item_1 = list(range(1, 10))

    def __getitem__(self, index):
        return self.get_item_1[index]

gi_1 = GetItem1()
print(gi_1[1])  # => 2
```
но в нем наверняка есть еще куча всяких интересных вещей, потому что именно благодаря нему я так же могу реализовать срез собственной последовательности:
```Python
print(gi_1[1:6:2])  # => [2, 4, 6]
```
прежде чем я стану разбираться в срезах я хочу привести более конструктивный пример использования `__getitem__()`.
# этажи
представим, что у меня есть задача написать класс, который описывает офисное здание, где на каждом этаже находится офис какой-нибудь компании.
в самом простом случае, я напишу что-то такое:
```Python
class Office:
	def __init__(self, floors):
		'''Делаем словарь, где ключи - номера этажей, а значения None.'''
		self._floors = {floor: None for floor in range(floors)}

	def set_office(self, floor_number, company_name):
		'''Записываем в словарь, что на этаже находится `company_name` офис.'''
		self._floors[floor_number] = company_name

	def get_floor_data(self, floor_number):
		'''Возвращаем имя офиса по номеру этажа.'''
		return self._floors[floor_number]
```
вроде неплохо, да? а что будет, если воспользоваться таким классом?
```Python
o = Office(10)
o.set_office(0, "Вахтер")
o.set_office(1, "НРБ")
o.set_office(2, "Сбер")
print(o.get_floor_data(2))  # => Сбер
```
вся проблема такого подхода в том, что он не очень похож на парадигму языка Python, где установка элемента словаря происходит другим синтаксисом, как и получение элемента из словаря.
далее я хочу поменять подход и использовать методы `__setitem__()` и `__getitem__()`, чтобы согласовать мой класс с парадигмой языка:
```Python
class Office:
	def __init__(self, floors):
		'''Делаем словарь, где ключи - номера этажей, а значения None.'''
		self._floors = {floor: None for floor in range(floors)}

	def __setitem__(self, floor_number, company_name):
		'''Записываем в словарь, что на этаже находится `company_name` офис.'''
		self._floors[floor_number] = company_name

	def __getitem__(self, floor_number):
		'''Возвращаем имя офиса по номеру этажа.'''
		return self._floors[floor_number]
```
как видно, кроме названия имен методов я не поменял ничего, но т.к. это магические методы, они позволят мен делать вот так:
```Python
o = Office(10)
o[0] = "Вахтер"
o[1] = "НРБ"
o[2] = "Сбер"
print(o[2])  # => Сбер
```
гораздо лучше, правда?
# defaultdict `__missing__`
это супер-интересная фича.
реализация этого метода позволяет прописать поведение на случай, если пользователь попытался найти в моей последовательности то, чего в ней нет.
однако работает это только если мы реализуем `defaultdict`.
кстати, что такое `defaultdict`?
`defaultdict` - это структура данных, которая позволяет автоматически присваивать значение по умолчанию тем элементам, у которых нет значения в словаре или тем элементам, которые пользователь пытается найти в словаре и которых там не существует.
```Python
from collections import defaultdict

d = defaultdict(list)
d[1] = "Orange"
d[2] = "Banana"
print(d[1], d[2], d[3])  # => Orange Banana []
```
как видно, при инициализации словаря можно указать тип данных, который будет использовать структура при попытке пользователя обратиться к несуществующему в ней элементу. именно поэтому при вызове `print()` в конце печатаются пустые скобки списка.
`defaultdict` может делать такой фокус благодаря магическому методу `__missing__(value)`:
```Python
class MyDefaultDict(defaultdict):
    
    def __missing__(self, value):
        return "Дурак!"

d = MyDefaultDict()
d[1] = "Orange"
d[2] = "Banana"
print(d[1], d[2], d[3])  # => Orange Banana Дурак!
```
