## база
### списковое включение
Списковое включение - это способ оперативно написать однострочный цикл `for`, который в результате своей работы вернет список.
```Python
x = 'ABC'
codes = [ord(x) for x in x]
x  # 'ABC'
codes  # [65, 66, 67]
```
На английском списковое включение произносится "list comprehension", поэтому его иногда сокращают до "listcomp".
Ходят слухи что списковые включения и вообще включения не стоит использовать в реальном коде на Python, потому что они визуально засоряют код + ухудшают его читаемость. Однако существует несколько аргументов против такой позиции:
1) Списковое включение работает быстрее.
2) Списковое включение всегда в результате своей работы возвращает список, а цикл `for` используется как многофункциональный инструмент.
3) Списковое включение занимает одну строку (или две, но не более).
Лучше всего пользоваться списковым включением только в тех случаях, когда нам действительно необходимо использовать список, полученный в результате работы listcomp.
### генераторное включение
Генераторное включение - это способ быстрого создания объекта-генератора. Механизм создания идентичен созданию спискового включения, только нужно сменить скобки:
```Python
x = 'ABC'
codes = (ord(x) for x in x)
codes  # <generator object <genexpr> at 0x742b768a9be0>
list(codes)  # [65, 66, 67]
```
На английском генераторное включение произносится "generator expression", поэтому его иногда сокращают до "genexp".
## область видимости
У списковых включений (+ генераторных, словарных и включений множеств) есть собственная локальная область видимости. Она "скрывает" переменные в части `for`:
```Python
codes = [ord(c) for c in x]
c
Traceback (most recent call last):
  File "<python-input-12>", line 1, in <module>
    c
NameError: name 'c' is not defined
```
Я не могу получить доступ к переменной `c`, потому что она существует только в рамках этого спискового включения. В это поведение можно вмешаться с помощью оператора `:=`.
```Python
codes = [last := ord(c) for c in x]
last
67
```
## списковое включение vs map + filter
Списковое включение строит список из последовательности или любого другого итерируемого типа путем фильтрации и трансформации элементов. То же самое можно было бы сделать с помощью функций `map` и `filter`.
Когда я писал код раньше я считал, что любая реализация с помощью встроенных функций будет быстрее, чем аналогичное выражение на языке Python. Поэтому всегда нужно искать способ реализовать код с помощью встроенных функций, например, так:
```Python
x = 'ABC'
codes = list(map(ord, x))
```
Однако списковые включения показывают, что реализация через встроенные функции не всегда может быть правильной с точки зрения времени:
```Python
from time import time

x = 'ABC'

start = time()
codes = [ord(x) for x in x]
end = time()
print(end - start)

start = time()
codes = list(map(ord, x))
end = time()
print(end - start)
```
Результат по времени не сильно различается, но в этом случае - списковое включение работает быстрее:
```Python
1.1920928955078125e-06  # списковое включение
2.384185791015625e-06  # реализация через map
```
## декартово произведение
Списковое включение может быть двойным:
```Python
>>> x = [f'{x} * {y} = {x * y}' for y in range(1, 10) for x in range(1, 10)]
>>> print(*x, sep='\n')
```
Так как лучше читать списковое включение справа на лево, то для этого конкретного спискового включения фраза была бы такой: "для каждого x в диапазоне от 1 до 10 нужно выполнить цикл, где для каждого y в диапазоне от 1 до 10 пишется строка `f'{x} * {y} = {x * y}'`". В коде эту конструкцию так же можно было бы отобразить так:
```Python
result = []
for x in range(1, 10):
	for y in range(1, 10):
		result.append(f'{x} * {y} = {x * y}')	
print(*x, sep='\n')
```
Любой из таких вариантов записи по сути является декартовым произведением. Декартово произведение - это когда мы умножаем каждый элемент последовательности_1 на каждый элемент последовательности_2.
Например, я хочу, чтобы для каждого варианта цвета футболки был так же указан размер. У меня есть последовательность_1 - футболки и последовательность_2 - размеры. Для цвета `red` футболки должны быть указаны все три размера и для цвета `white` футболки так же должны быть указаны все три размера. Для быстрого решения такой задачи можно выполнить декартово произведение внутри спискового включения:
```Python
>>> colors = ['red', 'white']
>>> sizes = ['S', 'M', 'L']
>>> tshirts = [(color, size) for size in sizes for color in colors]
>>> tshirts
[('red', 'S'), ('white', 'S'), ('red', 'M'), ('white', 'M'), ('red', 'L'), ('white', 'L')]
```
