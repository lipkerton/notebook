# static
Flask - это фреймворк, который наследуется от Bottle. 
Основное различие между Bottle и Flask в способности Flask поддерживать историю с шаблонами (templates).
Чтобы создать базовый скрипт на Flask нужно:
```Python
from flask import Flask


app = Flask(
	__name__, static_folder='./static', static_url_path='/static'
)
```
`static_folder` - это папка, которая указывает скрипту куда нужно обращаться за статикой. Если аргумент не указывать, то за статикой скрипт будет пытаться обращаться к папке `static_folder`.
`static_url_path` - а это указатель для статики куда нужно обращаться за статикой. Если не указывать, то аргумент будет ссылаться на папку, которая указана для аргумента `static_folder`. Например, у нас есть index.html:
```HTML
<center>
	<h1>THIS IS MY PAGE</h1>
</center>
```
И мы хотим в этом HTML прописать еще и картинку. 
Чтобы HTML-документ понял откуда ему нужно взять картинку нужно указать путь в `static_url_path`. Когда путь указан мы можем внутри `index.html` прописать наименование файла, который мы хотим взять по этому пути:
```HTML
<center>
	<h1>THIS IS MY PAGE</h1>
	<img src='/static/computer_icon.jpeg' alt='Sorry, image is gone'>
</center>
```
Здесь я добавил в HTML-файл тег `img` - он показывает картинку. У этого тега есть атрибуты - `src` и `alt`. `src` - это ссылка на картинку от корня, т.е. от главной папки проекта, где лежит сейчас наш скрипт на flask. `alt` - это фраза, которая будет продемонстрирована на странице, если картинка не будет найдена.
```Python
from flask import Flask


app = Flask(
	__name__, static_folder='./static', static_url_path='/static'
)


@app.route('/')
def homepage():
	return app.send_static_file('index.html')


app.run(port=6789, debug=True)
```
`@app.route()` - это декоратор. Он отвечает за маршрутизацию запросов. Например, в данном случае, если пользователь пошлет запрос на адрес `http://localhost:6789/`, то за ответ на его запрос будет отвечать функция `homepage()`, потому что именно над ней висит декоратор `@app.route('/')`, который будет все запросы такого характера направлять именно в эту функцию.
`app.send_static_file()` - это функция, которая возьмет из папки `./static` файл `index.html` и отправит его в качестве ответа на запрос пользователя.
`app.run()` - это функция, которая запустит наш локальный сервер на порте 6789 + активирует "режим разработки" a.k.a. "режим дебага". "Режим дебага" означает, что при любой ошибке доступа к странице вместо страницы будет показана сводка - где, как, когда и почему произошла ошибка.
# templates
В чем отличие Flask от Bottle? Он поддерживает более широкую систему шаблонов, которая называется jinja2. Чтобы ее использовать нужно немного переписать программу.

>[!important] Шаблон в контексте веб-программирования - это документ с наиболее распространенной структурой сайта в формате HTML, в который можно вставить требуемые штуки. Это нужно, чтобы не тратить время на заполнение самых основных элементов, которые есть на каждой веб-странице.

Для начала вместо папки `static` нужно создать папку `templates` и положить туда файл `index.html`. Теперь мы не будем указывать путь до статики - Flask должен подхватить ее сам из папки `templates`.
Потребуется достать еще одну штуку из flask - функцию `render_template()`:
```Python
from flask import Flask, render_template


app = Flask(__name__)


@app.route('/')
def homepage():
	return render_template('index.html')


app.run(port=6789, debug=True)
```
Почти ничего не изменилось, однако сразу стало видно, что в этот раз мы не передаем старых аргументов в класс `Flask`. Это так, потому что функция `render_template()` будет искать статические файлы в папке `templates`. Это поведение можно изменить, если явно в аргументах класса `Flask` прописать путь до папки с шаблонами:
```Python
app = Flask(__name__, template_folder='./templates')
```
Статика будет искаться как обычно в папке `static_folder`, но и это поведение можно изменить при желании:
```Python
app = Flask(__name__, static_folder='./static')
```
Картинки у меня изначально лежали в папке `static`, поэтому итоговая программа будет такой:
```Python
from flask import Flask, render_template


app = Flask(__name__, static_folder='./static')


@app.route('/')
def homepage():
	return render_tempalte('index.html')
```
# а что с аргументами?
если мы хотим передать аргументы в нашу функцию от пользователя, чтобы эта функция их как-то обработала, то можно воспользоваться двумя путями:
- аргументы в URL:
```
http://www.example.com/book/cool
```
- аргументы в URL, как параметры GET: 
```
http://www.example.com/book?name=cool
```
У каждого из способов есть свои достоинства и недостатки.
## аргументы в url
```Python
@app.route('/echo/<arg>')
def echo(arg):
	return f'Привет, {arg}'
```
В результате все запросы, которые будут отправлены по адресу `http://localhost:6789/echo/<что-нибудь>` будут попадать именно в эту функцию, а все, что находится после `echo/` попадет в параметр `arg`.
В случае с HTML-шаблонами мы можем придумать второй шаблон, например, такой с названием `prikol.html` - он тоже будет лежать в папке `templates`:
```HTML
<center>
	<h1>Меня зовут: {{ arg }}</h1>
	<img src=/static/comp_icon.jpeg alt='picture'>
</center>
```
Этот шаблон ждет аргумент на вход:
```Python
@app.route('/echo/<arg>')
def echo(arg):
	return render_template('prikol.html', arg=arg)
```
Такой подход нормальный, но он становится немного неудобным для меня по двум причинам:
1) Вообще все, что будет написано после `echo/` попадет в параметр `arg`.
2) Если мы захотим передать два аргумента, то придется извращаться, например, прописывая такую логику: `@app.route('/echo/<arg_1>/<arg_2>')`. А что если аргументов будет больше?
## аргументы в get
аргументы в GET передаются через структуру `ключ=значение`, если аргументов несколько, то они разделяются символом амперсанда `&`. А от тела URL они отделяются знаком вопроса. Получается вот такая конструкция:
```
http://www.example.com/echo?name=Peter&place=London
```
Чтобы их достать из запроса понадобится чуть больше усилий - нужно импортировать из Flask `request` - это объект класса `Request`.
```Python
@app.route('/echo/')
def echo():
	arg = request.args.get('arg')
	return render_template('prikol.html', arg=arg)
```
По сути класс `Request` дает нам доступ ко всей входящей информации с запросами клиентов.
