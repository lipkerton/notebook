Каждый путь должен быть связан с какой-то функцией, которая будет формировать ответ, когда пользователь пошлет запрос на этот адрес.
Архитектуру путей можно настраивать по-разному, но в общем случае она выглядит так - от `urls.py` в главной папке проекта идут ссылки на `urls.py` в приложениях проекта. 
`urlpatterns` в главном `urls.py` проекта, при наличии двух приложений - `catalog` и `homepage`, будут выглядеть так:
```Python
urlpatterns = [
	path('admin/', django.site.urls),
	path('', include('homepage.urls')),
	path('catalog/', include('catalog.urls')),
]
```
## первый response
Когда пользователь обращается на адрес, то django формирует экземпляр класса `HttpRequest`. В этом экземпляре лежит вся информация о запросе, который сделал пользователь. Например, в нем лежит относительный путь запроса: если запрос был сделан на адрес `http://localhost:8000/catalog/`, то в атрибуте `path` объекта `HttpRequest` будет лежать путь `catalog/`.
В ответ на запрос пользователя должен быть отправлен экземпляр класса `HttpResponse`. Минимально такой экземпляр принимает на вход обычную строку: `HttpResponse('Привет!')`.
То есть у нас должна быть функция, которая принимает в качестве аргумента `HttpRequest`, а на выходе отдает `HttpResponse`. Я хочу прописать такую функцию для первой страницы сервиса - для `homepage`; это я буду делать в `views.py` приложения `homepage`:
```Python
from django.http import HttpResponse


def index(request):  # в request придет HttpRequest.
	return HttpResponse('Привет!')
```
Готово. Но пока что путь никак не привязан к этой функции. Чтобы его привязать нужно пойти в `urls.py` приложения `homepage` и там сделать так:
```Python
from django.urls import path

from . import views  # импортируем функции из views.py

urlpatterns = [
	path('', views.index)
]
```
## аргументы
Я хочу передать аргумент внутрь одной из функций, которая будет возвращать `HttpResponse`. Как это сделать? 
### аргументы в url
Я могу предать аргумент прямо в запросе. Допустим пользователь хочет попасть в конкретный товар на странице `catalog/`. Я помечаю все товары цифрами. Соответственно, чтобы получить товар нужно перейти по ссылке `catalog/<номер_товара>`.
Я перейду в `catalog/urls.py`, чтобы настроить маршрут:
```Python
from django.urls import path


urlpatterns = [
	# здесь стоят пустые кавычки, потому что в главном urls.py
	# у нас уже есть наводка на главную страницу каталога
	# по маршруту `catalog/`.
	path('', catalog.main_page),
	path('<int:item_number>/', catalog.item_page)
]
```
`int` - автоматически проверяет, что в строку было введено число. Это "конвертер пути". Он поймет, что кроме числа в эту функцию ничего попадать не должно. `item_number` - это имя переменной, под которым в функцию `item_page` будет передан номер товара.
### параметры запроса
Аргументы так же можно передать в параметрах запроса, например, так:
```
http://127.0.0.1/test/test_3/?name_1=Морковь&name_2=Картошка&name_3=Орехи
```
Если есть параметры запроса, но не получится их "вытащить" через функцию `path` - их нужно вытаскивать через view-функцию, а точнее через параметры `request`:
```Python
name_1 = request.GET.get("name_1")
```
Так как внутри константы `request.GET` храниться обычный словарь, мне кажется, что его значения можно вытащить с помощью метода `values()`.
```Python
context = {"items": request.GET.values()}
```
Нужно держать в голове, что функция `values()` вернет объект-генератор, который еще нужно перебрать внутри шаблона с помощью управляющей конструкции `for`.
