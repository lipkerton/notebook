## где лежат пути и как их писать
все пути хранятся в папках приложений в файле под именем `urls.py`.
В главной папке самого проекта (в той, что с файлом `settings.py`) тоже есть такой файлик и он уже содержит один путь.
Этот путь ведет на стандартную админку Django. Сам файл выглядит так:
```Python
from django.contrib import admin
from django.urls import path

urlpatterns = [
    path('admin/', admin.site.urls),
]
```
Это относительный путь. Строится он относительно домена. Это означает, что по адресу `http://127.0.0.1:8000/admin/` можно найти админку.
Функция `path()` отвечает за связь пути и функции, которая будет обрабатывать ответ. В таком же формате можно добавить еще несколько путей.
```Python
urlpatterns = [
	path('', <функция_которая_отвечает_за_главную_страницу>),
	path('catalog/', <функция_которая_отвечает_за_каталог>),
	path('admin/', admin.site.urls)
]
```
Важный момент состоит в том, что Django читает этот список сверху вниз при получении запроса, поэтому он отправит запрос на обработку в первое же попавшееся совпадение.
## аргументы
Если пользователь вводит адрес вида `https://www.example.com/catalog/1/`, то он, очевидно, хочет получить страницу первого товара в каталоге товаров. Тогда у нас должна быть функция, которая примет эту единицу, найдет только те данные, которые принадлежат этому товару, и отдаст их в ответе на запрос.
Эта функция выглядит, например, так:
```Python
def get_item(request, pk):
	pass
```
Вопрос состоит в следующем: "А как нам получить этот самый аргумент pk?" Ну можно прописать специальную функцию для конкретного относительного адреса `/catalog/1/`, но это плохой вариант, потому что если товаров в каталоге будет не десять, а сто, то придется прописать для каждого такой вариант относительного пути.
Вариант получше - использовать переменные пути:
```Python
path('catalog/<pk>/', views.get_item)
```
Все просто - в шаблон пути записываем `<имя_переменной>` и эта переменная будет передана обязательным аргументом в нужную функцию.
Однако есть проблема - она связана с тем, что любая запись после имени `catalog/` и до сепаратора `/` будет считаться аргументом. Например, `1`, `2`, `100` будут считаться аргументами (нужное нам поведение), но еще аргументами будут считаться другие комбинации символов - `hello`, `cool`, `привет!` (ошибочное поведение в данной ситуации). Все потому, что каждый такой аргумент в пути преобразовывается в строку автоматически (т.к. путь, который нам приходит в запросе от клиента по сути является строкой).
