Основная архитектура для работы в сети - клиент-серверная. Клиентом может выступать браузер и любая другая программа, которая отправляет запросы на сервер. История этой главы о том как реализуются веб-клиенты и веб-серверы на Python.
## веб-клиенты
### стандартные библиотеки
В питоне третьей версии есть два встроенных пакета, которые работают с http и клиент-серверной архитектурой:
- urllib - работает на основе HTTP и способен строить скрипты, как для серверов, так и для клиентов.
- http - не до конца изучал чем конкретно этот пакет управляет.
Чтобы написать клиент можно воспользоваться скриптом на питоне, который будет отправлять запрос к ресурсу и получать ответ:
```Python
import urllib.request


resp = urllib.request.urlopen('http://www.example.com/')
print(resp)
print(resp.status)
```
Переменная `resp` вернет объект `HTTPResponse` и мы можем обратиться к этому объекту, чтобы получить, например, статус-код. Фишка в том, что как раз в этом объекте будет содержаться вся информация от нашего запроса к тестовому сайту. Эту информацию мы можем получить с помощью специальных методов и атрибутов объекта. А еще мы можем получить сам HTML документ, который нам прислали вместе с ответом на запрос; только этот документ придется сначала расшифровать, потому что он возвращается в типе данных `bytes`.
```Python
import urllib.request


resp = urllib.request.urlopen('http://www.example.com/')
print(resp.getheaders())

data = resp.read()
data = data.decode('utf-8'))
print(data[:50])
```
`data[:50]` возвращает первые 50 символов из HTML документа.
### сторонние пакеты
Для всех вопросов, которые возникают при решении веб-клиентских проблем, можно использовать стороннюю библиотеку requests. Но сначала ее нужно установить.
```Python
import requests


resp = requests.get('http://www.example.com/')
print(resp)
print(resp.status_code)
print(resp.text[:50])
```
