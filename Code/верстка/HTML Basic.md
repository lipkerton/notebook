HTML = Hyper Text Markup Language. Это буквально язык разметки веб-документа. Смысл заключается в том, что мы размещаем элементы на странице с помощью специальных тегов. Теги определяют чем является элемент и каким образом он должен отображаться. Теги могут вкладываться друг в друга - чтобы отделить уровни вложенности используется два пробела (по стандарту).
## работа с текстом
### заголовки
В HTML существует шесть тегов для обозначения заголовков:
```HTML
<h1>Это самый большой!</h1>
<h2>Это поменьше!</h2>
<h3>Это еще поменьше!</h3>
<h4>У меня кончились слова!</h4>
<h5>Почти все!</h5>
<h6>Самый маленький!</h6>
```
Я сохраню этот текст в `index.html` и запущу тест-сервер:
```Bash
python3.13 -m http.server 8000
```
Теперь, чтобы увидеть то, что получилось нужно перейти по адресу:
```
http://localhost:8000/
```
Получиться что-то такое:
![[FF02277A-A297-4F1B-B42B-60F8A78A5C47.jpeg]]
Почему так? А потому что мы не указали кодировку!
Нужно прописать:
```HTML
<meta charset="utf-8">
```
Теперь все будет работать как нужно:
![[E48D94BC-E280-476E-9AB9-BA6D5DD0629D.jpeg]]
### абзацы
Они делаются с помощью тега `<p>`:
```HTML
<p>Здесь абзац!</p>
<p>
  <h1>А это абзац с заголовком внутри</h1>
</p>
```
Получится что-то такое:
![[B2A3F0C7-82E4-459C-8049-2CE1B29E118D.jpeg]]
### разрыв строки
Разрыв строки делается с помощью одиночного тега `<br>`. Одиночный он потому, что у него нет закрывающего тега - разрыв строки будет выполнен там, где установлен тег `<br>`:
```HTML
Это строка<br>с двумя ра<br>зрывами!
```
Получится так:
![[668E7E61-FA1A-4168-84C3-F95116DEB5FA_4_5005_c.jpeg]]
### шрифты
Кроме обычного шрифта есть еще `bold` (жирный шрифт) и `italic` (курсив). Они обозначаются по первой букве:
```HTML
Это <b>жирный</b> текст! А это <i>курсив</i>!
```
### списки
Списки бывают двух типов - маркированные (с точками) и нумерованные (с цифрами). 
#### нумерованный список
Начало списка обозначает открывающий тег `<ol>`, а конец списка закрывающий тег `</ol>`. Пункты списка задаются парными тегами `<li>`. Вот так это выглядит на примере:
```HTML
<ol>
  <li>Это первый пункт</li>
  <li>Это второй пункт</li>
  <li>Это третий пункт</li>
</ol>  
```
Итог будет таким:
![[A7242DE2-9C70-40F2-87CE-1A9852CFDE31_4_5005_c.jpeg]]
Кстати, `<ol>` - это сокращение от "ordered list".
#### маркированный список
Начало списка задает открывающий тег `<ul>`, а конец списка закрывающий тег `</ul>`. Пункты списка все так же задаются парными тегами `<li>`:
```HTML
<ul>
  <li>Это первый пункт</li>
  <li>Это второй пункт</li>
  <li>Это третий пункт</li>
</ul>
```
Итог будет таким:
![[741E5F7A-F9E4-4C79-93B7-7988C8B9D976_4_5005_c 1.jpeg]]
Кстати, `<ul>` - это сокращение от "unordered list".

## работа с особыми штуками
Если бы HTML был просто языком, с помощью которого можно написать красивый текстовый файл, то было бы непонятно почему он является таким особенным, по сравнению с обычным способом редактирования текстовых файлов.
В названии HTML есть слово "Hypertext". Это слово означает, что документ ссылается еще и на другие документы такого же формата. То есть он содержит в себе какие-то элементы, которые позволяют перейти на другой документ.
Простым языком это означает, что HTML документ содержит не только текст, а еще и ссылки на другие страницы. Это могут быть картинки, другие HTML документы, видеоролики и проч. Нужно научиться вставлять такие особые вещи внутрь HTML документа.
### ссылка
Ссылка обозначается парным тегом `<a></a>`. Внутри тега прописывается текст, который будет отображаться на странице вместо самой ссылке. Этот текст по умолчанию будет подчеркнут и будет светиться синим. По тексту можно будет кликнуть, чтобы перейти на какой-то другой ресурс.
Здесь возникнет логичный вопрос: "А в какое место тега тогда вставляется сама ссылка?". Ссылка вставляется в атрибут тега.
Атрибут тега - это параметр, который указывается внутри открываюшего тега:
```HTML
<a href="https://docs.python.org/3/library/collections.abc.html#collections.abc.Sequence">Коллекции в Python</a>
```
`href` - это имя атрибута. За именем ставится знак равно и пишется значение атрибута. Дальше атрибут закрывается скобкой и пишется текст тега. Потом тег закрывается.
Получится так:
![[0CD9FA60-DEE0-43DB-862E-514C909CC163_4_5005_c.jpeg]]
Кстати, почему ссылка напечаталась в одну строку с предыдущим текстом? А все потому, что я не прописал абзац или разрыв строки.
Еще один момент - по умолчанию переход по ссылке открывает страницу (которая доступна по ссылке) в текущей вкладке, подменяя собой старую. В нашем случае это означает, что страница `index.html` заменится другой страницей в той же вкладке. Это поведение можно изменить - например, можно заставить страницу по ссылке открыться в соседней вкладке. 
Чтобы это сделать нужно применить необязательный параметр для тега `<a></a>` - `target`:
```HTML
<a href="https://docs.python.org/3/library/collections.abc.html#collections.abc.Sequence" target="_blank">Новая страница</a>
```
Если очень хочется, то поведение по умолчанию можно прописать очевидным образом:
```HTML
<a href="https://docs.python.org/3/library/collections.abc.html#collections.abc.Sequence" target="_self">Старая страница</a>
```
Но это поведение никак не отличается от поведения без параметра `target`.
### картинка
Картинка в тело HTML вставляется с помощью одиночного тега `<img>`. Путь до картинки указывается так же как и для ссылки - в атрибуте, но атрибут называется по-другому - `src`. 
Картинка может быть взята по ссылке, тогда в качестве пути будет использована буквально ссылка на картинку в интернете. Альтернативный подход - сохранить картинку на локальный компьютер и указать путь до нее на компьютере так же в атрибуте `src`.
Я сделал работу вторым способом - в директории рядом с файлом `index.html` разместил папку `media` и закинул в нее картинку, а путь указал относительно корня сервера.
Допустим сервер запускается из папки `my_page`, а в ней лежит `index.html` и `media`, тогда путь до картинки строится относительно места, из которого был запущен сервер - `media/<картинка>`.
```HTML
<img src="media/CatPicture.jpg">
```
Все! Картинка есть.
Тег является одиночным - закрывать его не нужно.
