Bootstrap - это фреймворк, который предоставляет как готовые стили сайтов (платно), так и готовые страницы стилей CSS и скриптов JS (бесплатно). Готовые страницы можно скачать и положить в собственную папку стилей проекта, а можно просто оставить ссылку на страницы, которые находятся на сервере Bootstrap.
## подключение
Оба варианта установки лежат на [официальном сайте](https://getbootstrap.com/docs/5.3/getting-started/download/) . Я лично предпочитаю вариант с локальной установкой компонентов фреймворка - для этого нужно просто скачать архив файлов CSS + JS.
Меня интересуют стили так что скрипты JS я пока трогать не буду. Внутри папки стилей лежит очень много файлов и, базово, они различаются способом оформления элементов на странице. Я возьму дефолтный - `bootstrap.css` и переложу его в папку `/css/bootstrap.css` внутри своего проекта. Теперь у меня такая структура:
```Bash
├── css
│   ├── bootstrap.css
│   └── style.css  # мой старый файл стилей
├── index.html
└── testing_ground.html  # мой старый index.html
```
Теперь осталось только подключить файл в `index.html`:
```HTML
<head>
  <link rel="stylesheet" href="css/bootstrap.css">
</head>
```
## работа со стилями
Чтобы понимать смысл установки `bootstrap.css` лучше просто в него заглянуть. Там просто описаны стили для классов. Все, что нужно сделать, чтобы применить Bootstrap: 1) взять в руку документацию по стилям, 2) раскидать классы по элементам своего HTML так, как нужно.
Для этого я сделал такой док:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <!-- подключаем стили bootstrap -->
    <link rel="stylesheet" href="css/bootstrap.css">
  </head>
  <body>
    <header></header>
    <main>
      <!-- первая карточка -->
      <div>
        <h5>Петр Петухов</h5>
		<div>
	      <h6>16 января 2025 года</h6>
	      <p>
	        Сегодня крутой день - солнышко светит, птицы поют!
	        Сочи - самый крутой город в России!
		  </p>
		  <a href="">Читать целиком</a>
        </div>
      </div>
    </main>
    <footer></footer>
  </body>
</html>
```
Допустим я хочу создать страницу, на которой будет список постов. Эти посты представлены в виде карточек - можно развернуть карточку и перейти на отдельную страницу. Каждая карточка подписана именем и датой, а внутри нее есть содержание и ссылка на полный текст.
Если не применять стили, то изначально в браузере это будет выглядеть так:
![[99FFF0FC-5D31-4667-B96D-2F1D5D16E2E7_4_5005_c.jpeg]]
Не очень презентабельно, особенно если учесть, что весь этот текст находится в крайнем левом углу экрана без каких-либо отступов.
Для создания карточки я хочу поместить текст в рамку. Что говорит документация? Документация говорит, что для начала нужно задать тегу `<div>` класс  `card`:
```HTML
<div class="card">
  <h5>Петр Петухов</h5>
  <div>
    <h6>16 января 2025 года</h6>
    <p>
	  Сегодня крутой день - солнышко светит, птицы поют!
	  Сочи - самый крутой город в России!
    </p>
    <a href="">Читать целиком</a>
  </div>
</div>
```
В сущности от этого практически ничего на странице не измениться - добавится полоска под записью, которая будет обозначать границу карточки. Попробую еще что-то:
```HTML
<div class="card">
  <h5 class="card-header">Петр Петухов</h5>
  <div class="card-body">
    <h6 class="card-title">16 января 2025 года</h6>
    <p class="card-text">
	  Сегодня крутой день - солнышко светит, птицы поют!
	  Сочи - самый крутой город в России!
    </p>
    <a href="" class="btn btn-primary">Читать целиком</a>
  </div>
</div>
```
Здесь сразу произошло много всего. Для начала я покажу результат:
![[C5399EAE-1392-465B-A264-E46919DC8146.jpeg]]
Имя выделилось в заголовок благодаря классу `card-header`. Сама карточка приобрела "тело" - тег `card-body`. Внутри `card-body` так же есть заголовок `card-title`, а еще есть текст карточки в `card-text`. Одна из крутых штук - кнопка: я запаковал ссылку в формат кнопки с помощью тега `btn btn-primary`.
Стили очень помогли, однако карточка все равно выглядит плохо. Главная проблема - размер карты (она растянута на всю страницу). Нужно настроить и его. Можно воспользоваться стандартным методом CSS - `margin`. Пример:
```HTML
<div style="margin: 5%">
  <div class="card">
    <h5 class="card-header">Петр Петухов</h5>
    <div class="card-body">
	  <h6 class="card-title">16 января 2025 года</h6>
	  <p class="card-text">
	    Сегодня крутой день - солнышко светит, птицы поют!
        Сочи - самый крутой город в России!
	  </p>
	  <a href="" class="btn btn-primary">Читать целиком</a>
    </div>
  </div>
</div>
```
Я добавил еще один тег `<div>` и прописал для него индивидуальный стиль - `margin: 5%`. Это означает, что я хочу, чтобы отступ этого объекта от всех других на странице был равен 5%. В итоге получится так:

Уже гораздо лучше.
## разметка страницы
Но вообще у Bootstrap есть куча настроек, которые связаны с упорядочиванием элементов на странице. Например, на странице часто можно увидеть не один сплошной текст на всю ширину, а несколько разных колонок текста - боковые и центральные колонки. Такой стиль и еще несколько других, которые были бы похожи можно так же настроить с помощью стилей Bootstrap.
Например, я хочу разместить два элемента на странице в строку:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <div>
      <div style="background: red;">Первый блок div</div>
      <div style="background: green;">Второй блок div</div>
    </div>
  </body>
</html>
```
Однако, если я напишу пример выше и запущу, то у меня на странице это отобразится примерно так:
![[Снимок экрана 2025-06-01 в 08.22.29.png]]
При этом сами элементы будут растянуты на весь экран - у меня еще куча свободного места. Я хочу поместить их в строку, поэтому использую клаcc `row` из Bootstrap + я буду использовать классы `col-3` и `col-9` для внутренних тегов `<div>`:
```HTML
<div class="row">
  <div class="col-3" style="background: red;">Первый блок div</div>
  <div class="col-9" style="background: green;">Второй блок div</div>
</div>
```
Получится примерно следующее:
![[E7513A44-CFA9-4339-81BF-BF560439096F_4_5005_c.jpeg]]
У меня здесь не видно, но на самом деле второй блок растянут буквально на оставшуюся часть страницы - первый блок занимает меньше места из-за параметров `col-3`.
Что вообще обозначает `col-3`? Эта цифра означает, что элемент будет занимать 3/12 от всей страницы. Bootstrap делит страницу на  12 частей, поэтому можно распределить элементы по горизонтали, руководствуясь этими долями.
Если распределить заголовки по строке так, что в сумме их колонки занимают больше 12 частей страницы, то колонка, которая не помещается будет перенесена на следующую строку.
```HTML
<div class="row">
  <div class="col-2" style="background: red;">
    Первый блок
  </div>
  <div class="col-9" style="background: green;">
    Второй блок
  </div>
  <div class="col-3" style="background: blue;">
    Третий блок
  </div>
</div>
```
Получится так:

## адаптивность под разные экраны
Bootstrap дает возможность настроить один и тот же интерфейс для разных форматов экранов. Осталось только настроить эту историю. C настройкой поможет официальный [док](https://getbootstrap.com/docs/5.0/layout/grid/#grid-options), где указаны классы, которые можно присвоить тегам, чтобы их отображение динамически подстраивалось под нужный экран.
Есть шесть разных классов экранов от самого маленького к самому большому. Мы можем настроить свои элементы HTML так, чтобы нужный формат был выбран исходя из текущих настроек окна клиента. Чтобы сделать так, нужно сначала определить в теге `<meta>` параметр `viewport` - он отвечает за настройку `viewport` браузера, которая описывает зону окна, где виден контент.
```HTML
<meta name="viewport" content="width=device-width, height=device-height">
```
Я выставил два параметра - ширину и длину. Оба параметра приравнены к значению текущего размера окна. Когда размер окна поменяется - они так же будут изменены.
Теперь нужно задать поведение элементам - они должны реагировать на различное поведение окна.
```HTML 
<div class="row">
  <div class="col-md-12 col-3">Первый div</div>
  <div class="col-md-12 col-3">Второй div</div>
</div>
```
За поведение отвечают классы `col-md-12` и `col-3`. Они описывают как должны вести себя элементы при среднем (`md`) и самом маленьком размерах экрана. `12` указывает на то, что `div` должен быть растянут на все возможные 12 колонок на мониторе, а `3` указывает на то, что на самом маленьком мониторе элементы должны занимать не более 3/12 экрана.

## контейнеры
Контейнеры - это штуки, в которые оборачивают смысловые блоки на странице, чтобы они не растягивались на всю ширину экрана. Чтобы "положить" тег в контейнер, ему достаточно прописать класс `container`. Пример:
```HTML
<div class="container">
  <div class="row">
    <div class="col-md-12 col-3">Первый div</div>
    <div class="col-md-12 col-3">Второй div</div>
  </div>
</div>
```
