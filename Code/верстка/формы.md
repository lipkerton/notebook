Формы - это способ разместить на HTML странице элементы, с которыми можно взаимодействовать. Например, это могут быть поля для заполнения или кнопки.
# простая форма
Чтобы сделать простейшую форму:
```HTML
<form>
  <label>Введите имя:</label>
  <input type="text" name="user_name" required>
  <input type="submit" value="Отправить">
</form>
```
При ее запуске на экране появится такой результат:
![[26131F6A-3AAD-49BC-ABB8-5E6EC882A316_4_5005_c.jpeg]]
При этом, если ввести имя и нажать кнопку, то поле отчиститься, а результат *как будто* будет отправлен. В отображении ссылки появится query-параметр:
![[form_1.png]]
## чекбокс
Для добавления чекбокса используется тот же тег, что и для добавления поля для ввода.
Подпись для чекбокса так же выполняется в теге `label`. Однако стоит понимать, что тег для подписи по сути может быть любым, например, `span`. Тем не менее, у `label` есть преимущество при использовании с чекбоксами - он позволяет сделать надпись, клик по которой так же будет активировать чекбокс.
```HTML
<label for="is_human">Я человек</label>
<input id="is_human" type="checkbox" name="human">
```
Получится такой результат:
![[form_2.png]]
При этом чекбокс будет активироваться даже от нажатия на надпись *"Я человек"*.
Если заполнить форму и отправить данные, то появится еще один новый параметр:
![[form_3.png]]
## разместить формы
Перемещать формы по странице можно с помощью стандартных методов HTML, например, с помощью абзацев:
```HTML
<form>
	<p>
		<label>
			Введите имя:
		</label>
		<input type="text" name="user_name" required>
	</p>
	<p>
		<label for="is_human">Я человек</label>
		<input id="is_human" type="checkbox" name="human">
	</p>
	<p>
		<input type="submit" value="Отправить">
	</p>
</form>
```
Результат будет таким:

# action method
Чтобы форма могла быть полезной для нее определяют два атрибута - `action` определяет куда информация из запроса будет отправлена, а `method` определяет метод, который будет использован для отправки данных.
По умолчанию данные отправляются на ту же страницу, а используемый метод - `get`. Как раз поэтому при нажатии кнопки отправить страница просто перезапускается, а в пути появляются query-параметры.
Я попробую указать другой адрес страницы, чтобы при отправке данных меня перекидывало именно на нее. Сначала нужно придумать вторую страницу:
```HTML
<!DOCTYPE html>
<html lang="ru">
    <head>
        <title>
            Sorry
        </title>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
    </head>
    <body>
        Sorry, website is currently unavailable!
    </body>
</html>
```
Теперь на кнопку отправить нужно добавить `action` с адресом новой страницы в системе. У меня они лежат в одной директории, поэтому достаточно указать имя:
```HTML
<form action="test_2.html">
	...
</form>
```
Теперь нажатие с любыми данными на кнопку "Отправить" будет перекидывать меня на то, что я написал в другом шаблоне.
## post method
Как я уже сказал, стандартный метод для формы - `get`. Именно поэтому все данные передаются в query-параметрах, а не в теле самого запроса.
Чтобы исправить ситуацию нужно обозначить, что используемым методом является `post`.
```HTML
<form action="test_2.html" method="post">
	...
</form>
```
Теперь при нажатии по кнопке "Отправить" меня снова отправит на `test_2.html`, но в этот раз в query-параметрах ничего не будет - все переданные данные будут находится в теле запроса:
![[form_5.png]]
В `Payload` видно два параметра - `user_name` и `human`. Это как раз те имена параметров, которые я указывал в соответствующих `input`:
```HTML
<input type="text" name="user_name">
<input type="checkbox" name="human">
```
