## база
Хотя HTML элементы можно писать просто так, без всякого порядка, руководствуясь только тем как они отображаются на экране в браузере, у HTML документов есть своя общепринятая структура.
Для начала следует объявить версию HTML, которая будет использоваться в документе. Текущая версия языка - HTML5 и чтобы ее объявить нужно в самом начале HTML документа написать:
```HTML
<!DOCTYPE html>
```
Хотя версия здесь не указана напрямую - браузер поймет, что использована последняя версия HTML, если не указано иное.
Далее, весь код страницы всегда оборачивается в парный тег `<html>`:
```HTML 
<!DOCTYPE html>
<html>
<!-->Вся разметка будет находиться здесь!</!-->
</html>
```
Для тега `<html>` существует дополнительный аргумент: `lang`. В него записывают язык, который используется внутри документа. Я использую русский, поэтому так и напишу:
```HTML
<!DOCTYPE html>
<html lang="ru">
</html>
```
Внутри самого тега `<html>` код делится как минимум на две части - `<head>` и `<body>`. Это тоже парные теги. Вся основная разметка страницы всегда пишется в `<body>`. А что пишется в `<head>`? Это я разберу попозже.
В итоге стартовая страница любого HTML дока всегда похожа на что-то такое:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
  </head>
  <body>
  </body>
</html>
```
## тег `<head>` и мета
Что пишется в теге `<head>`? В нем прописывают различные данные, которые не будут показаны пользователю. Зачем они нужны? Они определяют маленькие настройки для всей HTML страницы.
### `<meta>`
Когда я писал код без структуры  ([[HTML Basic]]), то упомянул тег `<meta charset="utf-8">`, который должен был бороться с неверным определением кодировки русского текста на странице. Хотя этот тег работает и в том формате, он не выдает никакой визуализации на страницу + он обеспечивает настройкой весь HTML документ - значит его можно поместить в `<head>`.
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
</html>
```
По такому же принципу в `<head>` принято помещать вообще все метаданные, которые могут быть в проекте. Например, туда же можно запихнуть краткое описание странички:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <meta name="description" content="Краткое описание страницы!">
  </head>
</html>
```
Туда же я могу поместить все ссылки на нужные файлы скриптов, которые отрабатывают на странице:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <meta name="description" content="Краткое описание страницы!">
    <script src="hello.js"></script> 
  </head>
</html>
```
### `<title>`
Еще сюда же я бы запихнул заголовок (он будет отображаться как имя вкладки). Заголовок пишется в теге `<title>`.
```HTML
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <meta name="description" content="Краткое описание страницы!">
    <script src="hello.js"></script> 
	<title>Заголовок</title>
  </head>
</html>
```
### `<link>`
Так же в заголовке могут появляться теги `<link>`, которые описывают связь между текущей страницей и каким-нибудь внешним ресурсом.
Они в общем случае выглядят так:
```HTML
<link rel="тип_связанного_ресурса" href="ссылка_на_ресурс">
```
Я хочу с помощью тега `<link>` добавить на страницу иконку в заголовок вкладки. Для этого я сначала заполучу иконку из интернета и положу новый файл в папку `media`. Тогда остается только написать в заголовке ссылку на эту иконку:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <meta name="description" content="Краткое описание страницы!">
    <script src="hello.js"></script>
    <link rel="icon" href="media/iconic_icon.ico" type="image">
    <title>Заголовок</title>
  </head>
</html>
```

>[!important] Иконка может не подгружаться из-за кэша. Нужно сбросить кэш перед тем как проверять.

Что такое атрибут `rel` для тега `<link>`? Он определяет отношения между ресурсом, который доступен по ссылке и текущим документом. Тогда что делает `type`? В этом атрибуте указывается тип контента, который привязывается к странице.
Важно всегда помнить про формат и размеры самой иконки, т.к. для разных клиентских приложений они могут быть разными. Для настройки размера у тега `<link>` есть атрибут `sizes`, в котором прописывают отношение сторон, например: `sizes="32x32"`.
## тег `<body>`
Тег `<body>` сам делиться на несколько смысловых блоков. Блоки именно "смысловые", потому что они нужны в HTML документе исключительно для удобства навигации по нему и потому что такова стандартная практика.
### колонтитулы
Во-первых, у тега `<body>` есть "верх" и "низ". По аналогии с word-документом - это колонтитулы страницы. Они обозначаются парными тегами `<header></header>` и `<footer></footer>`.
Тег `<header>` по смыслу нужен для репрезентации "верха" страницы - там может быть навигация сайта, телефоны, логотип.
Тег `<footer>` по смыслу нужен примерно для того же - там размещается навигация, контактная информация и проч.
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <!-- Новый тег -->
    <header>
      Верхний колонтитул
    </header>
    <!-- Новый тег -->
    <footer>
      Нижний колонтитул
    </footer>
  </body>
</html>
```
### навигация
Навигацию сайта обычно помещают в тег `<nav></nav>` и этот тег может быть в любой части сайта, потому что сайты, например, могут сделать навигацию и в `<header>` и в `<footer>` сайта.
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
	<meta charset="utf-8">  
  </head>
  <body>
    <header>
      Верхний колонтитул
      <!-- Новый тег -->
      <nav>Навигация</nav>
    </header>
    <footer>
      Нижний колонтитул
      <!-- Новый тег -->
	  <nav>Навигация</nav>
    </footer>
  </body>
</html>
```
### основная часть
Обозначается парным тегом `<main>` и содержит основную информацию на сайте. Может быть разделена на "подпункты" с помощью вспомогательных парных тегов `<section>`, `<article>`, `<div>`.
1) `<section>` - обозначает смысловую секцию.
2) `<article>` - смысловые блоки поменьше.
3) `<div>` - маленькие секции.
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <header>
      Верхний колонтитул
      <nav>Навигация</nav>
    </header>
    <!-- Новый тег -->
    <main>
      Основной текст страницы
      <section>
        <article>
		  <div></div>
        </article>
      </section>
    </main>
    <footer>
      Нижний колонтитул
      <nav>Навигация</nav>
    </footer>
  </body>
</html>
```
### дополнительно
Чтобы обозначить какую-то дополнительную информацию, которая слабо связана с основным сюжетом страницы можно использовать смысловой парный тег `<aside></aside>`. Его можно вставить в любую часть документа.
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <header>
      <nav></nav>
    </header>
    <main>
      <section>
        <article>
          <div></div>
        </article>
      </section>
    </main>
    <footer>
      <nav></nav>
      <!-- Новый тег -->
      <aside></aside>
    </footer>
  </body>
</html>
```
## DOM
DOM - Document Object Model (Объектная модель документа). Это структура, которую браузер создает в памяти, когда анализирует HTML документ. Если присмотреться к этой записи:
```HTML
<html>
  <head>
    <meta charset="utf-8">
  </head>
  <body>
  </body>
</html>
```
То ее можно интерпретировать так же как это делает команда `tree` с директориями в Linux:
```
document 
└── html  
    ├── head  
    │   └── meta
    └── body
```
В сущности это обычная древовидная структура.
Сообразно любой древовидной структуре - в HTML структуре есть дочерние и родительские элементы. У дочернего элемента может быть только один родитель, а у родителя может быть сколько угодно много дочерних элементов. 
>[!important] Каждый элемент в DOM-дереве называется "узел". Самый "верхний" узел называется "корневой узел".
