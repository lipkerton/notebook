## база
HTML описывает то как элементы выстраиваются в конструкцию на странице - он распределяет смысловые блоки, показывает где будут находится части страницы и проч.
Однако, чтобы украсить страницу нам нужно использовать язык стилей - CSS. CSS не указывает конструкцию страницы - он редактирует как будут выглядеть элементы внешне на странице: работает с цветами, отступами элементов друг от друга и проч.
Стили могут быть написаны как в самом HTML документе, так и в стороннем файле.
## пишем в HTML
### стиль для каждого тега
Я напишу стартовый код без стилей:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <header>
	  <nav>
	    <ul>
	      <li><a href="/">Главная страница</a></li>
	      <li><a href="about/">О сайте</a></li>
	    </ul>
	  </nav>
    </header>
  </body>
</html>
```
И он будет выглядеть как мусор:
![[S01_80_1677516145.png]]
Я привык, что на сайте меню навигации обычно рисуют сверху - элементы размещены в линию от одного поля экрана до другого. Чтобы реализовать такую механику можно использовать CSS тег `display`.
Но для начала - где вообще пишутся CSS теги? Они пишутся в теге `style` для какой-нибудь группы в формате `<имя тега>: <значение>;`. 
Например, в данном куске есть тег `<nav>`, который внутри себя содержит маркированный список ссылок. Как раз для него мы можем задать атрибут `style`. Допустим я хочу раскрасить содержимое этого тега в салатовый цвет:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <header>
      <nav style="background: #c9f600;">
        <ul>
          <li><a href="/">Главная страница</a></li>
          <li><a href="/about">О сайте</a></li>
        </ul>
      </nav>
    </header>
  </body>
</html>
```
Получится что-то такое:
![[8CDD69C3-13E9-4EB5-B9BF-6128A566B942_4_5005_c.jpeg]]
Это уже своего рода победа, однако все стало только  на чуть-чуть лучше. Здесь можно вернуться к тегу `display` - именно его мы разместим. Но где?
>[!important] Цвета в HTML и CSS задаются в формате RGB с помощью шестнадцатиричной записи.

Я попробую прописать `display` сразу после прошлого параметра цвета:
```HTML
<nav style="background: #c9f600; display: inline-block;"> 
```
`inline-block` означает, что элемент нужно отобразить в строку:
![[4F08E222-67B7-4270-B542-31A919618FCB_4_5005_c.jpeg]]
Получилось не совсем то, чего я ожидал. Здесь полезно задуматься о смысле команды `inline-block` - она размещает весь элемент в строку, т.е. в данном случае я попросил CSS разместить весь элемент `<nav>` в одну строку, вместе с его подэлементами.
Вместо этого нужно разместить подэлементы внутри `<nav>` в другом порядке:
```HTML
<nav style="background: #c9f600;">
  <ul>
    <li style="display: inline-block;">
      <a href="/">Главная</a>
    </li>
    <li style="display: inline-block;">
      <a href="/">О сайте</a>
    </li> 
  </ul>
</nav>
```
Вот теперь все выглядит так как было задумано:
![[F462D34D-5C91-4F00-85B3-D4E4152CE56B_4_5005_c.jpeg]]
Теперь я хочу изменить шрифт. Для начала увеличить его и затем изменить сам стиль. В первом случае я использую тег `font-size` - он требует число пикселей на вход, а во втором случае `font-family` - семейство шрифтов.
```HTML
<nav style="background: #c9f600">
  <ul>
    <li style="display: inline-block; font-size: 20px; font-family: sans-serif;">
      <a href="/">Главная</a>
    </li>
    <li style="display: inline-block; font-size: 20px; font-family: sans-serif;">
      <a href="/about">О сайте</a>
    </li>
  </ul>
</nav>
```
Теперь страница будет выглядеть так:

### cтили внутри head
Когда я писал про верстку HTML дока вообще я указал, что существуют смысловые блоки, которые никак не отражаются для пользователя на странице, но структурируют сам документ, чтобы он был удобен для разработчика. Это правда, но еще структуризация удобна тем, что мы можем задавать параметры стилей для групп тегов. Например, в коде выше я прописал одинаковый стиль для  тега `<li>` дважды - было бы неплохо иметь способ прописать единожды код стиля и использовать его в каждом теге `<li>`.
У нас есть такая возможность. Эту процедуру можно осуществить в любом куске HTML документа, но для выставления общих по документу настроек мы используем тег `<head>`, поэтому именно там я напишу новый формат стилей.
Стили, которые мы хотим установить для всего документа оборачиваются в тег `<style>`. Далее мы пишем для каких групп тегов мы хотим установить стиль, затем ставим фигурную скобку и прописываем пары `<имя>: <значение>;`, но уже без кавычек.
Итак:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <style>
      nav {
        background: #c9f600;
      }
      li {
        display: inline-block;
        font-size: 20px;
        font-family: sans-serif;
      }
    </style>
  </head>
</html>
```
Когда стили прописаны так не нужно больше использовать атрибут `style` для каждого тега отдельно. Однако нужно иметь в виду, что такой дизайн будет применен ко всем тегам на странице с именами `nav` и `li` соответсвенно.
На этом этапе уже понятно, что даже такой способ задания стилей не совсем правильный, потому что на странице может располагаться несколько тегов `<nav>` и тем более несколько тегов `<li>` - не для всех мы хотим видеть одинаковый дизайн. Было бы славно разделить каким-то образом группы тегов с одним дизайном и с другим, не используя селектор по имени тега.
>[!important] Селектор - это имя для одного или нескольких тегов на странице. Например, `<li></li>` - это отдельный тег, а `li` - это имя тега, т.е. по которому можно с помощью селектора выбрать этот тег и все прочие с такими же именами.
### селектор по имени класса
Селектор по имени тега не очень практичная штука, поэтому умные белые люди придумали атрибут `class`, который можно прописать для каждого элемента на странице, чтобы указать внутри любую последовательность букв. Эту последовательность можно выбирать с помощью селектора и задавать всем элементам в ней нужные нам стили.
Чтобы сделать селектор по имени класса имя класса нужно сначала прописать в атрибут `class`:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <header>
	  <!-- задаем имя класса -->
      <nav class="header_nav">
        <ul>
          <!-- задаем имя класса -->
          <li class="header_nav_li active">
            <a href="/">Главная</a>
          </li>
		  <!-- задаем имя класса -->
          <li class="header_nav_li">
            <a href="/about">О сайте</a>
          </li>
        </ul>
      </nav>
    </header>
  </body>
</html>
```
>[!important] В этом примере есть строчка `<li class="header_nav_li active">` - через пробел указаны сразу два класса. Такое может быть. Можно указать хоть десять. Разделителем будет считаться пробел.

Далее мы можем точно так же как и в прошлый раз прописать тег `<style>` в `<head>`, но теперь вместо селекторов по имени тега мы будем использовать селектор по имени класса:
```HTML
<style>
  .header_nav {
    background: #c9f600;
  }
  .header_nav_li {
    display: inline-block;
    font-size: 20px;
    font-family: sans-serif;
  }
</style>
```
Теперь стили будут относиться не ко всем заголовкам с одним именем, а только к тем заголовкам, для которых указан нужный класс.
>[!important] Точка перед названием класса ставится, чтобы было понятно, что обращение происходит именно к классу.
### селектор по идентификатору
Кроме класса, в HTML есть еще атрибут `id`. Если класс может включать в себя кучу элементов на странице, то `id` включает в себя только один элемент. Нельзя присвоить другому элементу идентификатор, если он был уже использован. Механика установки идентификатора такая же, как и механика установки класса - нужно просто установить атрибут `id`:
```HTML
<nav id="header_nav">
</nav>
```
А чтобы обратиться к `id` в теге `<style>` нужно написать:
```HTML
<style>
  #header_nav {
    background: #c9f600;
  }
</style>
```
>[!important] Решетка перед названием идентификатора ставится, чтобы было понятно, что обращаемся именно к идентификатору.
### комбинированные селекторы
Когда мы пишем селектор перед фигурной скобкой, то можно указать сразу несколько. Фишка этой системы еще и в том, что мы можем дать адрес того тега, который мы хотим украсить.
Таким образом, например, я могу, не пользуясь классом, сказать: "Я хочу обновить только те теги `<a>`, которые лежат внутри тега `<li>`". Вот пример:
```HTML
<style>
  li a {
    color: white;
  }
</style>
```
Выше я использовал запись тега `<li class="header_nav_li active">`, чтобы выделить белым цветом активную ссылку в заголовке. То есть я хотел сделать так, чтобы у меня на странице было показано на какой вкладке находится юзер. Однако если я сейчас пропишу:
```HTML
<style>
  active {
    color: white;
  }
</style>
```
То ничего не произойдет, потому что таким образом я лишь добавил новое свойство самому пункту, а не тексту внутри ссылки.
Чтобы выполнить задуманное мне нужно либо прописать для самой ссылки класс  `active`:
```HTML
<a href="/" class="active">
```
Либо внутри стилей указать, что мне нужен не просто класс `active`, а элемент внутри класса `active`:
```HTML
<style>
  active a {
    color: white;
  }
</style>
```
>[!important] Путь до нужного элемента в селекторе указывается с помощью пробелов. Например, запись `li a` указывает на то, что в теге `<li>` есть тег `<a>`.

Для селекторов вообще есть классный синтаксис:
1) `*` означает, что мы выбираем любой тег. Соответсвенно `header *` означает, что мы выбираем все теги, которые вложены в тег `<header>`.
2) `p.class_name` эта запись означает, что мы выбираем теги `p`, которые принадлежат к классу `class_name`.
3) `p .class_name` означает, что мы выбираем все теги внутри тега `p`, которые принадлежат к классу `class_name`.
4) `#anton p` означает, что мы выбираем все теги `p` внутри элемента с `id="anton"`.
## пишем в файл
Стили полезно писать в файл с расширением `.css`. Во-первых, я люблю, чтобы весь код был там, где ему нужно быть, а не в одном единственном месте, во-вторых, можно сделать один файл со стилями и применить его ко всем страницам сайта.
Чтобы сделать такой файл я рядом с `index.html` создам папку `css`:
```Bash
.
├── css
│   └── style.css  # здесь стили.
└── index.html
```
В док со стилями я тупо перенесу все то, что писал внутри тега `<style>`:
```CSS
#header_nav {
  background: #83a000;
}
.header_nav_li {
  display: inline-block;
  font-size: 20px;
  font-family: sans-serif;
}
.active a {
  color: white;
}
```
Теперь нужно внутри нашего HTML дать ссылку на этот док. Это можно сделать через тег `<link>`:
```HTML
<link rel="stylesheet" href="css/style.css">
```
## наследование
Я писал о [[структура HTML#DOM|DOM-дереве в HTML документах ]]. В CSS эта тема раскрывается с двойной силой, потому что наследование тегов напрямую влияет на некоторые элементы внутри документа.
>[!important] Кстати, именно из-за такой наследующейся структуры стили называются "каскадными" - стиль может быть унаследован каскадно всеми потомками.
### наследование CSS-правил
Во-первых, существуют определенные CSS-правила, которые не только устанавливаются для самого узла, но и наследуются всеми его потомками.
В контексте уже обсужденного - именно такую логику исповедует CSS-правило `color`, т.к. при его установке мы получим новый цвет не только для самого тега, но и для его дочерних элементов:
```HTML
<section style="color: red; font-size: 40px;">
  <h1>привет</h1>
  <h2>привет</h2>
</section>
```
Результат будет  таким:
![[Code/Media_Code/8EF1A3D1-83A0-4CA1-A5E3-227AA44AD3B7.jpeg]]
Оба CSS-правила были унаследованы заголовками, которые вложены в тег `section`. Так же наследуются:
1) `font-family`- тип шрифта.
2) `font-style`- тип начертания (курсив, жирный шрифт и проч.).
3) `font-weight`- насыщенность шрифта.
В сущности наследуется CSS-правило или нет можно проверить на [сайте](https://developer.mozilla.org/ru/docs/Web/CSS/Reference#справочный_указатель) - там эта характеристика прописана для каждого тега.
### немного про насыщенность
Для шрифта можно вручную прописать настройку насыщенности - сделать шрифт тонким или самым жирным. Насыщенность измеряется от 100 до 1000, где 1000 - самый жирный шрифт. Изменения в параметрах насыщенности можно иногда не заметить, потому что используется `font-size` - у меня эти настройки конфликтуют.
Чтобы изменить насыщенность следует использовать `font-weight`:
```HTML
<section style="color: red; font-weight: 100;">
  <h1>привет</h1>
  <h2>привет</h2>
</section>
```
Здесь я выставил минимальную толщину текста:
![[B18C05BB-AEBF-4A74-A38A-24C27D3FD2A4.jpeg]]
Получается так, что данный вариант настройки просто задает размер текста.
### правила, которые не наследуются
Так же есть правила, которые не наследуются потомками, например, `border` - цвет и размер обводки:
```HTML
<section style="color: red; font-size: 40px; border: 1px dashed red;">
  <h1 style="border: 2px solid green;">привет</h1>
  <h2>привет</h2>
```
Не наследуется эта штука по очень простой причине - обводка может быть задана разная, как для всего блока, так и для отдельных элементов в нем:
![[6A324A62-D2C5-4205-9239-6D8FA5F433CA.jpeg]]
По тому же принципу определяется ненаследуемость всех прочих свойств:
1) `background` - цвет фона может быть выбран как для всего блока, так и для отдельных элементов в нем.
2) `padding` - внутренний отступ вокруг элемента может быть выбран как для всего блока, так и для отдельных элементов в нем.
3) `margin` - внешний отступ по тому же принципу.
4) `width` - ширина элемента.
5) `height` - "рост" элемента.
>[!important] Главное руководствоваться принципом: если настройка влияет не только на элементы одного свойства (например, не только на текст), то она скорее всего не наследуется, потому что может быть использована и в случае всего блока, и в случае отдельных элементов блока.
### немного про отступы
Существует два вида отсутпов:
1) Отступ внешней границы элемента от всех остальных снаружи.
2) Отступ элемента от границы внутри другого элемента.

Лучше всего представить это на примере коробок - коробка сама по себе может находится в отдалении от других коробок вокруг нее, однако внутри коробки тоже лежат предметы и они могут находится на каком-то расстоянии от стенок коробки.
Короче:
1) `padding` - внутреннее расстояние от границы до элемента внутри границы.
2) `margin` - внешнее расстояние от элемента до другого элемента.

## приоритет тегов
Что произойдет, если мы укажем стиль для тега `<nav>`, а потом укажем стиль для класса, к которому принадлежит тег `<nav>`? Например:
```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <style>
      nav {
        color: red;
      }
      .nav_class {
        color: green;
      }
    </style>
  </head>
  <body>
    <header></header>
    <main>
      <nav class="nav_class">
	    <h1>привет</h1>
	    <h2>привет</h2>
      </nav>
    </main>
    <footer></footer>
  </body>
</html>
```
Очевидно, что текст не может быть одновременно зеленым и красным, поэтому будет выбран какой-то конкретный цвет. Но по какому принципу выберется одна из настроек? По заранее установленному в правилах CSS приоритету: самыми приоритетными считаются те стили, которые прописаны индивидуально через атрибут `style`; далее, идет идентификатор - он написан так же для одного тега; затем класс и, наконец, селектор по тегу. Если существует два селектора с одним приоритетом для тега, то выбирается тот, что ниже.
>[!important] "Правило специфичности" - чем менее специфичен селектор, тем более приоритетным он будет. Специфичность определяется по количеству элементов, которые охватывает селектор.
