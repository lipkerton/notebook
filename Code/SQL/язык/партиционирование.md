Партиционирование - это процесс деления таблицы на части по какому-то признаку.
Например, у нас есть очень много отчетов за разные периоды в одной таблице. Так как обращаемся мы чаще всего к свежей отчетности, то не имеет смысла хранить вообще всю отчетность в одной таблице. Можно разнести отчетность за разные периоды по нескольким разным таблицам.
У меня есть таблица:
```SQL
drop table if exists transaction;
create table transaction (
    transaction_id bigint generated always as identity,
    transaction_date date not null,
    account_number varchar(50) not null,
    amount numeric(12, 2) not null,
    primary key (transaction_id, transaction_date) -- исправленный вариант
) partition by range (transaction_date);
```
Для этой таблицы я могу создать партицию:
```SQL
create table transcation_2023 partition of transaction
for values from ('2023-01-01') to ('2024-01-01');
```
Если посмотреть теперь информацию по всем таблицам с помощью команды `\d`, то можно увидеть, что основная таблица теперь приобрела статус `partitioned table`:
![[partition_t_1.png]]
Я создам еще партицию и дополнительно создам *дефолтную* партицию:
```SQL
create table transaction_future partition of transaction default;
```
Дефолтная партиция - это партиция "по умолчанию", куда будут распределены данные, если они не подходят под условия других партиций. Да, PostgreSQL автоматически будет распределять по партициям все данные, которые вставляются в таблицу `transaction`.
Если я вставлю данные:
```SQL
insert into transaction (account_number, amount, primary key)
values
('2023-01-25', '701', 10),
('2024-01-25', '702', 11);
```
То они автоматически будут убраны в нужные партиции, но при этом я могу увидеть их при обращении к изначальной таблице:
![[partition_t_2.png]]
Созданные индексы для основной БД, которые мы сделаем по полю для партиционирования, так же будут собирать данные и с партиций.
