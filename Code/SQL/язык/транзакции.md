## база
Транзакции - это способ сделать сочетание нескольких операций на языке SQL атомарными. Допустим у нас есть нужда удалить данные из одной части БД и вставить эти же данные в другую. Можно было бы написать обычный запрос, однако что будет если данные удаляться из первой части, но не добавятся во вторую (например, из-за ошибки). Транзакции призваны решить такую проблему - внутри транзакции операции либо выполняться все и вернут результат, либо не выполнится ни одна.
Стандартная транзакция в PostgreSQL начинается с ключевого слова `begin` и заканчивается ключевым словом `end`. Между этими двумя словами может быть насколько угодно большой набор разных запросов.
```SQL
begin;
insert into test_table(<колонки>)
select <колонки> from another_table where <условие>;
delete from test_table where <условие>;
commit;
```
Откатить транзакцию можно с помощью команды:
```SQL
rollback;
```
## savepoint
Иногда я пишу большие транзакции и было бы очень обидно на середине транзакции из-за какой-то ошибки откатывать ее полностью к самому началу. Чтобы такого не случилось умные белые люди придумали сейвпоинты.
Сейвпоинт можно поставить на любом этапе транзакции. При возникновении ошибки можно вернуться к этому сейвпоинту с помощью команды `rollback`.
Например, у меня есть задача заполнить две таблицы данными. Первая таблица - `pvz` описывает адрес и координаты пункта выдачи, куда необходимо прислать товары, вторая таблица - `pvz_tartif` ссылается на конкретный товар из первой таблицы и описывает цену этого товара.
```SQL
create table if not exists pvz (
	pvz_id serial primary key,
	address varchar(2000) not null,
	latitude numeric(8, 6) not null check(latitude between -90 and 90), 
	longitude numeric(9, 6) not null check(longitude between -180 and 180)
);
create table if not exists pvz_tariff (
	pvz_tariff_id serial primary key,
	pvz_id bigint references pvz(pvz_id) not null,
	delivery_price int not null check(delivery_price > 0)
);
```
Теперь я начну транзакцию и буду вставлять данные. Первый сет данных будет правильным, затем я поставлю чекпоинт, а второй сет данных будет неправильным - вызовет ошибку и PostgreSQL попросит сделать откат.
```SQL
begin;
insert into pvz(address, latitude, longitude) values
('г. Москва, ул. Пушкина, д. Колотушкина', 80.752200, 90.618990)
returning pvz_id; -- возвращаю id чтобы было проще заполнить поле референса в следующем инсерте.
insert into pvz_tariff(pvz_id, delivery_price) values
(1, 228);
savepoint point1; -- делаем savepoint.
insert into pvz(address, latitude, longitude) values
('г. Москва, ул. Пушкина, д. Колотушкина', 200, 90.618990)
returning pvz_id;
-- здесь возникнет ошибка.
rollback to point1; -- откатываемся к сейвпоинту.
```
Форма создания сейвпоинта такая:
```SQL
savepoint <имя>;
```
Имя может быть любым в пределах разумного.
Формат отката:
```SQL
rollback to <имя сейвпоинта>;
```
>[!important] Если сделать несколько сейвпоинтов и откатиться на более ранний, то все сейвпоинты, которые были сделаны после него удаляться.
