Транзакции - это способ сделать сочетание нескольких операций на языке SQL атомарными. Допустим у нас есть нужда удалить данные из одной части БД и вставить эти же данные в другую. Можно было бы написать обычный запрос, однако что будет если данные удаляться из первой части, но не добавятся во вторую (например, из-за ошибки). Транзакции призваны решить такую проблему - внутри транзакции операции либо выполняться все и вернут результат, либо не выполнится ни одна.
Стандартная транзакция в PostgreSQL начинается с ключевого слова `begin` и заканчивается ключевым словом `end`. Между этими двумя словами может быть насколько угодно большой набор разных запросов.
```SQL
begin;
insert into test_table(<колонки>)
select <колонки> from another_table where <условие>;
delete from test_table where <условие>;
commit;
```
Откатить транзакцию можно с помощью команды:
```SQL
rollback;
```
