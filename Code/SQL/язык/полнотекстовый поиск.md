# база
Полнотекстовый поиск - это штука, которая разлагает тест на лексемы и дает возможность искать по ним нужные нам значения. Используется, когда в таблице есть поля с каким-то текстом и нам нужна возможность выполнять быстрый поиск по этому тексту.
Такой механизм есть в PostgreSQL и он работает в т.ч. с русским текстом.
Чтобы достать лексемы из предложения можно сделать так:
```SQL
select to_tsvector('russian', 'Пример для полнотекстового поиска!');
```
В ответ на эту команду мы получим такое сообщение:
```SQL
                   to_tsvector
-------------------------------------------------
 'поиск':5 'полнотекстов':4 'пример':1 'текст':2
```
Как раз в этом сообщении можно понять, что такое "лексема": это просто обрубок слова. Почему это обрубок? Потому что так проще осуществлять поиск.
Например, мы хотим найти все слова "полнотекстовый" в поле. Очевидно, что у этого слова есть еще и склонения, которые мы пропустим, если будем искать только по полному совпадению. Лексема помогает избежать такой ситуации - из каждого слова достается неизменяемая часть и вот уже по ней происходит поиск.
*Слово `vector` в данном случае скорее всего ссылается на то, что у нас представлен текстовый вектор "Пример для текстового поиска".*
Теперь я хочу найти конкретное слово из строки выше:
```SQL
select to_tsvector('russian', 'Пример для полнотекстового поиска') @@ to_tsquery('russian', 'полнотекстовый')
```
`to_tsquery()` - это функция, в которую мы запишем то слово, которое будем искать.
*Слово `query` в данном случае описывает по стандарту "запрос", т.е. "полнотекстовый" - это запрос.*
Результат:
```SQL
 ?column?
----------
 t
```
Буква `t` в данном случае означает `true` - слово найдено.
>[!important] Забавно, что при поиске по самой лексеме "полнотекстов" такой же запрос возвращает `false`.
# а как такую штуку разместить в таблицу?
Для этого при создании таблицы лучше всего указать специальную колонку, которая будет автоматически делать лексемы из другой колонки + хранить их:
```SQL
create table article(
	article_id generated always as identity primary key,
	title text not null,
	content text not null,
	tsv_content tsvector generated always as (to_tsvector('russian', content)) stored
);
```
После того как БД создана нужно еще создать особый индекс, который будет нам помогать эффективно искать слова:
```SQL
create index aticle_tsv_content_ginidx on article using gin(tsv_content);
```
Теперь поиск по словам можно добавить в условие простого SQL запроса.
Я закину данные:
```SQL
insert into article (title, content) values
('Как использовать PostgreSQL', 'Этот материал расскажет вам про основы PostgreSQL и его возможности.'),
('Полнотекстовый поиск', 'В PostgreSQL есть мощные инструменты для полнотекстового поиска.'),
('Работа с индексами', 'Индексы помогают ускорять поиск данных в базе.');
```
И сделаю select-запрос:
```SQL
select article_id, title, content from article
where tsv_content @@ to_tsquery('russian', 'индекс')
```
Ура! Значения найдены.
