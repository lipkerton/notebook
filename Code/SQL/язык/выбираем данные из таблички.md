## выбрать все в таблице
```SQL
SELECT * FROM test_table;
TABLE test_table;
```
`TABLE` и `SELECT` в данном случае выполняют одну и ту же функцию. В результате обоих запросов вернется таблица со всеми колонками и строками.
## выбрать конкретные колонки
```SQL
SELECT column FROM test_table;
SELECT column_1, column_2 FROM test_table;
```
В первом случае берем одну колонку, а во втором две, таким же способом можем взять и три.
## выбрать с условием
```SQL
SELECT column FROM test_table
WHERE column_id == 1;
```
Выбираем все колонки, где `column_id` равен одному. Все стандартные операторы сравнения и логические операторы применимы здесь.
Логические операторы:
- `AND` - возвращает правду только если состояния справа и слева от оператора правдивы.
- `OR` - возвращает ложь только если состояния справа и слева от оператора правдивы.
- `NOT` - переворачивает ложь на правду, а правду на ложь.
Операторы сравнения:
- `|==`
- `|>=`
- `|<=`
- `|!=`
- `|>`
- `|<`
```SQL
SELECT column FROM test_table
WHERE (column_id == 1 AND name == 'Коля') OR number >= 10;
```
## сортировки
Обычная сортировка по возрастанию выполняется так:
```SQL
SELECT column FROM test_table
ORDER BY column_id;
```
Сортировка по убыванию требует, чтобы в конце стояло слово `DESC`:
```SQL
SELECT column FROM test_table
ORDER BY column_id DESC;
```
Сортировка по нескольким колонкам будет выглядеть так:
```SQL
SELECT column_1, column_2 FROM test_table
ORDER BY column_1, column_2;
```
Она работает так: вся выборка сортируется по первому значению в `ORDER BY`, а если значения совпали, то включается сортировка по второму значению. Для каждого значения можно указать способ сортировки - по возрастанию или по убыванию.
```SQL
SELECT column_1, column_2 FROM test_table
ORDER BY column_1 DESC, column_2;
```
## выбираем только уникальные значения
Вообще уникальное значение можно выбрать и при помощи `GROUP BY`, т.е. сгруппировав все значения по одной колонке:
```SQL
SELECT author
FROM authors
GROUP BY author;
```
Но группировка все же нужна не для того, чтобы выбирать уникальные значения. Для этой задачи есть специальный синтаксис - `DISTINCT`:
```SQL
SELECT DISTINCT author
FROM authors;
```
