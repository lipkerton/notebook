по умолчанию сервер БД PostgreSQL для каждого пользователя БД создает отдельное соединение. если мы говорим о продакшене, то в случае работы Django + PostgreSQL для каждого HTTP запроса в Django будет создано отдельное подключение к БД. 
у такого подхода есть недостаток - максимальное количество соединений с БД ограничено настройками PostgreSQL и мощностью сервера, на котором стоит кластер. это означает, что если мой ресурс посетит каких-нибудь 100 пользователей, то БД просто не позволит работать всем другим подключениям.
чтобы исправить эту проблему существует специальный инструмент - пуллер соединений. разновидностей есть несколько, но тут я скажу только о *pgbouncer*. он управляет соединениями к БД, создает очереди, где они нужны, освобождает соединения и т.д.
