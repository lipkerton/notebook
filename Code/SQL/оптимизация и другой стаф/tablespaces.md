Табличное пространство - это реальная форма организации данных в PostgreSQL. "Реальная" она потому, что все прочие понятия из организации данных ("схема", "таблица" и проч.) являются только логическими, то есть они не представлены в файловой системе компьютера.
Табличное пространство, напротив, представлено в файловой системе компьютера и оно является обычной папкой. Существует табличное пространство, которое PostgreSQL использует по умолчанию - оно уже создано.
Не смотря на то, что табличное пространство по сути является каталогом в системе мы не можем перенести этот каталог, чтобы использовать с другим кластером или не можем перенести его по частям. Вернее, физически это возможно, но ничего не будет работать, т.к. табличное пространство связано с конкретным кластером мета-данными (если его перенести, то связи будут нарушены => оно не будет работать).
## заготовка под тп
Я создавал его в Linux Debian. Механика проста - нужно выбрать папку, которая будет хранилищем для табличного пространства.
Для выбора лучше следовать такой логике - сами файлы с данными мы будем хранить на медленном и большом диске, а все индексы и оптимизацию будем хранить на быстром SSD. Получается нужно два пространства, но для начала нужно создать хотя бы одно.
Я выбрал `/mnt/pg_tablespaces/ts_data`:
```Bash
mkdir -p /mnt/pg_tablespaces/ts_data
```
Теперь есть нюанс - папка принадлежит мне и моей группе. Для табличного пространства важно, чтобы оно управлялось именно пользователем `postgres`, поэтому надо все права передать ему:
```Bash
sudo chown postgres:postgres /mnt/pg_tablespaces/ts_data
```
Теперь осталось перейти в пользователя `postgres` и запустить `psql`:
```Bash
su postgres
psql
```
## создание тп
Чтобы создать тп нужно использовать команду `create`:
```SQL
create tablespace <имя> location '/mnt/pg_tablespaces/ts_data';
```
Теперь можно создавать таблицы внутри нового табличного пространства:
```SQL
create table <имя>(<колонки>) tablespace <имя тп>;
```
## доп - посмотри файлы тп
Для наглядности можно пойти в файлы новой таблички и глянуть что там лежит. Но для начала нужно вставить в табличку данные, иначе ничего нельзя будет разглядеть:
```SQL
insert into <имя>(<колонки>) values ('hello');
```
Чтобы данные действительно добавились, а таблица обновилась нужно принудительно сделать контрольную точку:
```SQL
checkpoint;
```
Что теперь? В директории `/mnt/pg_tablespaces/ts_data` должна появиться какая-то структура файлов - они не предназначены для чтения пользователем, поэтому эта структура может оказаться немного хаотичной. У меня получилась такая:
```Bash
/mnt/pg_tablespaces/ts_data
`-- PG_17_202406281
    `-- 5
        |-- 49206
        |-- 49209
        `-- 49210
3 directories, 3 files
```
Где-то внутри одного из файлов с цифрами - `49206, 49209, 49210` - лежит наше слово `hello`.
Проверить это можно с помощью команды `xxd`:
```Bash
xxd <имя файла>
```
Так как все символы слова `hello` входят в набор кодировки ASCII, мы сможем увидеть буквально само слово среди всей бинарной информации.
