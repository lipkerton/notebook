## база
Каждая транзакция имеет свой уровень изоляции. В PostgreSQL существует четыре уровня изоляции транзакции (от самого мягкого до самого жесткого):
1) `read uncommitted`
2) `read committed`
3) `repeatable read`
4) `serializable`
При создании транзакции ей можно указать нужный уровень изоляции. Если уровень не указывать, то по умолчанию будет использован уровень `read committed`.
Признак разделения уровней транзакций - по количеству аномалий.
Аномалия - это неожиданный результат, который возникает при попытке выполнить несколько одновременных запросов в рамках одной БД.
Один из примеров аномалии - "грязное чтение". В этом случае транзакция_1 видит изменения в таблице, которые внесла транзакция_2, при том, что транзакция_2 еще не применила операцию `commit`, чтобы зафиксировать это изменение. Эта аномалия проявляется на уровне `read uncommitted`.
## read (un)committed
В PostgreSQL нет уровня изоляции `read uncommitted`. Этот уровень формально означает, что мы можем видеть еще незакоммиченные изменения внутри других транзакций.
На практике - это означало бы, что внесение изменений в БД транзакцией_1 может быть увидено транзакцией_2, которая так же может внести изменения, на основании еще незакоммиченных изменений транзакции_1. Если в транзакции_1 мы удалили с баланса 900 рублей (1000 - 900 = 100), а затем в транзакции_2 удалили еще 100 рублей, то, очевидно по итогу второй транзакции баланс будет нулевым в этом поле; но что если первая транзакция вдруг откатилась, а вторая закоммитилась уже после совершения отката? Мы ожидаем увидеть на балансе 900, а будет 0:
```SQL
-- транзакция_1
lessons=> begin;
BEGIN
lessons=*> update account set balance=balance-900 where account
_id = 1;
UPDATE 1
-- транзакция_2
lessons=> begin;
BEGIN
lessons=*> update account set balance=balance-100 where account
_id = 1;
UPDATE 1
-- транзакция_1
lessons=> rollback;
```
Опять же в PostgreSQL такой фокус не сработает - все транзакции, если видят, что данные меняются какой-то другой транзакцией, будут терпеливо ждать пока меняющая транзакция закоммитит изменения или откатится.
При большом желании можно прописать команде `begin` указание на уровень изоляции транзакции `read committed`:
```SQL
begin transaction isolation level read committed;
```
Но смысла в этом немного, т.к. все транзакции изначально обладают этим уровнем изоляции.
Так же можно все таки начать транзакцию в режиме `read uncommitted`:
```SQL
begin transaction isolation level read uncommited;
```
Но и в этом большого смысла нет, т.к. уровни `read committed` и `read uncommitted` никак не отличаются в PostgreSQL - грязного чтения в этой СУБД не существует.
