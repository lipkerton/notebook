При создании БД я столкнулся с проблемой - локали. Если не устанавливать локаль для БД, то скорее всего сортировки будут выполняться неправильно + появятся проблемы с учетом регистра символов и т.д.
## как установить локаль версия 1
Локаль задается, когда мы создаем БД:
```SQL
create database lessons
	with
	template=template0
	encoding='UTF8'
	lc_collate='ru_RU.UTF-8'
	lc_ctype='ru_RU.UTF-8'
	owner student;
```
Сразу видно какие параметры отвечают за нее:
1) `encoding` - указание какую кодировку будет использовать БД.
2) `lc_collate` - указание на то какая локаль будет использоваться при выполнении сортировки БД.
3) `lc_ctype` - здесь определяется каким образом будут классифицированы буквы, например, через эту настройку PostgreSQL отличит большую русскую букву от маленькой.
Очень красивая команда, но у меня она не работает:
```SQL
ERROR:  invalid LC_COLLATE locale name: "ru_RU.UTF-8"
HINT:  If the locale name is specific to ICU, use ICU_LOCALE.
```
PostgreSQL не знает такой локали, хотя ранее знал. Возможно, это как-то связано с тем, что ранее я запускал ее в версии PostgreSQL из докера, а теперь пытаюсь запустить для PostgreSQL из исходников.
В любом случае - нужно сначала проверить есть ли у меня хотя бы какая-то русскоязычная локаль:
```SQL
select collname, collprovider from pg_collation where collname like 'ru_RU%';
```
Параметры этой команды требуют объяснений:
1) `collname` - имя локали (например, `"ru_RU.UTF-8"` - это как раз такое имя локали).
2) `collprovider` - это библиотека, которая предоставляет сведения о локали; в данном случае она отображается одной буквой, потому что таких библиотек у PostgreSQL всего четыре - `default`, `builtin`, `libc`, `icu`.
3) `pg_collation` - это каталог, который описывает набор доступных локалей.
У меня получился такой результат:
```SQL
  collname   | collprovider
-------------+--------------
 ru-RU-x-icu | i
(1 row)
```
Одна русская локаль все же есть. Буква `i` во второй колонке ссылается на библиотеку `icu`. Про ту же библиотеку сказал сам PostgreSQL, когда мой прошлый запрос на создание БД упал с ошибкой:
```SQL
HINT:  If the locale name is specific to ICU, use ICU_LOCALE.
```
>[!important] International Components for Unicode - это кроссплатформенная библиотека  с кучей локалей в ней. Сравнивая с `libc` - `icu` не зависит от тех языков, которые в данный момент используются операционной системой, т.е. из нее можно подгрузить любой язык, который ей известен.

Если я тупо выпишу имя этой локали вместо `'ru_RU.UTF-8'`, то ничего не получится - PostgreSQL не найдет ее.  Так получится, потому что имя локали, которое здесь упомянуто известно только `icu`, соответственно, мне нужно как-то обратиться к этой библиотеке.
## как установить локаль версия 2
Теперь я хочу указать провайдера локали напрямую + прописать какую локаль из него я хочу получить:
```SQL
create database lessons
	with
	locale_provider=icu
	icu_locale='ru_RU'
	template=template0
	owner student;
```
В моем случае это сработало. Теперь нужно проверить правильно ли выполняется сортировка. Для этого я создам простую таблицу с одним полем и понапишу туда слов:
```SQL
create table test(value varchar(150));
insert into test(value) values ('Арбуз'), ('Яблоко'), ('Банан');
select * from test order by value;
```
В итоге должен получится отсортированный результат.
