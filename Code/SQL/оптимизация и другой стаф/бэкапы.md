бэкапы существуют в трех вариациях:
- логический
- физический
- использование WAL-журнала
логический бэкап по сути является набором SQL команд, с помощью которых копируется содержимое БД. его преимущество в том, что такое копирование можно гибко настроить, например, указать на конкретные места в БД, которые необходимо скопировать.
физический бэкап это полный дамп БД со всеми ее данными.
WAL-журнал - это журнал регистрации транзакций в БД, поэтому с его помощью можно создать бэкап, если архивировать его. изменения из WAL-журнала можно раскатывать на запасных серверах БД.
для создания логического бэкапа у PostgreSQL существует утилита в стандартной поставке:
```Bash
pg_dump -U username -d database -Fd -f backup.dump
```
чтобы восстановить БД можно ввести:
```Bash
pg_restore -U username -d database -Fd backup.dump
```
Параметры `-Fd` означают формат выходного файла. В данном случае будет папка.
для создания физического бэкапа у PostgreSQL так же есть утилита:
```Bash
pg_basebackup -U username -D /backup/ -F tar -z -P
```
