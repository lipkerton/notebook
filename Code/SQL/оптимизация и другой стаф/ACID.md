Это набор принципов, по которым определяется хорошая SQL транзакция.
A (Atomicity) - атомарность. Транзакция должна быть атомарной - это означает, что операции, которые выполнены лишь частично, должны быть отменены, а все их изменения должны откатиться.
C (Consistency) - согласованность. Все операции внутри транзакции должны быть согласованны с правилами, которые установлены в рамках БД. Это означает, что операции не должны нарушать check-проверки, типы данных колонок и проч. На выходе всегда должны получаться данные, которые согласованы с правилами БД и таблицы.
I (Isolation) - изолированность. Это правило означает, что конкурентно выполняемые транзакции не должны влиять друг на друга.
D (Durability) - долговечность. Это правило означает, что данные, которые были созданы в итоге работы транзакции, должны быть надежно сохранены. Работа транзакции будет успешной в этом случае, только когда данные будут записаны на диск. Транзакция должна гарантировать успешную запись.
