Допустим у нас есть таблица, которая содержит огромное количество записей. Записи в нее добавляются непоследовательно - так часто бывает с пользовательскими приложениями, т.к. нет никакой логики добавления данных в БД (пользователи не будут ждать пока приложение вычислит место их данных в миллионе строк других данных). Если мы попытаемся произвести поиск по такой таблице, то из-за отсутствия в ней какого-то порядка PostgreSQL организует `Seq Scan` по всему набору данных, даже если нужно найти всего одну строку.
Чтобы избежать такого поведения оптимизатора создается индекс. Индекс - это сущность, которая никак не влияет на саму таблицу с данными. Индекс создается на основании какого-то признака в соответствующей таблице и представляет собой жесткую отсортированную структуру, которой может воспользоваться оптимизатор для поиска значений.
Существует несколько разных типов индексов, но самый популярный - B-tree или Balanced Tree.
# B-tree
Если мы будем сканировать таблицу в поисках значения втупую, то у нас получится алгоритм `Brute Force` - перебор всех строк с данными, пока не найдем нужную. Сложность такого алгоритма - $O(n)$, т.е. линейная сложность (чем больше данных, тем больше времени потребуется - на перебор 1кк данных может уйти, например, 1кк секунд).
Однако если у нас есть отсортированный набор значений, то мы можем поступить умнее:
1) разбиваем набор надвое.
2) проверяем в какой из половин может находится наше значение.
3) идем в эту половину, сразу откидывая вторую (т.к. там нашего значения точно нет).
4) берем эту половину, как новый набор и повторяем процедуру, пока не найдем искомый элемент.
Сложность такого алгоритма - $O(log2(n))$. Это означает, что для поиска по набору из 8 строк нам понадобится совершить 3 операции: $1) 8 / 2 = 2) 4 / 2 = 3) 2 / 2 = 1$. То есть в худшем случае нужно будет вместо 8 операций сделать три. Именно так работает `B-tree`.
Почему `B-tree` называют "сбалансированным"? Потому что время поиска по нему примерно одинаково для каждой записи + все листья (нижние уровни таблицы) находятся в нем примерно на одном уровне.
# селективность
Селективность - отношение количества выбираемых данных к общему количеству данных в наборе данных. Например, я выбираю одну строку из ста, тогда моя селективность будет равна $1 / 100 = 0.01$.
Низкая селективность означает, что достается **большая** шасть данных таблицы.
Высокая селективность означает, что достается **меньшая** часть данных таблицы.
