С одной стороны у нас есть JOIN, который позволяет смешивать таблички, но с другой нужно понимать, что само это смешивание может происходить по разному. Например, если у нас есть таблица категорий фильмов и таблица самих фильмов, то понятно, что одна категория будет проставлена для многих фильмов и, наоборот, если мы посмотрим на эту ситуацию с точки зрения таблицы фильмов - многие фильмы относятся к одной категории.

Это базовый Паттерн связи между таблицами - он называется **один ко многим**.
