любая программа на C - это набор файлов или файл с расширением `.c`.
любая программа на С содержит точку входа - функцию `main()`.
поэтому если я хочу написать в консоли "hello world" мне нужно создать файл с нужным расширением и вставить туда функцию `main()`:
```C
main() {
...
}
```
технически функция всегда должна возвращать какое-то значение, поэтому перед наименованием функции часто стоит тип возвращаемого значения, например:
```C
int main() {
...
}
```
по умолчанию любая функция без указанного типа вернет `int`:
![[с15.png]]
обычно функция `main` содержит какие-то инструкции.
сейчас внутри нее я напишу инструкцию `printf()` - она печатает строковые сообщения в стандартный поток вывода:
```C
main() {
	printf("Привет, мир!\n");
}
```
проблема только в том, что все используемые функции в C хранятся в специальных заголовочных файлах. без заголовочного файла компилятор просто не поймет что я написал на экране:
```C
#include <stdio.h>

main() {
	printf("Привет, мир!\n");
}
```
заголовочные файлы дают компилятору информацию об используемых функциях, чтобы он мог их использовать.
`\n` является символом переноса строки. эта штука называется *escape-последовательность*. существует несколько таких последовательностей, но я о них говорить не буду.
