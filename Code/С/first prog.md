теперь разберу что в нем происходит.
первые пять строк - это что-то вроде импортов в Python.
команда `#include` - это директива препроцессора, препроцессор обрабатывает код перед выполнением компиляции.
текст, который идет за ней (`<stdio.h> <stdlib.h> <sys/time.h> <assert.h>`) это наименования заголовочных файлов (расширение `h` указывает на то, что передо мной заголовочный файл). заголовочные файлы содержат объявления, но не реализации. другими словами, такие файлы содержат объявления функций, но при этом они никак не задействуют эти функции.
есть один особенный `#include` - `#include "common.h"`. почему он записан в кавычках?
потому что угловые скобки (`<>`) означают, что `#include` должна выполнять поиск в директориях, где установлены стандартные библиотеки компилятора, а кавычки означают, что `#include` должна выполнить поиск сначала по пути, который указан в кавычках, и только потом в директории стандартных библиотек.
что означают библиотеки, которые я беру:
```
<stdio.h>    = stdio = STanDard Input/Output
<stdlib.h>   = stdlib = STanDard LIBrary
<sys/time.h> = `/sys/time.h` файл для работы со временем в Unix-системах.
```

следующий кусок кода начинает функцию в C:
```C
int
main(int argc, char *argv[])
```
первая строка - `int`, отвечает на вопрос "Какой тип данных возвращает функция?".
далее начинается функция `main`. функция `main` является главной функцией любой программы на С. именно с нее начинает свое выполнение вся программа.
в круглых скобках `(int argc, char *argv[])` переданы аргументы функции:
- `argc (argument count)` - это количество аргументов командной строки (целое число).
- `argv (argument vector)` - это массив строк, который содержит сами аргументы.
например, если я запущу программу `./cpu1 Hello`, то:
- `argc = 2` потому что я передаю два аргумента `./cpu1` и `Hello`.
- `argv[0] = "./cpu"; argv[1] = "Hello"`
границы функции определяются фигурными скобками.
внутри фигурных скобок сразу же описывается условие:
```C
if (argc != 2)
{
	fprintf(strderr, "usage: cpu <string>\n");
	exit(1);
}
```
`if (argc != 2)` - это условная конструкция, а `argc != 2` - это само условие: "если количество аргументов командной строки не равно двум, то выполни блок кода в фигурных скобках":
```C
fprintf(stderr, "usage: cpu <string>\n");
exit(1);
```
`fprintf(stderr, ...)` - функция вывода. `stderr` - обозначение стандартного потока ошибок, а дальше идет сообщение для вывода `"usage: cpu <string>\n"`. можно сменить поток вывода ошибок, выбрав `stdout`, в этом случае вывод будет печататься в стандартный поток вывода.
`exit(1)` - функция завершения программы с кодом возврата `1`, который означает неудачу.

что же лежит в функции `Spin()`?
по идее задумка всей программы состоит в том, что я запускаю бесконечный цикл, который постоянно повторяет одну и ту же строку каждую секунду своей работы. следовательно, функция `Spin()` должна каким-то образом считать секунды и давать сигнал о том, что секунда прошла.
возвращаясь к директивам препроцессора, которые я описал выше, важно вспомнить, что там была такая штука:
```C
#include "common.h"
```
эта директива ищет файл по переданному относительному или абсолютному пути. при такой записи она будет искать файл в текущей папке. именно в нем и должна быть написана функция `Spin()`:
```C
void Spin(int seconds) {
	struct timeval start, now;
	gettimeofday(&start, NULL);
	while (1) {
		gettimeofday(&now, NULL);
		if (now.tv_sec - start.tv_sec >= seconds) break;
	}
}
```

однако не все так просто
сначала я подумал, что файл `.h` нужен, чтобы в нем можно было описывать функции, которые можно импортировать, однако мой недалекий мозг не пронял что именно должно происходить в этом файле.
в файлах `.h` описываются объявления функций - они будут интерфейсом, для тех файлов с расширением `.c`, которые будут использовать функцию.
например:
```C
// common.h
void Spin(int seconds);
```
само тело функции должно быть описано в файле с расширением `.c`.
моя программа небольшая - я бы мог уместить ее всю в одном файле, но я хочу попробовать связать два скрипта между собой.
для этого заведу файл `spin.c`:
```C
#include <stdio.h>
#include <stdlib.h>
#include <sys/time.h>

void Spin(int seconds) {
	struct timeval start, now;
	gettimeofday(&start, NULL);
	while(1) {
		gettimeofday(&now, NULL);
		if (now.tv_sec - start.tv_sec >= seconds) break;
	}
}
```

как работает код, который я описал выше?
первые три строки - директивы препроцессора.
затем я объявляю функцию `Spin(int seconds)`, которая вернет `void`, т.е. ничего.
`struct timeval` - это структура, которая хранит время. структуры в C это буквально набор переменных, который объединен одной абстракцией. в данном случае - время.
функция `gettimeofday(&start, NULL);` получаем текущее время и передаем на вход функции адрес структуры `start`, чтобы функция заполнила ее временем. `NULL` - является вторым аргументом, т.к. ранее второй параметр функции использовался для заполнения часового пояса.
в атрибуты структуры `start` - `start.tv_sec` и `start.tv_usec` записывается текущее время.
далее сразу начинается бесконечный цикл:
```C
while (1) {
	...
}
```
внутри цикла я получаю текущее время снова:
```C
gettimeofday(&now, NULL);
```
и делаю условие выхода из цикла:
```C
if (now.tv_sec - start.tv_sec >=seconds) break;
```
данный цикл будет повторять себя пока разница между текущим временем и начальным временем не будет больше или равна полученным в функцию секундам.
