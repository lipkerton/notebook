еще одна программа из книжки:
```C
#include <unistd.h>
#include <assert.h>
#include <fcntl.h>
#include <sys/stat.h>

int main(int argc, char *argv[]) {
	int fd = open("/tmp/file", O_WRONLY | O_CREAT | O_TRUNC, S_IRWXU);
	assert(fd > -1);
	int rc = write(fd, "hello world\n", 13);
	assert(rc == 13);
	close(fd);
	return 0;
}
```
`unistd.h` содержит объявления системных вызовов втч позволяет вызывать функции `open()` и `write()`. `assert.h` - предоставляет функцию `assert()`. `fcntl.h` содержит в себе флаги `O_WRONLY`, `O_CREAT`, `O_TRUNC`, а `sys/stat.h` нужен для флага `S_IRWXU`.
я начну с функции `open()`.
она принимает первым аргументом адрес, где будет создан наш файл.
второй аргумент - комбинация флагов через побитовое ИЛИ. побитовое ИЛИ здесь нужно для того, чтобы выполнить одно, два или все три действия сразу. флаги:
- `O_WRONLY` - открыть файл только для записи.
- `O_CREAT` - если файл не создан, то его нужно создать.
- `O_TRUNC` - если файл уже существует, то нужно очистить его.
третий аргумент - это режим доступа к файлу. `S_IRWXU` указывает права на новый файл. из-за этого флага только у владельца файла будут права на чтение, запись и выполнение файла.
следующая строчка проверяет, что `open()` вернула число больше $-1$. минус единица в данном случае означает, что функция вернула ошибку.
функция `write()` получает на вход открытый файл, строку, которую следует напечатать, и количество байт, которое мы хотим записать в файл. количество байт равно $13$, потому что внутри строки лежит $11$ символов + $1$ символ переноса строки + $1$ завершающий байт для нуля (С всегда ставит по умолчанию ноль в конце).
после этого я проверяю, что функция действительно вернула то число байт, которое мы хотели записать и закрываю файл.
