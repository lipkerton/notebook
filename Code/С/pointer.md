# указатель
а что такое указатель?
насколько я понял указатель позволяет сказать "переменная x смотрит на участок в памяти, где хранится значение переменной y". чтобы проверить мою догадку лучше всего посмотреть как работает память в С.
память типичной машины представляет собой массив последовательно пронумерованных или проадресованных ячеек, с которыми можно работать по отдельности или связанными кусками. применительно к *любой* машине верны следующие утверждения:
- один байт может хранить значение типа `char`
- двухбайтовые ячейки могут рассматриваться как целое типа `short`
- четырехбайтовые - как целое типа `long`
**указатель** - это группа ячеек (весом 1-8 байт), в которых может храниться адрес.
например, если `c` имеет тип `char`, а `p` указатель на `c`, то это означает, что `p` был присвоен адрес  ячейки `c`.
я написал такой код:
```C
#include <stdio.h>

void main() {
    int c = 1;
    int *p = &c;

    printf("Привет, мир!\nЧисло `c`: %d\nЧисло `p`: %d\n", c, p);
}
```
когда я пишу `int *p` на самом деле я объявляю переменную `p`, которая является указателем на область памяти, где хранится значение с типом `int`.
когда я пишу `&c` на самом деле я пытаюсь получить от переменной адрес ячейки памяти, где хранится ее значение.
в чем прикол?
например, в том, что через указатель я могу управлять значением в ячейке памяти:
```C
#include <stdio.h

void main() {
	int c = 1;
	int *p = &c;
	
	*p = 2;
	
	printf("Привет, мир!\nЧисло `c`: %d\nЧисло `p`: %d\n", c, p)
}
```
теперь переменная `c` так же станет равна двум:
![[c3.png]]
важно понимать:
```C
int a = 1;
int *p;  // объявляю указатель на ячейку памяти с значением `int`.
p = &a;  // записываю в указатель адрес значения в переменной `a`.
*p = 2;  // беру адрес в памяти, хранимый `p`, и пишу в него число `2`.
```
у меня было смущение, что запись:
```C
int *p = &a;
```
очень похожа на запись:
```C
*p = &a;
```
что было бы неправильно. т.к. в последнем случае я записал бы номер адреса в памяти внутрь ячейки памяти, на которую указывает `p`.
с указателями вообще можно творить самые больные вещи:
```C 
int a = 1;
int *b = &a;
int **c = &b;
int ***d = &c;
int ****e = &d;
****e = 0;
printf("%d", a);  // => 0
```
# указатель на строку
я встретил такой код:
```C
char *a = "Hello world!";
```
фишка типа `char` в том, что он может хранить только одну букву. но в данном случае он хранит ссылку на ячейку памяти, где хранится `H` - первая буква строки.
если я попробую напечатать `a`:
```C
char *a = "Hello world!";
printf("%s", a);  // => Hello world!
```
