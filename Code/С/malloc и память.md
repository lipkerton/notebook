`malloc` = *memory allocation*
`allocation` = *выделение*
механизм динамической *аллокации* памяти. используется, чтобы аллоцировать один блок виртуальной памяти в HEAP.
динамическое выделение памяти - это просьба программы *во время работы программы* к ОС компьютера дать память для работы. программа может изменять размер такой памяти, освобождать такую память.
фишка динамического выделения в том, что память не освобождается автоматически - ее нужно освобождать отдельной инструкцией.
для того, чтобы было представление о том как работает память лучше всего подглядывать в картинку:
![[Memory-Layout-of-C-Program.png]]
у каждой программы есть своя секция памяти. секция памяти программы делится на несколько частей. у каждой ячейки памяти программы есть адрес. адреса спускаются сверху вниз, поэтому есть *high address* и *low address*. в самом начале памяти лежат переменные окружения и аргументы командной строки, потом идет STACK и HEAP (между ними лежит область памяти, которая свободна специально для расширения STACK вниз, а HEAP вверх). в самом низу есть область *text* - она хранит сам код программы. между HEAP и *text* память занимает *data segment*; он делится на две части: *uninitialized data* - память хранит глобальные и статические переменные, которые не были инициализированы программистом (таким переменным присваивается нулевое значение в памяти), *initialized data* - память хранит глобальные и статические переменные, которые были инициализированы программистом.
программа, которая демонстрирует статические и глобальные переменные в *data segment*:
```C
// global vars (initialized data segment)
int globalVar = 10;
char[] message = "Hello!";

int main() {
	
	// static vars (initiaized data segment)
	static int staticVar = 11;
	
	printf("Global variables: %d\n", globalVar);
	printf("Static variables: %d\n", staticVar);
	printf("Message: %s\n", message);

	return 0;
}
```
локальная переменная появляется в функции, например, в `main()` только на время действия функции. это означает, что как только функция что-то возвращает, то ее переменные исчезают.
# stack
это временное хранилище локальных переменных является частью механизма *function call stack*, который так же называется просто STACK. вообще *fuction call* является коробкой, которая создается каждый раз, когда функция вызывается, и которая уничтожается каждый раз, когда функция возвращает значение. такие коробки называются **stack frames** или просто **frames**. стек обычно рисуют растущим вниз, потому что это тупо интуитивно - функции в программе идут сверху вниз + в большинстве архитектур адреса в стеке растут в сторону уменьшения, например, первой функции может быть присвоен какой-нибудь максимальный адрес, а последней нулевой.
я напишу еще одну программу:
```C
#include <stdio.h>

const float EULER = 2.71828f;
const int COUNT = 10;

void fill_exp(float* dest) {
	dest[0] = 1.0f; 
	for (int i = 1; i < COUNT; i++) {
		dest[i] = dest[i - 1] * EULER;
	}
}

void print_floats(float* values, int n) {
	for (int i = 1; i < n; i++) {
		printf("%f\n", values[i]);
	} 
}

int main() {
	float values[COUNT];
	fill_exp(values);
	print_floats(values, COUNT);
	return 0;
}
```
здесь объявлено две глобальные константы - `EULER` и `COUNT`. так как значение у них уже есть, то я помню, что это переменные, которые хранятся в `initialized data` сегменте памяти программы.
внутри функции `main()` я создаю переменную `values` с типом `float`. что означает приписка `[COUNT]`? на базовом уровне понятно, что это создание тн последовательности, но не понятно каким образом она работает.
на самом деле таков синтаксис объявления последовательности. если я пишу `float values[COUNT]`, то на выходе я получу последовательность, которая рассчитана на `COUNT` количество элементов. так происходит потому, что такое объявление выделяет память под десять элементов типа `float` на стеке.
следующая строка в функции `main()` особенно важна в этом контексте: `fill_exp(values)`. я передаю в функцию не весь массив, а только первый элемент массива. нужно посмотреть что именно принимает на вход функция `fill_exp` в качестве параметра:
```C
void fill_exp(float* dest) {
	...
}
```
она принимает на вход указатель на ячейку памяти с значением типа `float`, т.е., когда мы передаем на вход `values`, функция видит это как адрес первого элемента массива в памяти. благодаря тому, что `dest` - это указатель на первый элемент массива, я могу менять значения всех остальных элементов в массиве:
```C
void fill_exp(float* dest) {
	dest[0] = 1.0f;
	for (i = 1; i < COUNTER; i++) {
		dest[i] = dest[i - 1] * EULER;
	}
}
```
вообще этот кусочек кода дает окошко в большую тему
# арифметика указателей
история гласит, что некоторые арифметические операции мы можем совершать с указателями. но если указатель указывает на ячейку памяти, то в каком смысле мы можем прибавить к указателю единицу? что получится в итоге?
здесь нужно посмотреть внимательнее на этот цикл: 
```C
void fill_exp(float* dest) {
	dest[0] = 1.0f;
	for (i = 1; i < COUNTER; i++) {
		dest[i] = dest[i - 1] * EULER
	}
}
```
если указатель указывает  только на первый элемент, то как я могу получить второй элемент, если применю такую запись `dest[1]`?
на самом деле такая запись эквивалентна записи  `dest + 1`.
теперь, чтобы понять как я могу к адресу памяти прибавить единицу и получить следующий элемент нужно написать другой код:
```C
#include <stdio.h>

int main() {
	int arr[5] = {10, 20, 30, 40, 50};
	int *ptr = arr;
	printf("Адрес первого элемента: %p\n", ptr);
	printf("Адрес второго элемента: %p\n", ptr + 1);
	return 0;
}
```
здесь я делаю массив из пяти элементов и указатель на первый элемент массива, а затем печатаю указатель и сумму указателя и единицы. вывод такой программы у меня:
![[c8.png]]
в экземпляре кода я создавал массив из элементов типа `int`. компилятор всегда выделяет под тип `int` четыре байта в памяти и эти четыре байта я могу увидеть здесь - адрес первого элемента имеет в конце $0$, но когда я добавил единицу я уже получил `0x7fff0a66114`, т.е. новый адрес на четыре байта больше.
это означает, что добавляя единицу к указателю я на самом деле добавляю к нему именно столько байт, сколько весит тип этого указателя. таким образом я получаю следующий элемент.
что будет если вычесть из указателя на первый элемент значение?
```C
int main() {
	...
	printf("Адрес какого-то элемента: %p\n", ptr - 1);
	printf("Элемент по какому-то адресу: %p\n", *(ptr - 1));
}
```

теперь можно вернуться к предыдущей теме - к функции `fill_exp()`. как все уже поняли, я в ней беру указатель на первый элемент массива, который позволяет мне установить значение всех других элементов через арифметику указателей.
вообще зачем мне передавать в функцию именно указатель на первый элемент, а не весь массив? затем, что так экономнее - при передаче всего массива в функцию пришлось бы создавать локальную копию этого массива внутри функции с выделением дополнительной памяти. если же я передаю указатель, то я просто передаю указатель на место в памяти, где начинается массив, и этого достаточно, чтобы работать с ним.
итак, я передал указатель и для пустого массива `values` объявил элементов в количестве `COUNT`. крутость такого подхода в том, что мне ничего даже не нужно возвращать из функции - я меняю саму память, где должны лежать элементы массива.

этот массив должен напечатать в стандартный поток вывода элементы массива.
и пора, наконец, обратить внимание на слона в комнате - цикл `for`:
```C
for (int i = 1; i < COUNT; i++) {
	...
}
```
механика такова:
```C
for (инициализация: условие; итерация) {
	тело цикла;
}
```
для эксперимента я написал такой код:
```C
#include <stdio.h>

int main() {
	for (int a = 1, b = 10; a < b || b > a; a++, b--) {
		printf("a: %d\tb: %d\n", a, b);
	}
	return 0;
}
```
как видно, в каждом из трех блоков цикла `for` можно перечислить несколько элементов через запятую. внутри условия я так же поставил знак `||`, который является логическим `ИЛИ` в C. в блоке итерации я определяю какие события будут происходить с переменными цикла каждую итерацию. например, на первом шаге из значения переменной `b` будет вычтена единица и к значению переменной `a` будет прибавлена единица.

---
ключевое ограничение stack в том, что переменные живут столько же сколько живет вызов функции. отсюда следует правило - не нужно возвращать из функции указатель на локальную переменную, т.к. область памяти где находится ее значение будет очищена после завершения вызова.
нужно понимать, что когда я передаю в функцию указатель на область в памяти - это абсолютно безопасно, ведь переменная для этой области в памяти существует пока существует ее *родительская функция*. точно так же безопасно возвращать значение переменной из функции - компилятор копирует возвращаемое значение в stack frame вызывающей функции.
