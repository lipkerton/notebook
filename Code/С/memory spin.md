вторая программа из книжки:
```C
#include <stdio.h>  // для printf
#include <stdlib.h> // для malloc
#include <unistd.h> // для getpid()
#include <assert.h> // для assert()
#include "common.h"

int main(int argc, char *argv[])
{
    int *p = malloc(sizeof(int));
    assert(p != NULL);
    printf("(%d) address of p: %08x\n", getpid(), (unsigned) p);
    *p = 0;
    while (1) {
        Spin(1);
        *p = *p + 1;
        printf("(%d) p: %d\n", getpid(), *p);
    }
    return 0;
}
```
что тут происходит?
это программа, которая демонстрирует работу виртуальной памяти.
первые две строчки подключают стандартные библиотеки - ввод-вывод и просто "стандартную библиотеку". 
затем написана инструкция `#include <unistd.h>`. `unistd.h` = `UNIx STandarD`. это библиотека, которая содержит объявления для большинства системных вызовов UNIX и POSIX API.
кстати, что такое POSIX? **POSIX** - это общий для UNIX систем стандарт написания ПО (aka *стандарт совместимости*), который гарантирует, что программа для одной из UNIX систем будет работать и на всех остальных.
**POSIX = Portable Operating System Interface X**. Это стандарт, который появился, когда на основе Unix начали делать OS. Изначально, когда код UNIX был написан на языке ассемблера, разработчики переписали его на C, что позволило портировать ПО с одного железа на другое. Однако затем на основе UNIX стали появляться другие системы, которые делали собственные уникальные интерфейсы - исходный код одной программы нельзя было просто перекомпилировать под другую систему. *POSIX* решает эту проблему устанавливая стандарт для *интерфейсов*, чтобы программы можно было компилировать схожим образом для любой UNIX системы.
в коде далее:
```C
int *p = malloc(sizeof(int));
```
функция `sizeof()` вычисляет размер типа данных или переменной в памяти.
`malloc()` запрашивает указанный размер памяти у HEAP-памяти. в данном случае я прошу четыре байта - размер типа `int`.
в следующей строке я применяю функцию `assert()` из заголовочного файла `assert.h`. она принимает на вход выражение и проверяет его; в случае ложного результата вернется ошибка, а в случае правдивого ничего не произойдет.
затем печатается строка. интересно в ней то, что используется функция `getpid()` - она создает системный вызов, который должен вернуть идентификатор текущего процесса (каждый идентификатор процесса уникален).
еще интересно, что здесь я впервые вижу приведение типов:
```C
(unisgned) p
```
я привожу указатель типа `int` к типу `unsigned`, что это значит? я пока понятия не имею почему автор сделал так. `int` занимает четыре байта в памяти, как и `unsigned int`, поэтому смена типа указателя мне не кажется целесообразной.
спецификатор формата `%08x` говорит о том, что в консоль будет выведено шестнадцатеричное число, которое состоит из восьми знаков. в данном случае ноль означает, что недостающие знаки слева будут заменены нулями.
после печати в консоль информации о ячейке памяти я записываю в память ноль и начинаю бесконечный цикл. в цикле вызывается функция `Spin(1)` и печатается снова строка.
