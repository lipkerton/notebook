Это команды, которые помогают переливать обновления между ветками Git и забирать обновления из удаленного репозитория на локальный.
# git fetch
Это безопасная команда, которая просто обновляет на локальной машине информацию относительно состояния удаленного репозитория. 
Например, у меня есть желание посмотреть какие обновления были залиты в удаленный репозиторий пока я спал. Это можно сделать с помощью:
```Bash
git log orgin/main
```

Локально у меня развернут коммит `new files`, а удаленно я вижу коммит `new lines`. Так произошло, потому что я откатил локально репозиторий к состоянию прошлого коммита.
Если я попробую посмотреть лог для локального репозитория, то увижу:
![[git_log_4.png]]
На данном этапе я просто демонстрирую состояние дел моих репозиториев, но что если я скажу, что мой лог `git log origin/main` показывает неверную информацию о состоянии удаленного репозитория? Так происходит потому, что я с другого устройства обновил ветку `main`, но на этом устройстве я об этом еще не знаю. А почему я об этом тут не знаю?

---

А как я вообще вижу состояние удаленного репозитория, когда использую `git log origin/main`? Я вижу его состояние, потому что на самом деле я смотрю в текущее состояние "трекинг-ветки" - это ветка, которая размещена локально и предназначена для поддержания связи между локальным и удаленным репозиторием. То есть:
```Bash
git log  # смотрю в текущую локальную ветку
git log origin/main  # смотрю в локальную трекинг-ветку
```
Ни в одном из этих случаев я не получаю данные об актуальном состоянии удаленного репозитория.

---

Как же мне получить актуальные данные? Как раз в этом помогает `git fetch`:
```Bash
git fetch
```
До команды `git fetch` трекинг-ветка `origin/main` содержала не актуальную информацию:

Но после команды трекинг-ветка обновится:

# git merge
При описании работы `git fetch` ее часто можно встретить в связке с `git merge`. Почему? Потому что `git fetch` выполняет обновление локальных трекинг-веток - обязательное условие для корректной работы `git merge`.
`git merge` выполняет слияние двух веток разработки в одну линию коммитов. Делает он это на основании текущего состояния трекинг-веток.
Так как я уже сделал `git fetch` и обновил трекинг-ветки, я могу накатить те обновления с удаленного репозитория, которых у меня еще нет локально:
```Bash
git merge
```
