# команды
`git merge --squash <hash>` - мержит текущую ветку с веткой из хэша, но все коммиты входящей ветки складываются в один коммит.
`git reset --hard <hash>` - **ОСТОРОЖНО!** Эта команда переделывает весь текущий локальный репозиторий в тот, что хранится по хэшу. Эта штука оказалась полезной, когда я делал ветку от другой ветки (а не от `main` ветки) и при MR новой стал видеть все изменения обеих веток в `changes`. Не прикольно видеть те изменения, которые ты не вносил в этой ветке. А эта команда помогает начать историю изменений с чистого листа.
# индекс
У Git существует особая область между удаленным и локальным репозиториями. Эта область называется `индекс`. В ней собираются все коммиты, которые я делаю локально перед тем как запушить их в удаленный репозиторий.
Внутри этой области я управляю тем, что в итоге попадет в удаленный репозиторий. Например, если нужно удалить файлы из удаленного репозитория, то их сначала следует убрать из индекса:
```Bash
git rm --cached <имя файла>
```
При использовании такой команды локально файл будет сохранен, но при этом в следующем пуше его уже не будет. Чтобы точно убрать из удаленного репозитория файл его следует сразу после удаления убрать в `.gitignore`.
Команда:
```Bash
git add <путь до файла или директории>
```
Добавляет файл или директорию в индекс.
# rebase
Допустим получилось так, что в коммит попал нежелательный файл, который следует убрать из удаленного репозитория + убрать из истории коммитов.
Историю коммитов можно редактировать с помощью команды:
```Bash
git rebase -i <имя коммита от которого нам нужно показать историю>
```
Например:
```Bash
git rebase -i HEAD~3
```
Покажет мне три последних коммита.
Далее, есть несколько вариантов действий:

Перед каждым из действующих коммитов написано слово `pick`, что означает, что они используются. Это слово можно заменить на одно из нижеследующих слов. Для меня самые интересные - `squash` и `drop`:
1) `squash` - объединяет изменения коммита с предыдущим коммитом.
2) `drop` - удаляет коммит из истории коммитов.
