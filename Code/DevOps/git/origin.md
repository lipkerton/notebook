`origin` - это "заглушка" на ссылку на репозиторий в гите.
При создании репозитория гита локально мы выполняем такие команды:
```Bash
git init
git add .
git commit -m "initial commit"
```
И после них можем запушить репозиторий на сервер с помощью:
```Bash
git push git@github.com:lipkerton/git-test.git
```
Или не совсем можем - у меня появилась ошибка:
![[git_push_1.png]]В ней говориться, что для моей локальной ветки не существует подходящей ветки на удаленном сервере. Мне предлагают запушить свою локальную ветку с указанием на ту ветку, которая должна принимать пуши с этой локальной на удаленном сервере. Я это сделаю:
```Bash
git push --set-upstream git@github.com:lipkerton/git-test.git main
```
В описании аргументов для `push` про `--set-upstream` сказано, что это команда, которая после удачного пуша в ветку устанавливает репозиторий `git@github.com:lipkerton/git-test.git` и его ветку `main` в качестве значений по умолчанию для всех последующих `push` и `pull`.
Теперь по идее, если я начну делать просто команду `git push`, то изменения локального репозитория будут пушиться в `main` без дополнительных аргументов.
Как я сказал выше - `origin` является заглушкой на ссылку удаленного репозитория гита. Эту заглушку при инициализации репозитория мы можем добавить вручную:
```Bash
git init
git add .
git commit -m "Initial commit"

git remote add origin git@github.com:lipkerton/git-test.git
```
Это значит, что теперь при выполнении первичного пуша можно пользоваться такой командой:
```Bash
git push --set-upstream origin main
```
В случае с одним единственным местом для пуша (`git@github.com:lipkerton/git-test.git`) эта фишка не особенно нужна. Однако, когда у нас добавляется больше удаленных серверов, на которые мы можем пушить, то именование ссылок может помочь.
Например, на одном из проектов у меня есть сразу две точки назначения для пуша - GitHub и GitLab. Список точек можно посмотреть так:
```Bash
git remote -v
```
Вывод примерно такой:
![[git_remote_1.png]]Здесь у меня создано сразу две пары заглушек - `gitlab` и  `origin`.
А почему вообще это называется "origin"? Я не знаю, но именно такое название дает заглушке Git, когда используешь клонирование репозитория. При клонировании автоматически создается заглушка `origin`.
Кстати, а почему создается именно пара записей (для `fetch` и для `push`)? Потому что Git применяет разделение по умолчанию для операций записи и операций чтения. Он это делает потому, что можно настроить разные конфигурации - назначить один эндпоинт для чтения репозитория, а другой для записи в репозиторий. Например, с помощью этой функции можно выставить ограничения на запись.
