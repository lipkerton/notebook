## база
типы - типами называют разные виды значений. Под значениями в данном случае имеются в виду шутки, вроде строк, чисел, чисел с плавающей точкой и т.д. Примерно каждый язык программирования предоставляет "стандартный пакет типов", т.е. пакет поддерживаемых типов и инструменты для работы с ними.
В JS я уже показал какие-то типы:
```JavaScript
> let a;
undefined
> a = 15  // число
15
> a = "Hello!"  // строка
'Hello!'
> a = 15.1  // число с плавающей точкой
15.1
```
Типы в JS условно можно разделить на два вида:
1) элементарные - числа, строки, булевские значения, `null`, `undefined`,
2) объектные типы - объекты, массивы.
>[!important] `null` и `undefined` являются именно элементарными типами, хотя не являются строками или числами. Они называются "специальные величины". К специальным величинам так же принадлежит тип `Symbol`.
### изменяемость и неизменяемость
Элементарные типы в JS являются неизменяемыми, а объектные типы являются изменяемыми.
### преобразуемость
JS может автоматически свободно преобразовывать данные из одного типа в другой, например, булевское значение `true` при выполнении суммы с ним может быть преобразовано интерпретатором в число:
```JavaScript
> true + true
2
```
Именно из-за "либеральных" преобразования типов при выполнении операций сравнения лучше пользоваться оператором тройного равенства, а не его двойным аналогом - тройное равенство не допускает автоматического преобразования типов при выполнении сравнения.
## что означает "объект"?
Объект - это любое число, которое не представляет собой число, строку, булевское значение, `null` или `undefined`.
Объект - это коллекция свойств, где каждое свойство имеет имя и значение (либо элементарное значение, либо другой объект).
Я уже описывал пример такого объекта ранее:
```JavaScript
> a = {
... name:'Peter',
... second_name:'Petukhov',
... birth_date:'25.01.2000'
... };
{ name: 'Peter', second_name: 'Petukhov', birth_date: '25.01.2000' }
```
Здесь свойство - это связка `name:'Peter'`, имя это `name`, а значение - `'Peter'`. Из примера можно сделать вывод, что объект в общем случае - это просто неупорядоченная коллекция свойств.
Существует так же специальный вид объекта - массив:
```JavaScript
> a = [1, 2, 3, 4, a]
[
  1,
  2,
  3,
  4,
  { name: 'Peter', second_name: 'Petukhov', birth_date: '25.01.2000' }
]
```
Вот это уже упорядоченная коллекция нумерованных значений:
```JavaScript
> a[0]
1
> a[a.length - 1]
{ name: 'Peter', second_name: 'Petukhov', birth_date: '25.01.2000' }
```
Существуют так же другие объектные типы:
1) `RegExp` - регулярные выражения.
2) `Date` - работа с датой и временем.
3) `Error` - работа с ошибками.
4) `Set` - множества.
5) `Map` - "отображение ключей на значения" (???).
## а функции и классы?
Функции и классы в JS так же являются значениями, которыми можно манипулировать. Например, я могу сделать несколько функций и "положить" их в список:
```JavaScript
> const x = x => x + 1;
undefined
> const y = y => y + 1;
undefined
> x(1)
2
> y(2)
3
> a = [x, y]
[ [Function: x], [Function: y] ]
> a[0](1)
2
> a[1](2)
3
```
Теперь я сделаю класс и положу внутрь объекта сам класс + его экземпляр:
```JavaScript
> class Test {
... constructor(name) {
... this.name = name
... }}
undefined
> let t = new Test('Peter')
undefined
> a.push(t)
3
> a
[ [Function: x], [Function: y], Test { name: 'Peter' } ]
> a.push(Test)
4
> a
[ [Function: x], [Function: y], Test { name: 'Peter' }, [class Test] ]
```
## сборщик мусора 
Сборщик мусора в JS автоматически освобождает память от неиспользуемых объектов и других значений. Как мне кажется, механизм работает схожим образом с сборщиком мусора Python - ориентируется на текущий счетчик ссылок объекта: если на объект не существует актуальных ссылок в скрипте, то он стирается из памяти.
## константы и переменные
Константы обозначаются ключевым словом `const` - формально запись через `const` означает, что значение переменной уже не будет меняться на протяжении программы.
Переменные обозначаются ключевым словом `let` (ранее `var`) - такая запись означает, что значение переменной может измениться.
Константы и переменные являются нетипизированными. Это значит, что когда мы объявляем переменную мы не указываем ее тип.
