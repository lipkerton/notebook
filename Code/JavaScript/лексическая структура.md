Из необычного - особенность установки точки с запятой в конце строки для обозначения конца какой-нибудь конструкции.
В остальном все почти то же самое, что и было в Python:
1) Пишем код с помощью символов Unicode.
2) В качестве имен переменных есть возможность использовать любые символы Unicode (кроме смайликов).
3) JavaScript чувствителен к регистру.
4) Есть зарезервированные слова.
## точка с запятой
В большинстве случаев разрыв строки воспринимается интерпретатором JS, как конец какой-то синтаксической конструкции, то есть как точка с запятой.
Такая запись:
```JavaScript
let a
a
=
3
```
Будет нормально воспринята интерпретатором.
Определить когда запись нормально будет воспринята интерпретатором, а когда нет можно по смыслу конструкции. Если с точки зрения языка символ, который идет на следующей строке может быть продолжением символа, который идет в предыдущей, то JS прочитает две разорванных строки, как одну. А если у меня, например, на следующей строке идет начало функции, то JS поймет, что разрыв строки обозначает точку с запятой.
### необходимость точки с запятой №1
Однако есть случаи, где точка с запятой необходима. Например, такая запись:
```JavaScript 
x = a + f
(a + b).toString()
```
Скорее всего не будет репрезентировать логику того, кто ее писал. Очевидно, что по задумке выражение сверху должно вычислиться, а вслед за ним должно сработать выражение снизу. Однако в этой записи JS подумает, что скобки во второй строке относятся к `f` и постарается вызвать функцию с таким аргументом. Это случиться, потому что с точки зрения интерпретатора конструкция в следующей строке может продолжить конструкцию в предыдущей. Мы можем разделить конструкции, если установим точку с запятой:
```JavaScript
x = a + f;
(a + b).toString()
```
Существует так же традиция ставить точки с запятыми в самом начале строки перед подобными конструкциями, чтобы любое изменение в коде не могло повлиять на результат их работы.
```JavaScript
x = a + f
; (a + b).toString()
```
### необходимость точки с запятой №2
Когда нужно вернуть значение из функции в JS используется слово `return`. Оно так же используется, когда нужно просто вернуть из функции `null` и прервать ее. Так как `return` прерывает работу с функцией, то следующая строка уже воспринята не будет, поэтому интерпретатор считает только то, что написано после `return`, а любой разрыв строки зачтет, как окончание команды.
```JavaScript
return
true;
```
Здесь `true` никогда не будет возвращен, потому что до него сработает пустой `return`.
