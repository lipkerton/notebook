## переменная
Объявить переменную:
```JavaScript
let x;  // для начала объявляем.
x = 0;  // присваиваем значение.
console.log(x);  // => 0
```
Переменные типа `let` могут хранить и строки, и даже числа с плавающей точкой:
```JavaScript
let a, b;
a = 0.01;
b = "Hello world!";
console.log(a);  // => 0.01
console.log(b);  // => Hello world!
```
В JS есть сразу две разновидности типа `неизвестные данные` или `данные отсутствуют`. Есть обычный `null` и еще `undefined`:
```JavaScript
let a, b;
a = null;
b = undefined;
console.log(a);  // => null;
console.log(b);  // => undefined;
```
## объекты
Самый важный тип данных в JS - объект:
```JavaScript
let book;
book = {
	topic: "JavaScript",
	edition: 7
};
book.topic  // => "JavaScript"
book.edition  // => 7
book["topic"]  // => "JavaScript"
book["edition"]  // => 7
```
Объекты содержат в себе пары `имя: значение`, а после создания объекта к таким парам можно обращаться по ключу (по имени) и получать значение. В сущности это очень похоже на словарь в Python. Так же объекту можно присвоить новые свойства либо через точку `.`, либо через `[]`:
```JavaScript
book.author = "Flanagan";
```
Внутри объекта могут лежать и другие объекты:
```JavaScript
book.contents = {};
```
Пустые скобки в данном случае обозначают пустой объект.
Если обратиться к несуществующему в объекте свойству, то получится ошибка `TypeError`:
```JavaScript
book.contents.ch01.sect1;
TypeError: undefined is not an object (evaluating 'book.contents.ch01.sect1')
```
Однако в ситуации, когда не знаешь есть ли у объекта нужное свойство можно воспользоваться знаком вопроса `?`:
```JavaScript
book.contents.ch01?.sect1;
```
Тогда ошибка не появится.
Объекты могут хранить внутри себя списки:
```JavaScript
let obj = {x: [1, 2, 3], y: [4, 5, 6]};
obj.x  // => [1, 2, 3]
```
## массивы
JS поддерживает массивы. Но массивы, как и в Python, являются здесь изменяемыми и могут содержать разные значения - число, строку, еще один массив, объекты и проч.
```JavaScript
let primes = [1, 2, 3, 4];
primes  // => [1, 2, 3, 4]
```
Теперь заменим элемент на строку:
```JavaScript
primes = [1, 2, 3, "Hello world!"];
primes  // => [1, 2, 3, "Hello world!"]
```
Индексация работает:
```JavaScript
primes[0]  // => 1
primes[4] = 4;  // меняем элемент по индексу.
primes  // => [1, 2, 3, 4]
```
А вот срезов здесь нет:
```JavaScript
primes[0:2]
SyntaxError: Unexpected token ':'. Expected ']' to end a subscript expression.
```
Можно получить длину массива из атрибута `length`:
```JavaScript
primes.length;  // => 4
primes[primes.length - 1]  // получаем последний элемент.
```
Массивы могут хранить объекты:
```JavaScript
obj = [{x: [1, 2, 3]}, {y: [4, 5, 6]}];
obj[0].x  // => [1, 2, 3]
```
## операции
Операции - это самый распространенный тип выражения в JavaScript. Выражение в JavaScript - это фраза на языке, которая вычисляется и после возвращает значение. Так, использование квадратных скобок для получения атрибута объекта - является выражением.
### математические
Операций есть много. Начну с обычных математических:
```JavaScript
3 + 2
3 - 2
3 * 2
3 / 2  // деление выдает число с плавающей точкой.
3 ** 2  // возведение в степень.
3 % 2  // нахождение остатка.
```
Целочисленого деления как в Python (`//`) тут нет.
Есть сокращение этих операций, как в Python:
```JavaScript
let count = 0;
count += 2;
count *= 3;
```
А еще есть специальные операции увеличения и уменьшения на единицу:
```JavaScript
count++;  // => называется инкремент.
count--;  // => называется декремент.
```
### сравнения
JavaScript использует все стандартные операции сравнения:
```JavaScript
let x = 2, y = 3;
x < y
x > y
x <= y
x >= y
x != y
x == y
```
А еще у него есть свой нестандартный способ сравнивать объекты - через знак тройного равенства (и неравенства):
```JavaScript
x === y
x !== y
```
На таком простом примере не совсем очевидно что делает тройное равенство. На самом деле оно проверяет идентичность двух объектов. В Python значения `0, [], {}` и проч. равны `false`. В JS, если мы будем проверять с помощью двойного равенства, будет такая же картина. Однако если мы будем сравнивать через тройное равенство, то оно уже потребует не просто равенства двух операторов, но их идентичности. Пример:
```JavaScript
0 == false;  // => true
0 === false;  // => false
```
###  логические
Все по стандарту - И, ИЛИ, НЕ.
```JavaScript
(x == 2) && (y == 3)  // логическое И.
(x > 2) || (y == 3)  // логическое ИЛИ.
!(x == y)  // логическое НЕ.
```
## терминология
В книге описано несколько понятий: 
1) Выражения - любой корректный блок кода, который возвращает значение.
2) Операторы - то, что используется для вычисления значения (например, оператор присваивания, оператор сравнения, логический оператор).
3) Управляющие структуры - подвид операторов; под управляющими структурами имеются в виду циклы, условные операторы и проч.
