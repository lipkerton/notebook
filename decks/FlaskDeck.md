Какой объект нужно создать, чтобы запустить минимальный сервер + задействовать в нем статику? #flashcard 
Нужно сделать экземпляр класса `Flask` и передать в него параметры `__name__`, `static_folder`, `static_url_path`:
```Python
app = Flask(
	__name__, static_folder='./static', static_url_path='/static'
)
```

Для чего нужен аргумент `static_folder` классу `Flask`? #flashcard 
Он нужен, чтобы прописать относительный (или абсолютный) путь до папки, которая будет хранить файлы статики. Это необходимо, чтобы метод `send_static_file()` и другие подобные знали откуда брать статику, чтобы отправить ее пользователю.
<!--ID: 1744971556605-->

Для чего нужен аргумент `static_url_path` классу `Flask`? #flashcard 
В сущности это указатель на папку со статикой в URL-формате. Это означает, что это путь от корневой папки проекта (от папки в которой лежит исполняемый файл). Через этот аргумент HTML-файлы в первую очередь будут искать статику. Однако он не обязательный - если этого аргумента не будет, то статика будет искаться по аргументу `static_folder`.

Для чего нужен метод `send_static_file()` и что в него нужно передавать? К экземпляру какого класса он применяется? #flashcard 
Это метод, который возвращает пользователю статический файл. Передать в этот метод нужно имя статического файла, чтобы он нашел его в папке статики. Этот метод применяется к экземпляру класса `Flask`.

Для чего нужен метод `run()` и что в него нужно передавать? К экземпляру какого класса он применяется? #flashcard 
Это метод, который запускает сервер и заставляет его слушать запросы пользователей. Передать в него можно аргументы `port` и `debug`. Аргумент `port` определяет на каком порте сервер будет слушать запросы. Аргумент `debug` принимает либо `True`, либо `False`; если значение `True`, то сервер запустится в режиме разработки - это означает, что любая ошибка обращения к ресурсу будет подробно описана.

Что такое `@app.route()`? #flashcard
Это декоратор и маршрутизатор. В скобки передается путь, например, `/` или `/catalog`, - это путь, на который пользователь может послать запрос. Маршрутизатор перекинет запрос пользователя в функцию над, которой стоит соответствующий маршрутизатор. `app` - это экземпляр класса `Flask`.
Если пользователь обращается к ресурсу `catalog`, то его запрос будет перекинут в функцию `catalog` таким образом:
```Python
@app.route('/catalog')
def catalog():
	pass
```

Как можно передать аргументы от клиента в функцию на сервер? #flashcard 
Два варианта:
1) Прямолинейно записать в URL в таком формате:
```
http://www.example.com/echo/<name>/<title>
```
2) Передать вместе с параметрами GET-запроса:
```
http://www.example.com/echo?name=Peter&title=Cool
```

Как можно вытащить аргументы из запроса внутри функции? #flashcard 
Если это аргументы, которые находятся в самом запросе, то таким образом:
```Python
@app.route('/echo/<name>/<title>')
def echo(name, title):
	pass
```
А если это аргументы, которые переданы в качестве GET-параметров, то таким:
```Python
@app.route('/echo/')
def echo():
	from flask import request
	name = request.args.get('name')
	title = request.args.get('title')
```
