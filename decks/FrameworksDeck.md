Для чего нужен аргумент `static_folder` классу `Flask`? #flashcard 
Он нужен, чтобы прописать относительный (или абсолютный) путь до папки, которая будет хранить файлы статики. Это необходимо, чтобы метод `send_static_file()` и другие подобные знали откуда брать статику, чтобы отправить ее пользователю.

Для чего нужен аргумент `static_url_path` классу `Flask`? #flashcard 
В сущности это указатель на папку со статикой в URL-формате. Это означает, что это путь от корневой папки проекта (от папки в которой лежит исполняемый файл). Через этот аргумент HTML-файлы в первую очередь будут искать статику. Однако он не обязательный - если этого аргумента не будет, то статика будет искаться по аргументу `static_folder`.

Для чего нужен метод `run()` и что в него нужно передавать? К экземпляру какого класса он применяется? #flashcard

Что такое `@app.route()`? #flashcard
Это декоратор и маршрутизатор. В скобки передается путь, например, `/` или `/catalog`, - это путь, на который пользователь может послать запрос. Маршрутизатор перекинет запрос пользователя в функцию над, которой стоит соответствующий маршрутизатор. `app` - это экземпляр класса `Flask`. Если пользователь обращается к ресурсу `catalog`, то его запрос будет перекинут в функцию `catalog` таким образом:
```Python
@app.route('/catalog')
def catalog():
	pass
```

Как можно вытащить аргументы из запроса внутри функции? #flashcard 
Если это аргументы, которые находятся в самом запросе, то таким образом:
```Python
@app.route('/echo/<name>/<title>')
def echo(name, title):
	pass
```
А если это аргументы, которые переданы в качестве GET-параметров, то таким:
```Python
@app.route('/echo/')
def echo():
	from flask import request
	name = request.args.get('name')
	title = request.args.get('title')
```

Как можно настроить отдельный маршрутизатор (отдельный `urls.py`) для каждого приложения (Django)? #flashcard 
Нужно в главном `urls.py`, который находится рядом с файлом `settings.py`, пробросить маршруты в отдельные для каждого приложения `urls.py` с помощью функции `include`:
```Python
from django.urls import path, include
urlpatterns = [
	path('', include('homepage.urls')),
	path('catalog/', include('catalog.urls'))
]
```
При этом в самих приложениях `urls.py` будет выглядеть так (на примере `catalog`):
```Python
from django.urls import path
from . import views
urlpatterns = [
	path('', views.catalog_main_page)
]
```

Как передать переменную из URL в функцию (Django)? #flashcard 
Чтобы это сделать нужно прописать в маршрутизаторе шаблон такого вида: `catalog/<pk>/`. Django будет считать, что он может подставить на место `<pk>` какое-нибудь значение из полученного запроса. Далее, для шаблона вызывается функция и эта функция по умолчанию обязательно будет принимать аргумент (даже если он не будет использован внутри нее):
```Python
def func_with_args(request, pk):
	pass
```

Как базово устроена маршрутизация запросов в Django? #flashcard 
От пользователя приходит запрос `HttpRequest`. Этот запрос просит конкретный ресурс на сайте по ссылке. Сначала он отправляется в маршрутизатор (`urls.py`) - ссылка поочередно сравнивается с шаблонами, которые есть в `urls.py`. Как только находится подходящий под ссылку шаблон объект `HttpRequest` отправляется в view-функцию для этого шаблона. View-функция обрабатывает запрос и возвращает в ответ объект`HttpResponse`.

Что такое миграции в Django? #flashcard 
Миграции в Django - это инструмент, который позволяет разработчику не писать сырые SQL запросы, а прописать все что нужно в виде моделей на Python. После того как модели описаны Django может автоматически создать всю необходимую структуру в выбранной СУБД. Формально таким образом Django сам управляет БД, создавая в ней таблицы и связи.

Что нужно сделать, чтобы применить миграции? #flashcard 
Прежде чем применить миграции нужно сначала создать файлы миграций, т.е. сначала мы проанализируем существующие модели и напишем файлы для будущих миграций. Это можно сделать с помощью команды:
```Bash
python manage.py makemigrations
```
Далее, можно применить сами миграции на основе файлов миграций:
```Bash
python manage.py migrate
```

Как выглядят параметры GET запроса? #flashcard 
Под "параметрами запроса" имеются в виду аргументы, которые записываются после тела запроса. От дела запроса они отделяются знаком `?` и далее маркируются в формате `<имя_аргумента>=<значение_аргумента>`. Друг от друга они отделяются знаком `&`:
```
http://localhost:6789/catalog?item=1488&size=228
```
