Что такое Docker? #flashcard 
Docker - это приложение, которое позволяет запаковывать проекты в контейнеры.

Что такое контейнер в Docker? #flashcard 
Контейнер - это среда для конкретного проекта или части проекта (например, для бэкенд части проекта), где развернута ОС и все необходимые дополнения для проекта. Контейнер создается на основании образа.
<!--ID: 1758870121708-->


Что такое образ в Docker? #flashcard 
Образ - это первая ступень перед созданием контейнера. В образ пишутся все шаги, которые необходимо выполнить перед развертыванием полноценного контейнера. Образ составляется на основании `Dockerfile`.
<!--ID: 1758870121709-->


Как собрать образ?  #flashcard 
Сначала нужно сделать `Dockerfile`. Затем:
```Bash
docker build -t <имя образа> <место Dockerfile>
```

Как запустить контейнер на основании образа? #flashcard 
```Bash
docker run --name <имя контейнера> -p <порт хоста>:<порт контейнера> <имя образа>
```

Что означает связка `8000:8000` в контексте записи портов в `Dockerfile` и в `docker-compose.yml`? #flashcard 
Первый порт - это порт, обращаясь на который хост может достучаться до контейнера. А второй порт - это порт на котором контейнер прослушивает обращения по первому порту.

Что такое `volume` в Docker? #flashcard
`volume` - это том (="хранилище"), который может независимо от контейнера сохранять какие-то данные. Например, если у нас есть БД в контейнере, то неплохо было бы хранить данные БД отдельно от контейнера с ней на случай, если контейнер понадобится переместить или перезапустить.

Что такое `docker-compose`? #flashcard 
`docker-compose` - это часть экосистемы докера, которая решает проблему запуска и взаимодействия сразу нескольких контейнеров. Например, у меня есть мое приложение FastAPI и еще я хочу добавить к нему БД - это два контейнера, которые должны как-то общаться между собой.

Как создать минимальный `docker-compose`, который будет собирать один образ и запускать один контейнер на основании `Dockerfile` (контейнер должен слушать хоста на 8000 порте, запросы хоста должны маршрутизироваться в контейнер так же через 8000-й порт)?  #flashcard 
```YAML
services:
	blog:
		build: .
		ports:
			- "8000:8000"
```

Как добавить к сервису в `docker-compose` новый том для хранения данных? #flashcard 
```YAML
services:
	blog:
		build: .
		ports:
			- "8000:8000"
		volumes:
			- blog:/path/to/dir  # путь к папке, содержимое которой мы хотим хранить.
# "регистрация" тома
volumes:
	blog:
```

Что такое `network` в контексте Docker? #flashcard 
`network` - это сеть, в которой состоит контейнер. Сети нужны, чтобы контейнеры могли общаться друг с другом. Например, контейнер с БД и контейнер с приложением на FastAPI можно заставить общаться между собой, если объединить их в одну сеть - так они будут обмениваться сообщениями, будто стоят в одной системе.

Как заставить контейнеры Docker запускаться в нужном порядке при оркестрации? #flashcard 
```YAML
services:
	blog:
		depends_on:  # заставляем этот контейнер зависеть от запуска контейнера под названием `db`.
			- db
		build: .
		ports:
			- "8000:8000"
		volumes:
			- blog:/path/to/dir
	db:
		...
```
