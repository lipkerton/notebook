Что такое MVC? #flashcard 
Это архитектура приложения. Model-View-Controller. Model - часть приложения выполняет всю бизнес-логику. View - это "лицо" приложения, то, что показывается пользователю. Controller - центр архитектуры; обрабатывает пользовательские запросы, которые получает View, отравляет что-то на обработку в Model, готовит ответ для пользователя и приказывает View показать этот ответ.

Что такое MVP? #flashcard 
Это архитектура приложения. Model-View-Presenter. Model - часть приложения выполняет всю бизнес-логику. View - это "лицо" приложения, то, что показывается пользователю. Presenter - центр архитектуры, обрабатывает все, что происходит с View, отправляет что-то на обработку в Model, самостоятельно обновляет View, чтобы опубликовать ответ пользователю.

Что такое MVVM? #flashcard 
Это архитектура приложения. Model-View-ModelView. Model - часть приложения выполняет всю бизнес-логику. View - это "лицо" приложения, то, что показывается пользователю. ModelView - то, что делает компоненты автономными в этой модели; она не управляет другими компонентами приложения, она имеет свои объекты внутри себя, обновляя которые дает сигнал Model и View, что нужно выполнить работу с обновленными объектами.

Что такое DTO? #flashcard 
DTO = Data Transfer Object. Это объект, который переносит данные от одного компонента архитектуры к другому. Важно, что DTO - это комплект данных, на которые поступил запрос. Например, в CQRS часть приложения `Query` читает данные из БД и направляет на обработку пакетом в формате DTO.

Что такое OOP? #flashcard 
OOP = Object Oriented Programming. Способ конструирования программы через классы и их объекты, где каждый класс репрезентирует какую-нибудь абстракцию, например, "программист", а каждый объект представляет собой конкретный экземпляр этой абстракции, например, "программист-Толя". каждый объект в такой системе имеет публичный интерфейс и приватное состояние, которое доступно только ему.
<!--ID: 1770103093259-->


Чтобы реализовать OOP-программу существует несколько основных принципов. Что это за принципы? #flashcard 
Абстракция - классы являются репрезентацией абстракций. Наследование - классы могут наследоваться друг от друга; классы-наследники сохраняют методы и атрибуты классов-родителей + имеют доступ к их приватным интерфейсам. Полиморфизм - связанные классы могут иметь схожие методы и предоставлять одинаковый интерфейс для них (например, два класса-потомка могут переопределять родительский метод, но метод все еще сохраняет свое имя). Инкапсуляция - классы могут иметь свои приватные атрибуты и приватные методы, которые не являются частью их публичного интерфейса.
