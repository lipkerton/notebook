Зачем нужен venv? #flashcard 
Чтобы держать все нужные для проекта пакеты в одном месте внутри проекта. Например, это упрощает транспортировку проекта с компа на комп или в GitHub, т.к. из папки виртуального окружения можно вытащить все зависимости и их версии, чтобы сохранить их в специальном файле. Еще venv по умолчанию необходим, например, на macOS, который просто не позволяет устанавливать файлы с помощью `pip` без виртуального окружения.

```Bash

```

Как активировать venv на винде? #flashcard 
```PowerShell
.\<имя папки venv>\Scripts\activate
```
Иногда может возникнуть ошибка - винда может не позволить активировать сторонний скрипт. Тогда можно снять этот запрет командой:
```PowerShell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned
```

Как активировать на маке и линуксе? #flashcard

Что происходит при активации виртуального окружения? #flashcard 
Скрипт активации виртуального окружения изменяет переменную `PATH`. Он ставит путь до папки с виртуальным окружением перед путями к стандартным папкам Python. Делает он это для того, чтобы Python сначала искал пакеты в активированном виртуальном окружении, а потом уже в своих стандартных папках.
