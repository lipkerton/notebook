Почему запрос `select * from table` вреден? Назови основные причины. #flashcard

Как считается стоимость запроса в PostgreSQL? #flashcard 
По примерной формуле, которая заложена в оптимизатор: считаем I/O (input-output операции) и складываем с оценкой стоимости расходуемых ресурсов компьютера. Например:
```SQL
SELECT
	relpages, --количество страниц в таблице
	current_setting('seq_page_cost'), --стоимость чтения одной страницы
	relpages * current_setting('seq_page_cost')::int as total --итог
FROM pg_class where relname='species';
```
Так мы посчитаем стоимость прочтения всех страниц. Теперь нужно посчитать затраты процессора:
```SQL
SELECT
	reltuples, --количество строк
	current_setting('cpu_tuple_cost'), --стоимость чтения одной строки
	reltuples * current_setting('cpu_tuple_cost')::float as total
FROM pg_class where relname='species';
```

Что такое hash join? #flashcard

Тогда, чтобы соединить большую табличку с маленькой табличкой, нам нужно будет использовать хэш-функцию для перевода данных большой таблички в хэш, а затем сравнивать эти данные с данными маленькой таблички в оперативной памяти.

Что такое merge join? #flashcard

Используется реже, чем другие типы соединений, потому что оно предназначено только для тех случаев, в которых данные обеих таблиц отсортированы или могут быть очень просто отсортированы. Сравнение происходит по размеру столбцов соединяемых табличек: если столбцы одинаковы, то мы их соединяем, а если один или другой меньше, то идем по строчке дальше.

Какие виды индексов существуют? #flashcard 
1) Составные - по нескольким колонкам.
2) Покрывающие - это когда мы добавляем к основным данным в индексе дополнительные данные.

Какие настройки PostgreSQL лучше поменять, если работа СУБД происходит на хорошем сервере? #flashcard 
1) `work_mem` - по умолчанию равняется 4 MB, лучше обновить на 64 МБ или больше, если позволяют ресурсы. Это значение определяет сколько памяти (какой?) доступно для каждой операции запроса, прежде чем эта операция начнет писать данные во временные файлы на диске. Чем больше памяти - тем меньше нужно будет действий, которые связаны с медленной записью на диск.
2) `hash_mem_multiplier` - это множитель для `work_mem`, который используется оптимизатором для выполнения операций с hash join/
3) `random_page_cost` - это константа, которую оптимизатор использует в расчетах. Она означает стоимость чтения страницы с диска. Изначально стоит значение 4. Можно уменьшить до 1.1.
4) `shared_buffers` - увеличиваем буффер.

Что такое схема? #flashcard

Что такое табличные пространства? #flashcard
Это штуки, которые отвечают за физическое хранение данных на диске. Табличных пространств (tablespaces) в рамках одной БД может быть несколько - часть данных будет храниться в одном табличном пространстве, а другая часть в другом. Табличное пространство - это просто каталог в системе компьютера, который наполнен данными. Одно табличное пространство может хранить данные разных баз данных.
